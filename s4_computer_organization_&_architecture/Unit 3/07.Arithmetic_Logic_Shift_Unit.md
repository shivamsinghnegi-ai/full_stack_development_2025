# Arithmetic Logic Shift Unit (ALU)
---

## 1. What is an ALU?

The **Arithmetic Logic Shift Unit (ALU)** is the **core functional unit of the CPU** that combines all three types of micro-operations — arithmetic, logic, and shift — into a **single hardware unit**.

**Key Point:**
> ALU does **NOT store data** — it only **processes** it.
> Data comes FROM registers → ALU processes it → Result goes BACK to registers.

---

## 2. Building Blocks of an ALU

An ALU is built by combining 4 components:

| Component | Operations Handled |
|-----------|-------------------|
| **Arithmetic Circuit** | Add, Subtract, Increment, Decrement |
| **Logic Circuit** | AND, OR, XOR, NOT |
| **Shift Circuit** | Logical (shl/shr), Arithmetic, Circular (cil/cir) |
| **MUX (Multiplexer)** | Selects which result goes to the output |

>  All three units compute their results **simultaneously**. The MUX then **selects** which result to send to the output based on selection inputs.

## 4. Operation Selection Table

The ALU uses **4 selection lines (S3, S2, S1, S0)** and **Cin** to select the operation.

> **Quick Memory Trick for S3, S2:**
> - **S3=0, S2=0** → Arithmetic operation
> - **S3=0, S2=1** → Logic operation
> - **S3=1, S2=0** → Shift operation

### Complete Table

| S3 | S2 | S1 | S0 | Cin | Operation     | Description         |
|----|----|----|----|-----|---------------|---------------------|
| 0  | 0  | 0  | 0  |  0  | F = A         | Transfer A          |
| 0  | 0  | 0  | 0  |  1  | F = A + 1     | Increment A         |
| 0  | 0  | 0  | 1  |  0  | F = A + B     | Addition            |
| 0  | 0  | 0  | 1  |  1  | F = A + B + 1 | Add with Carry      |
| 0  | 0  | 1  | 0  |  0  | F = A + B̄     | Subtract with Borrow|
| 0  | 0  | 1  | 0  |  1  | F = A − B     | Subtraction         |
| 0  | 0  | 1  | 1  |  0  | F = A − 1     | Decrement A         |
| 0  | 1  | 0  | 0  |  X  | F = A ∧ B     | AND                 |
| 0  | 1  | 0  | 1  |  X  | F = A ∨ B     | OR                  |
| 0  | 1  | 1  | 0  |  X  | F = A ⊕ B     | XOR                 |
| 0  | 1  | 1  | 1  |  X  | F = Ā         | Complement (NOT A)  |
| 1  | 0  | 0  | 0  |  X  | F = shr A     | Logical Shift Right |
| 1  | 0  | 0  | 1  |  X  | F = shl A     | Logical Shift Left  |
| 1  | 0  | 1  | 0  |  X  | F = cir A     | Circular Shift Right|
| 1  | 0  | 1  | 1  |  X  | F = cil A     | Circular Shift Left |

>  **X means "don't care"** — for logic and shift operations, Cin has no effect.

### Group-wise Pattern (Easy Memorization)

```
S3=0, S2=0  (S1 S0 Cin vary) → ARITHMETIC group (7 operations)
S3=0, S2=1  (S1 S0 vary)     → LOGIC group      (4 operations)
S3=1, S2=0  (S1 S0 vary)     → SHIFT group      (4 operations)
```

## 5. Status Flags in ALU

After **every ALU operation**, 4 flag bits are automatically updated in the **Flag Register**.

| Flag | Full Name | Set to 1 When... | Example |
|------|-----------|------------------|---------|
| **C** | Carry Flag | Carry generated from MSB (unsigned overflow) | 255 + 1 in 8-bit |
| **Z** | Zero Flag | Result = 0000...0 (all zeros) | 5 − 5 = 0 |
| **S** | Sign Flag | MSB of result = 1 (result is negative) | Any negative result |
| **V** | Overflow Flag | Signed arithmetic overflow (sign bit wrong) | 127 + 1 in 8-bit signed |

### Example — Flags in Action

```
Operation: A = 1111 1111 (255),  B = 0000 0001 (1)
F = A + B = 1 0000 0000

C = 1  (carry out from MSB — 9th bit generated)
Z = 0  (result ≠ 0, ignoring carry)
S = 0  (MSB of 8-bit result = 0)
V = 0  (unsigned overflow, not signed)
```

```
Operation: A = 0000 0101 (5),  B = 0000 0101 (5)
F = A − B = 0000 0000

C = 0
Z = 1  ← Zero flag SET (result = 0)
S = 0
V = 0
```

>  **Flags are used by the Control Unit** to make decisions — for example, in conditional jump instructions: "Jump if Zero (Z=1)", "Jump if Carry (C=1)" etc.

## 6. How ALU Connects to the CPU

```
         ┌─────────────────┐
         │  Register File  │  (R0, R1, R2 ... Rn)
         └────┬────────┬───┘
              │        │
          Operand A  Operand B
              │        │
              ↓        ↓
           ┌──────────────┐
           │     ALU      │ ← Selection inputs (S3,S2,S1,S0,Cin)
           └──────┬───────┘
                  │
              Result F
                  │
         ┌────────┴────────┐
         │                 │
         ↓                 ↓
   Register File     Flag Register
   (store result)    (C, Z, S, V updated)
```

**Important Rules:**
- ALU inputs always come from **registers** — never directly from memory
- Before ALU can operate, data must be **loaded into registers** from memory
- After ALU operates, result must be **stored back** to register or memory

---

## 7. Quick Revision Summary

```
ALU = Arithmetic Unit + Logic Unit + Shift Unit + MUX

Selection Lines:
  S3=0, S2=0 → Arithmetic  (add, sub, inc, dec)
  S3=0, S2=1 → Logic       (AND, OR, XOR, NOT)
  S3=1, S2=0 → Shift       (shl, shr, cil, cir)

Flags (updated after every operation):
  C → Carry        (unsigned overflow)
  Z → Zero         (result = 0)
  S → Sign         (result is negative)
  V → Overflow     (signed overflow)

ALU Rule:
  Input from Registers → ALU processes → Output to Registers + Flags updated
```
