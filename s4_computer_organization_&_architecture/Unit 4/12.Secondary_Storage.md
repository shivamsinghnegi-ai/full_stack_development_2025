# Secondary Storage
 
### 1. What is Secondary Storage?
 
Secondary storage is **non-volatile, large capacity storage** used to permanently store data and programs.
 
Unlike RAM (volatile, fast, small), secondary storage:
- Retains data **even when power is off**
- Is **much larger** in capacity (GBs to TBs)
- Is **much slower** than RAM
- Is used for **permanent storage** — OS, programs, files, databases
 
**Examples:**
- HDD (Hard Disk Drive)
- SSD (Solid State Drive)
- USB Flash drives
- Optical discs (CD, DVD, Blu-ray)
- Magnetic tapes
 
---
 
### 2. HDD — Hard Disk Drive
 
HDD is the most commonly studied secondary storage in COA exams.
 
#### Physical Structure of HDD
 
```
                    Spindle (motor)
                         │
              ┌──────────┼──────────┐
              │  ┌───────┴───────┐  │
              │  │   Platter 1   │  │← Magnetic disk
              │  └───────────────┘  │
              │  ┌───────────────┐  │
              │  │   Platter 2   │  │← Magnetic disk
              │  └───────────────┘  │
              │  ┌───────────────┐  │
              │  │   Platter 3   │  │← Magnetic disk
              │  └───────────────┘  │
              └─────────────────────┘
                    Read/Write
                      Heads ←── Arm Assembly (actuator)
```
 
#### Key Components
 
| Component | Description |
|---|---|
| **Platter** | Circular magnetic disk where data is stored. HDDs have multiple platters stacked on a spindle. Made of aluminium/glass coated with magnetic material. |
| **Spindle** | Motor that rotates all platters at constant speed — typically 5400 RPM or 7200 RPM |
| **Read/Write Head** | Tiny electromagnetic device that reads/writes data by sensing/changing magnetic polarization. One head per platter surface. |
| **Actuator Arm** | Mechanical arm that moves the read/write heads across the platter surface |
| **Track** | Concentric circular path on a platter surface where data is stored |
| **Sector** | Smallest unit of storage on a track — typically 512 bytes or 4096 bytes |
| **Cylinder** | Set of all tracks at the same radial position across all platters |
 
---
 
#### HDD Data Organization
 
```
Platter (top view):
        ┌─────────────────────────────┐
        │     Track 0 (outermost)     │
        │   ┌─────────────────────┐   │
        │   │      Track 1        │   │
        │   │  ┌───────────────┐  │   │
        │   │  │    Track 2    │  │   │
        │   │  │   (innermost) │  │   │
        │   │  └───────────────┘  │   │
        │   └─────────────────────┘   │
        └─────────────────────────────┘
 
Each track is divided into sectors:
Track → [Sector 0][Sector 1][Sector 2]...[Sector N]
 
Cylinder = Track 2 on Platter 1 + Track 2 on Platter 2 + Track 2 on Platter 3
           (same radial position, all platters)
```

### 3. HDD Access Time — 3 Components 
 
To read or write data on HDD, three time components are involved:
 
#### Component 1 — Seek Time (Ts)
 
> Time taken for the **read/write head to move to the correct track.**
 
The actuator arm physically moves the head from its current track to the required track.
 
- **Minimum seek time** — moving to adjacent track
- **Maximum seek time** — moving from innermost to outermost track
- **Average seek time** ≈ **1/3 of maximum seek time** (commonly used in numericals)
 
Typical values: **5 ms to 15 ms**
 
```
Seek Time = time for arm to move to correct track
```
 
---
 
#### Component 2 — Rotational Latency (Tr) — Also called Rotational Delay
 
> Time taken for the **required sector to rotate under the read/write head** after the head reaches the correct track.
 
Once the head is on the right track, the platter keeps spinning. The head waits for the required sector to come under it.
 
- **Minimum rotational latency** = 0 (sector is already under head — lucky!)
- **Maximum rotational latency** = full rotation time (sector just passed the head)
- **Average rotational latency** = **half of full rotation time**
 
```
Average Rotational Latency = (1/2) × (1/RPM) × 60 seconds
                           = 30/RPM seconds
                           = (30/RPM) × 1000 ms
```
 
**Example:**
- Disk speed = 7200 RPM
- Time for one full rotation = 60/7200 = 1/120 seconds = 8.33 ms
- Average rotational latency = 8.33/2 = **4.17 ms**
 
---
 
#### Component 3 — Transfer Time (Tt)
 
> Time taken to **actually transfer the data** from the disk to memory (or vice versa) once the head is positioned.
 
```
Transfer Time = Data to transfer / Transfer Rate
```
 
Or:
 
```
Transfer Time = (Number of bytes to transfer) / (Disk transfer rate in bytes/sec)
```
 
Typical transfer rates: 100 MB/s to 200 MB/s for HDD
 
---
 
### 4. Total HDD Access Time Formula 
 
```
Total Access Time = Seek Time + Rotational Latency + Transfer Time
                  = Ts + Tr + Tt
```

### 5. HDD Access Time — Numericals 
 
#### Numerical 1 — Basic
 
**Given:**
- Average seek time = 8 ms
- Disk rotation speed = 6000 RPM
- Transfer rate = 4 MB/s
- Data to transfer = 8 KB
 
**Find: Total average access time**
 
**Step 1 — Rotational latency:**
```
One full rotation = 60/6000 = 0.01 seconds = 10 ms
Average rotational latency = 10/2 = 5 ms
```
 
**Step 2 — Transfer time:**
```
Transfer time = 8 KB / 4 MB per second
              = 8 × 1024 bytes / (4 × 1024 × 1024 bytes per second)
              = 8192 / 4,194,304
              = 0.00195 seconds
              ≈ 2 ms
```
 
**Step 3 — Total access time:**
```
Total = Seek + Rotational + Transfer
      = 8 + 5 + 2
      = 15 ms
```
 
---
 
#### Numerical 2 — Find Rotational Latency from RPM
 
**Given:**
- Disk speed = 7200 RPM
 
**Find: Average rotational latency**
 
```
Time for 1 rotation = 60 / 7200 = 1/120 second = 8.33 ms
Average rotational latency = 8.33 / 2 = 4.17 ms
```
 
---
 
#### Numerical 3 — Full calculation
 
**Given:**
- Seek time = 10 ms
- Rotational speed = 5400 RPM
- Sector size = 512 bytes
- Transfer rate = 5 MB/s
 
**Find: Total access time to read 1 sector**
 
```
Rotational latency = (60/5400)/2 = (11.11 ms)/2 = 5.56 ms
 
Transfer time = 512 bytes / (5 × 1024 × 1024 bytes/s)
              = 512 / 5,242,880
              = 0.0000977 s ≈ 0.1 ms
 
Total = 10 + 5.56 + 0.1 = 15.66 ms
```

### 6. Disk Scheduling (Brief — Awareness)
 
When multiple processes request different tracks, the OS uses **disk scheduling algorithms** to decide the order in which requests are served — to minimize total seek time.
 
| Algorithm | Rule |
|---|---|
| **FCFS** | First Come First Served — requests served in arrival order |
| **SSTF** | Shortest Seek Time First — serve nearest track first |
| **SCAN** | Elevator algorithm — head moves in one direction, serves all requests, then reverses |
| **C-SCAN** | Circular SCAN — head moves in one direction only, jumps back to start |
 
> **Exam note:** SSTF and SCAN are most commonly asked. Know the concept — detailed numericals are rare in COA (more common in OS subject).
