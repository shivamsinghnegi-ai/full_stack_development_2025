# Unit 4: Memory System



## Memory Hierarchy
 
### 1. Why Do We Need a Memory Hierarchy?
 
A CPU processes data extremely fast. But a single type of memory cannot satisfy all three requirements at once:
 
| Requirement | Problem |
|---|---|
| Fast speed | Fast memory (SRAM) is very expensive |
| Large capacity | Large memory (HDD) is very slow |
| Low cost | Cheap memory cannot keep up with CPU speed |
 
**Solution → Memory Hierarchy**
Use multiple levels of memory, each with a different speed, size, and cost — and let the system manage data movement between levels automatically.
 
---
 
### 2. The Memory Hierarchy (Top to Bottom)
 
```
        ┌─────────────┐
        │  Registers  │  ← Fastest, Smallest, Most Expensive
        └──────┬──────┘
               │
        ┌──────▼──────┐
        │  Cache (L1/L2/L3) │
        └──────┬──────┘
               │
        ┌──────▼──────┐
        │  Main Memory (RAM) │
        └──────┬──────┘
               │
        ┌──────▼──────┐
        │  Secondary Storage (HDD/SSD) │
        └──────┬──────┘
               │
        ┌──────▼──────┐
        │  Tertiary Storage (Tape, Optical) │  ← Slowest, Largest, Cheapest
        └─────────────┘
```
 
**Key Rule:**
- Going **UP** → Speed ↑, Cost ↑, Size ↓
- Going **DOWN** → Speed ↓, Cost ↓, Size ↑
 
---
 
### 3. Properties of Each Level
 
| Level | Technology | Speed | Size | Cost/bit | Volatile? |
|---|---|---|---|---|---|
| Registers | Flip-flops | ~1 ns | Bytes | Very high | Yes |
| Cache (L1) | SRAM | ~2–5 ns | KBs | High | Yes |
| Cache (L2/L3) | SRAM | ~5–20 ns | MBs | High | Yes |
| Main Memory | DRAM | ~50–100 ns | GBs | Medium | Yes |
| SSD | Flash | ~0.1 ms | TBs | Low | No |
| HDD | Magnetic | ~5–10 ms | TBs | Very low | No |
| Tape | Magnetic | Seconds | PBs | Lowest | No |
 
> **Volatile** = loses data when power is off.
> **Non-volatile** = retains data without power.
 
---
 
### 4. Principle of Locality
 
The memory hierarchy works because of two principles that describe how programs access data:
 
#### Temporal Locality
> **"If a memory location is accessed, it is likely to be accessed again soon."**
 
**Example:** A variable used inside a loop is accessed many times.
 
#### Spatial Locality
> **"If a memory location is accessed, nearby locations are likely to be accessed soon."**
 
**Example:** Accessing array elements — if `a[0]` is accessed, `a[1]`, `a[2]` etc. are likely accessed next.
 
**Why it matters:** Cache memory exploits both types of locality to keep frequently used data close to the CPU, reducing average access time.
 
---
 
### 5. Key Terms — Must Know
 
| Term | Meaning |
|---|---|
| **Hit** | Required data is found in the current level (e.g., cache) |
| **Miss** | Required data is NOT found; must go to the next lower level |
| **Hit Rate (h)** | Fraction of accesses that result in a hit |
| **Miss Rate** | 1 - Hit Rate |
| **Access Time** | Time to read/write data from a memory level |
