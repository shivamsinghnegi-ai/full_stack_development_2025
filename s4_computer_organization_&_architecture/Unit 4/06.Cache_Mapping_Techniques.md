# Cache Mapping Techniques
 
When data from main memory is brought into cache, it needs to be placed somewhere. There are **3 methods** to decide **where a block goes in cache.**
 
**Setup for all examples:**
- Main memory has **16 blocks** (numbered 0–15)
- Cache has **4 blocks** (numbered 0–3)
 
---
 
### 1. Direct Mapping
 
**Rule:** Each main memory block maps to **exactly one** specific cache block.
 
**Formula:**
```
Cache block = (Main memory block number) mod (Number of cache blocks)
```
 
**Example (4 cache blocks):**
 
| Main Memory Block | Cache Block (mod 4) |
|---|---|
| 0 | 0 |
| 1 | 1 |
| 2 | 2 |
| 3 | 3 |
| 4 | 0 ← same as block 0 |
| 5 | 1 |
| 6 | 2 |
| 7 | 3 |
| 8 | 0 ← same as block 0 and 4 |
 
**Problem — Conflict Miss:**
If CPU alternates between blocks 0 and 4 (both map to cache block 0), they keep replacing each other → **thrashing.**
 
| Advantage | Disadvantage |
|---|---|
| Simple and fast | Conflict misses (thrashing) |
| Cheap to implement | Low flexibility |
 
---
 
### 2. Fully Associative Mapping
 
**Rule:** A main memory block can be placed in **any cache block** — no restriction.
 
- A tag is stored with each cache block to identify which memory block it holds.
- On lookup: CPU checks **all cache blocks simultaneously** (parallel search).
 
**Example:**
Block 5 from main memory can go into cache block 0, 1, 2, or 3 — whichever is free.
 
**Replacement Policy** (when cache is full): LRU (Least Recently Used) is most common.
 
| Advantage | Disadvantage |
|---|---|
| No conflict misses | Expensive — complex hardware |
| Maximum flexibility | Parallel tag search needed |
 
---
 
### 3. Set-Associative Mapping  (Most Common in Practice)
 
**Rule:** Cache is divided into **sets**, each containing **k blocks** (called k-way associative). A main memory block maps to a **specific set** but can go into **any block within that set.**
 
**Formula:**
```
Set number = (Main memory block number) mod (Number of sets)
```
 
**Example — 2-way Set Associative (4 cache blocks → 2 sets of 2 blocks each):**
 
- Set 0 contains cache blocks 0, 1
- Set 1 contains cache blocks 2, 3
 
| Main Memory Block | Set (mod 2) | Can go into |
|---|---|---|
| 0 | Set 0 | Cache block 0 or 1 |
| 1 | Set 1 | Cache block 2 or 3 |
| 2 | Set 0 | Cache block 0 or 1 |
| 4 | Set 0 | Cache block 0 or 1 |
 
**Comparison of all three:**
 
| Feature | Direct Mapped | Fully Associative | Set Associative |
|---|---|---|---|
| Placement | Fixed (one slot) | Anywhere | Fixed set, any slot in set |
| Conflict misses | High | None | Low |
| Hardware cost | Low | High | Medium |
| Search | Simple | All blocks in parallel | Set blocks in parallel |
| Used in | Simple systems | TLB, small caches | **Most CPUs (L1, L2, L3)** |
 
> **Most modern CPUs use 4-way or 8-way set-associative cache.**
