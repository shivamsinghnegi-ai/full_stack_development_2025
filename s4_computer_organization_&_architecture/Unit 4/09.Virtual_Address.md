# Virtual Memory
 
### 1. What is Virtual Memory?
 
Virtual memory is a **memory management technique** that gives each process the **illusion of having a large, private address space** — even if the actual physical RAM is smaller.
 
It allows a program **larger than physical RAM** to execute by keeping only the **active parts in RAM** and storing the rest on the **hard disk (secondary storage).**
 
The portion of the disk used for this purpose is called:
- **Swap space** (Linux/Unix)
- **Page file / Virtual memory file** (Windows)
 
---
 
### 2. Why is Virtual Memory Needed?
 
| Problem | Solution via Virtual Memory |
|---|---|
| Program is larger than RAM | Only active pages kept in RAM, rest on disk |
| Multiple programs need memory simultaneously | Each gets its own virtual address space |
| Programs must be isolated from each other | Virtual addresses are private per process |
| Physical memory gets fragmented | Paging handles fragmentation automatically |
 
**Example:**
- RAM = 4 GB
- Program requires 6 GB
- Without virtual memory → program cannot run
- With virtual memory → OS loads needed parts into RAM, keeps rest on disk → program runs fine
 
---
 
### 3. Advantages of Virtual Memory
 
1. Programs larger than physical RAM can run
2. More programs can run simultaneously (multiprogramming)
3. Memory isolation between processes (security)
4. Programmer does not need to worry about physical memory size
5. Efficient use of RAM — only active parts loaded
 
---
 
### 4. Disadvantage of Virtual Memory
 
- **Slower** — accessing data from disk (page fault) is much slower than RAM
- If too many page faults occur → system becomes very slow (**thrashing**)

## Logical Address vs Physical Address 
 
This is one of the most fundamental concepts in virtual memory.
 
### 1. Logical Address (Virtual Address)
 
- The address **generated by the CPU** during program execution
- Refers to a location in the **virtual address space** of the process
- The program only knows logical addresses
- Also called **virtual address**
 
### 2. Physical Address
 
- The **actual address in physical RAM** (DRAM)
- The address that is actually sent to the memory hardware
- Programs never directly deal with physical addresses
 
### 3. Key Difference
 
| Feature | Logical Address | Physical Address |
|---|---|---|
| Generated by | CPU | Memory Management Unit (MMU) |
| Visible to | User program | Memory hardware |
| Space | Virtual address space | Physical address space |
| Size | Can be larger than RAM | Limited to actual RAM size |
 
---
 
### 4. MMU — Memory Management Unit
 
The **MMU** is hardware (inside or near the CPU) responsible for **translating logical addresses to physical addresses.**
 
```
CPU generates           MMU translates          RAM accessed
Logical Address  ──→   (using page table)  ──→  Physical Address
```
 
The user program always works with logical addresses. The MMU does the translation transparently.
 
---
 
## Address Binding
 
Address binding refers to the process of **mapping logical addresses to physical addresses.** It can happen at three stages:
 
| Stage | When | Details |
|---|---|---|
| **Compile Time** | During compilation | Physical address fixed at compile time — program must always load at same address. Inflexible. |
| **Load Time** | When program is loaded into RAM | Relocatable code — addresses fixed when loaded |
| **Execution Time** | During program execution | Addresses can change during execution — requires MMU hardware support. **Used in modern OSes.** |
 
> **Exam note:** Modern operating systems use **execution time binding** with MMU support — this enables virtual memory.
