# Read Only Memory (ROM)
 
### 1. What is ROM?
 
ROM is **non-volatile** memory — data is **permanently stored** and is **retained even when power is off.**
 
- In basic ROM, data is written **once at manufacturing time** and can only be **read** during normal operation.
- Used to store **firmware** — permanent software that controls hardware at the lowest level.
- Examples: BIOS/UEFI chip in a motherboard, embedded controller code, microcontroller programs.
 
### 2. ROM vs RAM — Key Difference
 
| Feature | ROM | RAM |
|---|---|---|
| Volatile? | **Non-volatile** (data safe without power) | **Volatile** (data lost without power) |
| Operation | Primarily **Read only** | Read + Write |
| Data written by | Manufacturer / programmer | CPU during normal operation |
| Used for | Firmware, boot code | Running programs, current data |
| Speed | Slower than RAM | Faster than ROM (in general) |

## Types of ROM
 
### 1. MROM — Mask ROM
 
- **Full name:** Mask Programmed ROM
- Data is **hardwired into the chip at fabrication** (during manufacturing) using a photolithographic mask.
- **Cannot be reprogrammed** — ever.
- Cheapest for **very large volumes** (e.g., millions of units).
 
| Property | Value |
|---|---|
| Programmed by | Manufacturer only |
| Reprogrammable? | No |
| Cost | Cheapest (high volume) |
| Use case | Mass-produced consumer electronics |
 
---
 
### 2. PROM — Programmable ROM
 
- **Full name:** Programmable Read Only Memory
- Comes blank from factory. The **user can program it once** using a special device called a **PROM burner / programmer.**
- Programming works by **blowing tiny fuses** inside the chip — once a fuse is blown, it cannot be restored.
- **OTP — One Time Programmable.** Once written, it is permanent.
 
| Property | Value |
|---|---|
| Programmed by | User (once) |
| Reprogrammable? | **No — One Time Programmable (OTP)** |
| How it works | Blowing internal fuses |
| Use case | Prototyping, small batch production |
 
> **Trick to remember:** PROM = **P**rogrammable once (then permanent like MROM).
 
---
 
### 3. EPROM — Erasable Programmable ROM
 
- **Full name:** Erasable Programmable Read Only Memory
- Can be **programmed by the user** and can also be **erased and reprogrammed.**
- Erasure is done by exposing the chip to **ultraviolet (UV) light** for ~20–30 minutes through a **quartz window** on top of the chip.
- After UV erasure, the chip can be reprogrammed using a PROM programmer.
 
| Property | Value |
|---|---|
| Programmed by | User (multiple times) |
| Erasure method | **UV light** (ultraviolet) |
| Window on chip | Yes — quartz glass window |
| Time to erase | ~20–30 minutes |
| Reprogrammable? | Yes — but must remove from circuit |
| Use case | Development, prototyping |
 
> **Trick to remember:** EPROM = **E**rasable with UV light → needs **E**xposure to light.
 
---
 
### 4. EEPROM — Electrically Erasable Programmable ROM
 
- **Full name:** Electrically Erasable Programmable Read Only Memory
- Can be **erased and reprogrammed electrically** — no UV light, no need to remove from circuit.
- Erasure and programming happen **byte by byte** (selective erase).
- Much more convenient than EPROM.
- **Slower and more expensive** than Flash memory.
 
| Property | Value |
|---|---|
| Programmed by | User (electrically) |
| Erasure method | **Electrical signals** |
| Removal from circuit? | **Not needed** — in-circuit programming |
| Erase granularity | **Byte by byte** |
| Speed | Slower write/erase than Flash |
| Use case | BIOS chips, smart cards, configuration storage |
 
> **Trick to remember:** EEPROM = **E**lectrically **E**rasable → Two E's → Electrical method, more flexible.
 
---
 
### 5. Flash Memory
 
- An advanced form of EEPROM — also **electrically erasable**, but erases data in **blocks (sectors)** instead of byte by byte.
- Much **faster** than EEPROM for bulk erase operations.
- Very **high density** → used in USB drives, SSDs, SD cards, smartphones.
- **Most common non-volatile storage** in modern embedded systems.
 
| Property | Value |
|---|---|
| Programmed by | User (electrically) |
| Erasure method | Electrical signals |
| Erase granularity | **Block/Sector (not byte)** |
| Speed | **Faster** than EEPROM |
| Density | Very high |
| Use case | USB drives, SSDs, memory cards, phones |
 
> **Flash vs EEPROM:** Both are electrically erasable — but Flash erases in **blocks** (faster, denser), EEPROM erases **byte by byte** (slower, more flexible).
 
---
 
### 6. ROM Types — Comparison Table
 
| Type | Programmed by | Erasable? | Erase Method | Reprogrammable? | Use Case |
|---|---|---|---|---|---|
| **MROM** | Manufacturer | No | — | No | Mass production |
| **PROM** | User (once) | No | — | No (OTP) | Small batch |
| **EPROM** | User | Yes | **UV Light** | Yes (remove chip) | Development |
| **EEPROM** | User | Yes | **Electrical** | Yes (in-circuit, byte-by-byte) | BIOS, smart cards |
| **Flash** | User | Yes | **Electrical** | Yes (in-circuit, block-wise) | USB, SSD, phones |
