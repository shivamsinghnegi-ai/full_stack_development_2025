# Paging 
 
### 1. What is Paging?
 
Paging is a **memory management scheme** that:
- Divides **logical (virtual) memory** into fixed-size blocks called **PAGES**
- Divides **physical memory (RAM)** into fixed-size blocks called **FRAMES**
- Page size = Frame size (always equal)
- Maps each page to a frame in physical memory
 
```
Logical Memory (Virtual)          Physical Memory (RAM)
┌──────────────────┐              ┌──────────────────┐
│    Page 0        │──────────→   │    Frame 3       │
├──────────────────┤              ├──────────────────┤
│    Page 1        │──────────→   │    Frame 7       │
├──────────────────┤              ├──────────────────┤
│    Page 2        │──────────→   │    Frame 1       │
├──────────────────┤              ├──────────────────┤
│    Page 3        │──────────→   │    Frame 5       │
└──────────────────┘              └──────────────────┘
```
 
Pages do NOT need to be in **contiguous** frames — they can be scattered anywhere in RAM. This eliminates **external fragmentation.**
 
---
 
### 2. Key Terms
 
| Term | Meaning |
|---|---|
| **Page** | Fixed-size block of logical (virtual) memory |
| **Frame** | Fixed-size block of physical memory (RAM) |
| **Page Size** | Size of each page = size of each frame |
| **Page Table** | Data structure that maps each page to its frame |
| **Page Number (p)** | Which page the logical address refers to |
| **Page Offset (d)** | Position within the page |
| **Frame Number** | Which frame in RAM the page is stored in |
 
---
 
### 3. Page Table
 
The **page table** is a per-process data structure maintained by the OS.
 
It stores the **mapping: Page Number → Frame Number**
 
```
Page Table (for Process P)
┌──────────────┬──────────────┐
│  Page Number │ Frame Number │
├──────────────┼──────────────┤
│      0       │      3       │
│      1       │      7       │
│      2       │      1       │
│      3       │      5       │
└──────────────┴──────────────┘
```
 
Each entry in the page table is called a **Page Table Entry (PTE).**
 
A PTE typically contains:
- **Frame number** — where the page is in RAM
- **Valid/Present bit** — is the page currently in RAM? (1 = yes, 0 = no — triggers page fault)
- **Dirty bit** — has the page been modified?
- **Protection bits** — read/write/execute permissions
- **Reference bit** — has the page been accessed recently? (used by LRU replacement)
 
---
 
### 4. Logical Address Structure 
 
A logical address is divided into two parts:
 
```
Logical Address = [ Page Number (p) | Page Offset (d) ]
```
 
| Part | Bits | Meaning |
|---|---|---|
| **Page Number (p)** | Upper bits | Which page |
| **Page Offset (d)** | Lower bits | Position within the page |
 
**How many bits for each?**
 
```
Page Offset bits = log₂(Page Size)
Page Number bits = Total address bits − Offset bits
Number of Pages  = 2^(Page Number bits)
Number of Frames = Physical Memory Size / Frame Size
```
 
---
 
### 5. Address Translation — Step by Step
 
**Steps:**
 
1. CPU generates a **logical address**
2. MMU splits it into **page number (p)** and **offset (d)**
3. MMU looks up **page table** using page number → gets **frame number (f)**
4. Physical address = **[Frame Number | Offset]**
5. RAM is accessed at that physical address
 
```
Logical Address:  [  p  |  d  ]
                      ↓
               Page Table Lookup
               p → f (frame number)
                      ↓
Physical Address: [  f  |  d  ]
                      ↓
              Access RAM at physical address
```
 
**Key point:** The **offset d is unchanged** — only the page number is replaced by the frame number.

## Address Translation — Numericals
 
### Numerical 1 — Basic Address Translation
 
**Given:**
- Logical address space = 16 bits (so logical address is 16 bits wide)
- Page size = 1 KB = 1024 bytes
- Page table:
 
| Page | Frame |
|---|---|
| 0 | 5 |
| 1 | 3 |
| 2 | 9 |
| 3 | 7 |
 
**Translate logical address: 2500 (decimal)**
 
**Step 1 — Find offset bits:**
```
Page size = 1 KB = 1024 = 2¹⁰
Offset bits = 10
```
 
**Step 2 — Find page number and offset:**
```
Page number = 2500 ÷ 1024 = 2  (integer division)
Offset      = 2500 mod 1024 = 452
```
 
**Step 3 — Look up page table:**
```
Page 2 → Frame 9
```
 
**Step 4 — Physical address:**
```
Physical address = (Frame × Page size) + Offset
                 = (9 × 1024) + 452
                 = 9216 + 452
                 = 9668
```
 
**Answer: Physical address = 9668**
 
---
 
### Numerical 2 — Binary Method
 
**Given:**
- Logical address = 0001 0100 1010 1100 (binary, 16 bits)
- Page size = 256 bytes = 2⁸
- Page table:
 
| Page | Frame |
|---|---|
| 0 | 2 |
| 1 | 7 |
| 2 | 0 |
| 3 | 4 |
 
**Step 1 — Find offset bits:**
```
Page size = 256 = 2⁸ → Offset = 8 bits
Page number bits = 16 − 8 = 8 bits
```
 
**Step 2 — Split logical address:**
```
Logical address = 0001 0100 | 1010 1100
                  Page No.  |   Offset
Page number = 0001 0100 (binary) = 20 (decimal)
Offset      = 1010 1100 (binary) = 172 (decimal)
```
 
*(Note: If page 20 is not in page table — this would be a page fault. Assume it maps to Frame 4 for this example.)*
 
**Step 3 — Physical address (binary):**
```
Frame 4 = 0000 0100 (binary)
Physical = 0000 0100 | 1010 1100
         = Frame bits + Offset bits
```
 
---
 
### Numerical 3 — Finding number of pages and frames
 
**Given:**
- Logical address space = 32 bits
- Physical memory = 512 MB
- Page size = 4 KB
 
**Find:** Number of pages, number of frames, bits for page number, bits for offset
 
**Step 1 — Offset bits:**
```
Page size = 4 KB = 4 × 1024 = 4096 = 2¹²
Offset bits = 12
```
 
**Step 2 — Page number bits:**
```
Page number bits = 32 − 12 = 20 bits
```
 
**Step 3 — Number of pages:**
```
Number of pages = 2²⁰ = 1,048,576 pages ≈ 1M pages
```
 
**Step 4 — Number of frames:**
```
Physical memory = 512 MB = 512 × 2²⁰ bytes = 2²⁹ bytes
Number of frames = Physical memory / Page size = 2²⁹ / 2¹² = 2¹⁷ = 131,072 frames
```
 
**Step 5 — Physical address bits:**
```
Physical address bits = log₂(Number of frames) + Offset bits = 17 + 12 = 29 bits
```
 
---
 
### Numerical 4 — Page Table Size
 
**Given:**
- Logical address space = 32 bits
- Page size = 4 KB
- Each page table entry = 4 bytes
 
**Find:** Size of page table
 
**Step 1:**
```
Offset bits = log₂(4 KB) = log₂(2¹²) = 12 bits
Page number bits = 32 − 12 = 20 bits
Number of pages = 2²⁰ entries
```
 
**Step 2:**
```
Page table size = Number of entries × Size of each entry
               = 2²⁰ × 4 bytes
               = 4 × 2²⁰ bytes
               = 4 MB
```
 
**Answer: Page table = 4 MB**
