## SSD — Solid State Drive
 
SSD stores data using **Flash memory** (NAND Flash) — no moving mechanical parts.
 
### HDD vs SSD Comparison 
 
| Feature | HDD | SSD |
|---|---|---|
| Technology | Magnetic platters | Flash (NAND) memory |
| Moving parts | Yes (arm, platters) | **No** |
| Access time | 5–15 ms | **0.05–0.15 ms** |
| Read/Write speed | 80–160 MB/s | **500–7000 MB/s** |
| Capacity | Up to 20 TB | Up to 8 TB (common) |
| Cost per GB | **Cheaper** | More expensive |
| Noise | Yes (mechanical) | **Silent** |
| Durability | Fragile (moving parts) | **More durable** |
| Power consumption | More | **Less** |
| Seek time | Yes (mechanical) | **No seek time** |

## RAID 
 
### 1. What is RAID?
 
**RAID = Redundant Array of Independent Disks**
 
RAID is a technology that combines **multiple physical disks** into one logical unit to achieve:
- **Better performance** (faster read/write)
- **Data redundancy** (protection against disk failure)
- **Or both**
 
---
 
### 2. Why is RAID Needed?
 
| Problem | RAID Solution |
|---|---|
| Single disk failure → all data lost | Redundancy — data stored on multiple disks |
| Single disk too slow for high-demand applications | Striping — parallel read/write across multiple disks |
| Single disk too small | Multiple disks appear as one large disk |
 
**Real world use:** Servers, data centers, NAS (Network Attached Storage), enterprise systems.
 
---
 
### 3. Two Key Concepts Before RAID Levels
 
#### Striping
 
> Dividing data into chunks and **writing them across multiple disks simultaneously.**
 
```
Data: [A1][A2][A3][A4]
 
With 2 disks:
Disk 1: [A1][A3]
Disk 2: [A2][A4]
```
 
**Benefit:** Read/write happens in **parallel** → faster. Two disks working together = 2x speed.
 
**Risk:** If one disk fails → all data lost (no redundancy).
 
---
 
#### Mirroring
 
> **Identical copies** of data are written on two or more disks simultaneously.
 
```
Data: [A1][A2]
 
Disk 1: [A1][A2]  ← Original
Disk 2: [A1][A2]  ← Exact copy
```
 
**Benefit:** If one disk fails → data safe on the other disk.
 
**Cost:** Need double the storage (50% efficiency).
 
---
 
#### Parity
 
> A calculated **error-checking value** stored alongside data. If one disk fails, the parity can be used to **reconstruct** the lost data.
 
```
Data:   [A1][A2][A3]
Parity: [A1 XOR A2 XOR A3]
 
If A2 fails:
A2 = Parity XOR A1 XOR A3   ← Reconstructed!
```
 
**Benefit:** Fault tolerance with **less storage overhead** than full mirroring.
 
---
 
### 4. RAID Levels 
 
---
 
#### RAID 0 — Striping (No Redundancy)
 
**Concept:** Data is **striped** across all disks — no redundancy.
 
```
Data: [A1][A2][A3][A4]
 
Disk 0: [A1][A3]
Disk 1: [A2][A4]
```
 
**Minimum disks:** 2
 
| Feature | Value |
|---|---|
| Redundancy | **None** |
| Performance | **Best** — full parallel read/write |
| Storage efficiency | **100%** — all space usable |
| Fault tolerance | **None** — 1 disk fails = all data lost |
| Use case | Temporary data, video editing, gaming (speed priority) |
 
> **Risk:** RAID 0 is the fastest but the most dangerous. No disk can fail.
 
---
 
#### RAID 1 — Mirroring (Full Redundancy)
 
**Concept:** Every disk has an **exact mirror copy.**
 
```
Disk 0: [A1][A2][A3]  ← Original
Disk 1: [A1][A2][A3]  ← Exact mirror
```
 
**Minimum disks:** 2
 
| Feature | Value |
|---|---|
| Redundancy | **Full — 1 disk can fail safely** |
| Read performance | Good (can read from either disk) |
| Write performance | Slower (must write to both disks) |
| Storage efficiency | **50%** — half the space is used for mirror |
| Fault tolerance | **1 disk failure tolerated** |
| Use case | OS drives, critical databases, boot drives |
 
---
 
#### RAID 5 — Striping with Distributed Parity  
**Concept:** Data is **striped** across disks. **Parity is distributed** (not on one dedicated disk) across all disks.
 
```
4 disks, RAID 5 layout:
 
Disk 0: [A1][B1][C1][Dp]
Disk 1: [A2][B2][Cp][D1]
Disk 2: [A3][Bp][C2][D2]
Disk 3: [Ap][B3][C3][D3]
 
Ap = Parity of A1, A2, A3
Bp = Parity of B1, B2, B3 ... etc.
Parity rotates across disks — no single bottleneck
```
 
**Minimum disks:** 3
 
| Feature | Value |
|---|---|
| Redundancy | **Yes — 1 disk failure tolerated** |
| Read performance | **Excellent** (striped reads) |
| Write performance | Moderate (parity must be calculated on writes) |
| Storage efficiency | **(N−1)/N** — e.g., 4 disks → 75% efficiency |
| Fault tolerance | **1 disk failure** |
| Use case | **Most popular for servers, NAS, file servers** |
 
**Storage calculation:**
```
Usable capacity = (Number of disks − 1) × Disk size
Example: 4 disks × 2 TB each → Usable = 3 × 2 TB = 6 TB
```
 
---
 
#### RAID 6 — Striping with Double Distributed Parity
 
**Concept:** Same as RAID 5 but **two parity blocks** per stripe — can tolerate **2 disk failures.**
 
```
Disk 0: [A1][Ap][Aq]
Disk 1: [A2][Bp][Bq]
Disk 2: [Ap][B1][Cq]
Disk 3: [Aq][B2][C1]
 
Two parity values (P and Q) per stripe
```
 
**Minimum disks:** 4
 
| Feature | Value |
|---|---|
| Redundancy | **Yes — 2 disk failures tolerated** |
| Read performance | Good |
| Write performance | Slower than RAID 5 (two parities to compute) |
| Storage efficiency | **(N−2)/N** — e.g., 6 disks → 67% efficiency |
| Fault tolerance | **2 disk failures** |
| Use case | Critical data storage where extra safety needed |
 
**Storage calculation:**
```
Usable capacity = (Number of disks − 2) × Disk size
Example: 6 disks × 2 TB → Usable = 4 × 2 TB = 8 TB
```
 
---
 
#### RAID 10 (RAID 1+0) — Mirror + Stripe 
 
**Concept:** **Combine RAID 1 (mirroring) and RAID 0 (striping).** First mirror pairs of disks, then stripe across mirror pairs.
 
```
4 disks:
 
Mirror Pair 1:          Mirror Pair 2:
Disk 0: [A1][A2]        Disk 2: [A3][A4]
Disk 1: [A1][A2]        Disk 3: [A3][A4]
(mirror of Disk 0)      (mirror of Disk 2)
 
Then stripe across pairs:
Pair 1 handles: [A1][A2]
Pair 2 handles: [A3][A4]
```
 
**Minimum disks:** 4 (always even number)
 
| Feature | Value |
|---|---|
| Redundancy | **Yes — 1 disk per mirror pair can fail** |
| Read performance | **Excellent** (striped) |
| Write performance | **Good** (mirrored + striped) |
| Storage efficiency | **50%** (due to mirroring) |
| Fault tolerance | **Up to 2 disk failures** (if not from same pair) |
| Use case | **High performance + reliability — databases, email servers** |
 
---
 
### 5. RAID Levels — Master Comparison Table 
 
| RAID Level | Technique | Min Disks | Fault Tolerance | Storage Efficiency | Read Speed | Write Speed | Use Case |
|---|---|---|---|---|---|---|---|
| **RAID 0** | Striping | 2 | **None** | **100%** | Best | Best | Speed only |
| **RAID 1** | Mirroring | 2 | 1 disk | **50%** | Good | Slow | OS, boot drives |
| **RAID 5** | Stripe + Dist. Parity | 3 | 1 disk | **(N−1)/N** | Excellent | Moderate | **Servers, NAS** |
| **RAID 6** | Stripe + Double Parity | 4 | **2 disks** | **(N−2)/N** | Good | Slow | Critical data |
| **RAID 10** | Mirror + Stripe | 4 | 1 per pair | **50%** | Excellent | Good | **DB, high perf** |

### 6. RAID Storage Capacity Numericals 
 
#### Numerical 1
 
**Given:** RAID 5 with 5 disks, each 4 TB. Find usable capacity.
 
```
Usable = (N−1) × Disk size = (5−1) × 4 = 4 × 4 = 16 TB
```
 
---
 
#### Numerical 2
 
**Given:** RAID 6 with 8 disks, each 3 TB. Find usable capacity.
 
```
Usable = (N−2) × Disk size = (8−2) × 3 = 6 × 3 = 18 TB
```
 
---
 
#### Numerical 3
 
**Given:** RAID 1 with 6 disks, each 2 TB. Find usable capacity.
 
```
RAID 1 = 50% efficiency
Usable = Total / 2 = (6 × 2) / 2 = 12 / 2 = 6 TB
```
 
---
 
#### Numerical 4
 
**Given:** RAID 10 with 8 disks, each 1 TB. Find usable capacity.
 
```
RAID 10 = 50% efficiency
Usable = Total / 2 = (8 × 1) / 2 = 4 TB
```
