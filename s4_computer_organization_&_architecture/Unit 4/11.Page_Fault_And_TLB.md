# Page Fault 
 
### What is a Page Fault?
 
A **page fault** occurs when the CPU tries to access a page that is **not currently in RAM** (it is on the disk).
 
The **valid/present bit** in the PTE is **0** → triggers a page fault interrupt.
 
### Steps when a Page Fault Occurs:
 
```
1. CPU accesses logical address
2. MMU checks page table → Valid bit = 0 (page not in RAM)
3. MMU raises a PAGE FAULT interrupt
4. CPU transfers control to OS (Page Fault Handler)
5. OS finds the page on disk (in swap space)
6. OS finds a free frame in RAM
   → If no free frame: select a victim frame (using replacement policy: LRU/FIFO)
   → If victim frame is dirty: write it back to disk first
7. OS loads the required page from disk into the free frame
8. OS updates the page table entry → valid bit = 1, frame number updated
9. Control returns to the process — the instruction is restarted
10. CPU accesses the logical address again → this time it succeeds
```
 
### Page Fault Effective Access Time (EAT)
 
```
EAT = (1 − p) × memory access time + p × page fault time
```
 
Where:
- **p** = page fault rate (probability of page fault)
- **memory access time** = normal RAM access time
- **page fault time** = time to handle a page fault (disk access ~10 ms)
 
**Example:**
- Memory access time = 200 ns
- Page fault time = 10 ms = 10,000,000 ns
- Page fault rate p = 0.001
 
```
EAT = (1 − 0.001) × 200 + 0.001 × 10,000,000
    = 0.999 × 200 + 10,000
    = 199.8 + 10,000
    = 10,199.8 ns ≈ 10.2 μs
```
 
This shows why keeping page fault rate **very low** is critical for performance.

### Thrashing
 
**Thrashing** occurs when the OS spends more time **handling page faults** than executing actual processes.
 
**Cause:** Too many processes in memory, each with insufficient pages → constant page faults.
 
**Effect:** CPU utilization drops drastically, system appears frozen.
 
**Prevention:**
- Working Set Model — give each process enough frames for its working set
- Reduce degree of multiprogramming

## TLB — Translation Lookaside Buffer
 
### The Problem with Page Table Lookup
 
Every memory access requires **two RAM accesses:**
1. Access page table in RAM → get frame number
2. Access actual data in RAM using physical address
 
This **doubles the memory access time** — defeating the purpose of fast memory.
 
### Solution — TLB
 
The **TLB (Translation Lookaside Buffer)** is a **small, very fast cache** (SRAM-based) inside the MMU that stores **recently used page table entries.**
 
```
CPU generates logical address
          ↓
    Check TLB first
    ↙            ↘
TLB Hit          TLB Miss
(page entry       (entry not in TLB)
 found in TLB)         ↓
    ↓            Access page table in RAM
Get frame no.     Get frame no.
directly          Update TLB with this entry
    ↓                  ↓
Access RAM        Access RAM
```
 
### TLB Hit vs TLB Miss
 
| | TLB Hit | TLB Miss |
|---|---|---|
| Page table accessed? | No | Yes |
| Accesses needed | 1 (data only) | 2 (page table + data) |
| Speed | Fast | Slower |
 
---
 
### Effective Memory Access Time with TLB 
 
```
EAT = (TLB Hit Rate × (TLB time + Memory time))
    + (TLB Miss Rate × (TLB time + 2 × Memory time))
```
 
Or simplified (TLB access time often included in memory time):
 
```
EAT = h × (t_tlb + t_m) + (1 − h) × (t_tlb + 2 × t_m)
```
 
Where:
- **h** = TLB hit rate
- **t_tlb** = TLB access time
- **t_m** = memory access time (one RAM access)
 
---
 
### TLB Numerical 1
 
**Given:**
- TLB access time = 10 ns
- Memory access time = 80 ns
- TLB hit rate = 0.90
 
**Solution:**
```
EAT = 0.90 × (10 + 80) + 0.10 × (10 + 80 + 80)
    = 0.90 × 90 + 0.10 × 170
    = 81 + 17
    = 98 ns
```
 
Without TLB: 2 × 80 = 160 ns. With TLB: 98 ns. **~1.6x faster!**
 
---
 
### TLB Numerical 2
 
**Given:**
- TLB hit rate = 0.95
- TLB time = 20 ns
- Memory time = 100 ns
 
**Solution:**
```
EAT = 0.95 × (20 + 100) + 0.05 × (20 + 100 + 100)
    = 0.95 × 120 + 0.05 × 220
    = 114 + 11
    = 125 ns
```

## Important Formulas — Summary 
 
```
Page Offset bits       = log₂(Page Size)
Page Number bits       = Total logical address bits − Offset bits
Number of Pages        = 2^(Page Number bits) = Logical address space / Page size
Number of Frames       = Physical memory size / Frame size
Physical address bits  = log₂(Number of frames) + Offset bits
Page table size        = Number of pages × Size of one PTE
 
Page Number            = Logical address ÷ Page size  (integer division)
Page Offset            = Logical address mod Page size
Physical address       = (Frame number × Page size) + Offset
 
TLB EAT = h × (t_tlb + t_m) + (1−h) × (t_tlb + 2 × t_m)
Page Fault EAT = (1−p) × t_m + p × t_page_fault
```

## Numerical Practice Set
 
**Q1.** Page size = 2 KB, Logical address = 5200. Frame for page 2 = 8.
Find physical address.
> Page no = 5200 ÷ 2048 = 2, Offset = 5200 mod 2048 = 1104
> Physical = (8 × 2048) + 1104 = 16384 + 1104 = **17488**
 
---
 
**Q2.** Logical address space = 16 bits, Page size = 512 bytes.
Find: offset bits, page number bits, number of pages.
> Offset = log₂(512) = 9 bits
> Page number bits = 16 − 9 = 7 bits
> Number of pages = 2⁷ = **128 pages**
 
---
 
**Q3.** Physical memory = 64 MB, Page size = 4 KB.
Find number of frames.
> 64 MB = 64 × 2²⁰ = 2²⁶ bytes
> 4 KB = 2¹² bytes
> Frames = 2²⁶ / 2¹² = **2¹⁴ = 16,384 frames**

> = 108 + 22 = **130 ns**
