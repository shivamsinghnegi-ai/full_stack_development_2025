# **25: Virtualization + Load Balancing**

*(How cloud systems scale and stay available)*

---

## 1. Why We Need to Understand the Load Balancing

Earlier, we understood:

* how one machine can run multiple virtual machines (VMs)

Now the next question is:

> **How do these multiple VMs actually handle real-world traffic?**

This is where **load balancing** comes in.

Virtualization gives us **multiple machines**.
Load balancing makes those machines **work together efficiently**.

---

## 2. The Core Problem: Uneven Load

Imagine a system without load balancing:

* 5 virtual machines running the same application
* All traffic goes to only 1 VM

Result:

* one VM overloads
* others remain idle
* system becomes slow or crashes

This defeats the purpose of virtualization.

---

## 3. What Is Load Balancing?

**Load balancing** is the process of:

> **Distributing incoming requests across multiple servers (VMs) to ensure optimal utilization and performance.**

Instead of:

* one VM handling everything

We get:

* many VMs sharing the load

---

## 4. How Virtualization Enables Load Balancing

Without virtualization:

* scaling means buying new physical servers
* slow and expensive

With virtualization:

* new VMs can be created quickly
* identical application instances can run across them

This allows:

* horizontal scaling
* flexible resource allocation

So the relationship is:

> **Virtualization provides multiple instances → Load balancing distributes traffic across them**

---

## 5. Basic Architecture (Conceptual Flow)

A typical setup looks like this:

1. User sends request
2. Request reaches load balancer
3. Load balancer selects a VM
4. VM processes the request
5. Response is returned

Key point:

* Users never directly interact with individual VMs
* They interact with the **load balancer endpoint**

---

## 6. Types of Load Distribution (Simplified)

Load balancers use different strategies:

### Round Robin

* requests distributed one by one across VMs
* simple and commonly used

---

### Least Connections

* request goes to VM with least active load

---

### Weighted Distribution

* more powerful VMs receive more traffic

---

Important idea:

> The goal is always **balanced utilization**, not equal distribution.

---

## 7. Scaling with Virtualization + Load Balancing

When traffic increases:

1. System detects higher load
2. New VMs are created
3. Load balancer starts routing traffic to new VMs

When traffic decreases:

1. Extra VMs are removed
2. Load balancer adjusts routing

This creates:

> **Elastic scaling — a core cloud capability**

---

## 8. Handling Failures (Very Important)

Real systems must assume:

> **VMs will fail**

Load balancers handle this using:

* health checks
* failure detection
* automatic rerouting

If a VM:

* crashes
* becomes slow
* stops responding

Then:

* it is removed from traffic
* other VMs take over

This ensures **high availability**.

---

## 9. Why Load Balancing Is Critical in Cloud

Load balancing enables:

### 1. High Availability

No single point of failure

### 2. Scalability

Traffic can grow without breaking the system

### 3. Performance

No single machine is overloaded

### 4. Fault Tolerance

Failures are isolated and handled automatically

---

## 10. Virtualization + Load Balancing = Cloud Behavior

Let’s connect everything clearly:

| Component      | Role                  |
| -------------- | --------------------- |
| Virtualization | Creates multiple VMs  |
| Load Balancer  | Distributes traffic   |
| Autoscaling    | Adjusts number of VMs |

Together they create:

> **A system that scales automatically and survives failures**

---

## 11. Common Mistakes

### Mistake 1: Assuming one server is enough

→ fails under load

---

### Mistake 2: No health checks

→ traffic sent to broken VMs

---

### Mistake 3: State stored on VM

→ user sessions break when traffic shifts

---

### Mistake 4: Improper scaling rules

→ either over-provisioning or crashes

---

## 12. Mental Model to Remember

> **Virtualization creates multiple workers.
> Load balancing decides who gets the next task.**

---

## Quick Recap

* Virtualization enables multiple VMs
* Load balancing distributes traffic across them
* Together they enable scalability and availability
* Failures are handled automatically
* This is the foundation of cloud reliability
