# 9: Software as a Service (SaaS)

*(The cloud model most people use—often without realizing it)*

---

## 1. Why SaaS Is the Most Visible Face of Cloud

When most people think of “cloud”, they are actually thinking of **SaaS**.

Why?

Because SaaS is the **only cloud layer end users directly interact with**:

* no servers
* no deployments
* no configuration
* just sign up and use

From an engineering point of view, SaaS is also the **hardest cloud model to design correctly**.

---

## 2. What Is Software as a Service (SaaS)?

**Software as a Service (SaaS)** is a model where:

* the entire application is hosted by the provider
* users access it via browser or app
* infrastructure, platform, and software are fully managed

The user only cares about:

* features
* reliability
* performance
* data safety

![Image](https://www.guru99.com/images/3/software-as-a-service-1.png?utm_source=codinggita.com)

![Image](https://redcrackle.com/images/blogPostContent/what-is-saas/20152502003237.jpeg?utm_source=codinggita.com)

In SaaS, **everything below the UI is invisible to the customer**.

---

## 3. Responsibility Model (Zero Infrastructure for Users)

Let’s be very explicit here.

### SaaS Provider Manages

* infrastructure (servers, networking, storage)
* platform (runtime, scaling, monitoring)
* application code
* updates and patches
* availability and disaster recovery

### User Manages

* account configuration
* their own data
* usage and access permissions

![Image](https://learn.microsoft.com/en-us/azure/security/fundamentals/media/shared-responsibility/shared-responsibility.svg?utm_source=codinggita.com)

![Image](https://www.cisecurity.org/-/jssmedia/project/cisecurity/cisecurity/data/media/img/uploads/2020/06/cis-hardened-images-shared-responsibility-model-2020-0608.png?hash=3F10DB5C1C633AC5DF943822FD625271\&rev=7f124b2db69c454fa55d5844d2859a78\&utm_source=codinggita.com)

This is why SaaS feels effortless—but is extremely complex behind the scenes.

## 4. Real-World SaaS Examples (Mapped to Reality)

### OTT Platforms

When you open **Netflix**:

* you don’t choose servers
* you don’t manage bandwidth
* you don’t handle scaling

Yet Netflix:

* serves millions concurrently
* personalizes content per user
* scales differently per region

That is SaaS done at global scale.

---

### Fintech Platforms

On **Groww**:

* users place trades
* see portfolios
* track markets

Behind the scenes:

* pricing engines
* risk checks
* settlement workflows
* compliance systems

All of this is hidden behind a **simple UI**.

---

## 5. The Core Engineering Challenge of SaaS: Multi-Tenancy

This is the most important SaaS concept.

### What Is Multi-Tenancy?

A single application instance serves:

* thousands or millions of users
* with **strict isolation of data and behavior**

![Image](https://www.simform.com/wp-content/uploads/2022/06/Multi-tenant-architecture.png?utm_source=codinggita.com)

### Why It’s Hard

* one bug can affect many users
* performance isolation is critical
* data leaks are catastrophic

SaaS systems must be:

* logically isolated
* physically efficient

---

## 6. How SaaS Scales Differently from PaaS/IaaS

In SaaS:

* scaling is not just infrastructure
* scaling is **application-level design**

Examples:

* per-user rate limits
* per-tenant quotas
* sharded databases
* regional traffic routing

This is why SaaS engineering leans heavily toward **system design skills**.

---

## 7. SaaS Updates and Continuous Delivery

Unlike traditional software:

* users don’t install updates
* everyone runs the latest version

This enables:

* rapid bug fixes
* continuous improvements
* controlled rollouts

But it also means:

* zero tolerance for breaking changes
* backward compatibility matters deeply

---

## 8. Advantages of SaaS (Why Businesses Love It)

### For Users

* no setup
* predictable pricing
* instant access
* automatic updates

### For Providers

* centralized control
* usage analytics
* faster iteration
* subscription revenue models

This alignment is why SaaS dominates modern software delivery.

---

## 9. Disadvantages and Risks of SaaS

### Limited Customization

* users adapt to the product
* not the other way around

### Trust Dependency

* provider controls uptime
* provider controls data handling

### Compliance Sensitivity

* data residency
* regulatory constraints
* audit requirements

SaaS success depends heavily on **trust**.

---

## 10. How SaaS, PaaS, and IaaS Coexist in Real Systems

A SaaS company internally uses:

* **IaaS** for specialized workloads
* **PaaS** for APIs and services
* **SaaS** for delivering value to users

SaaS is the **product layer**, not the entire stack.

---

## 11. Mental Model to Remember SaaS

> SaaS = **Outcome, not infrastructure**

Users buy:

* results
* reliability
* experience

They don’t care *how* it’s built.

Engineers must care deeply.

---

## Quick Recap

* SaaS delivers complete software over the internet
* Users manage nothing except usage
* Multi-tenancy is the core challenge
* Scaling is application-driven
* SaaS systems require strong system design
