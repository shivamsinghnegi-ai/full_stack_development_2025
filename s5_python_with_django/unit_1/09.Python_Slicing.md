# Python Slicing — How Indices Actually Work Internally

---

## What Happens When You Slice

When you write `s[start:stop:step]`, Python does not immediately start picking elements. It first runs an internal adjustment process on `start`, `stop`, and `step` to make sure they are valid. This process is called clamping.

This is why Python slicing never crashes with an `IndexError` even when you give it out-of-bounds numbers. Python silently fixes the values before the loop runs.

The complete internal process has three stages:

1. Adjust the `step`
2. Adjust the `start`
3. Adjust the `stop`

The rules for stage 2 and 3 are different depending on whether `step` is positive or negative.

---

## The List Used in All Examples

```python
s = [10, 20, 30, 40, 50]
#     0   1   2   3   4    (positive index)
#    -5  -4  -3  -2  -1    (negative index)

length = 5
```

---

## Stage 1 — Adjusting the Step

This runs first, before anything else.

```
Rule 1.1 — If step is not given, it defaults to 1
    s[1:4]    →   step becomes 1

Rule 1.2 — If step is 0, Python raises a ValueError immediately
    s[1:4:0]  →   ValueError: slice step cannot be zero
               →   This is the only error slicing ever raises
```

```python
# Rule 1.1
s[1:4]       # step not given, defaults to 1  →  [20, 30, 40]
s[1:4:]      # same, step is empty  →  [20, 30, 40]
s[::]        # start, stop, step all default  →  [10, 20, 30, 40, 50]

# Rule 1.2
s[1:4:0]     # ValueError: slice step cannot be zero
```

---

## Stage 2 and 3 — Adjusting Start and Stop

The rules split into two paths depending on the sign of step.

---

## Path A — When step is Positive (step > 0)

This is the default case — moving left to right through the list.

### Default Values When Start or Stop is Not Given

```
If start is not given  →  start = 0          (beginning of list)
If stop is not given   →  stop  = length      (end of list)
```

```python
s[2:]     # stop not given, becomes length=5  →  s[2:5]  →  [30, 40, 50]
s[:3]     # start not given, becomes 0        →  s[0:3]  →  [10, 20, 30]
s[:]      # both default  →  s[0:5]  →  [10, 20, 30, 40, 50]
```

### Clamping Rules for Start (step > 0)

```
Rule A1 — If start is negative, add length to it
    start = start + length

Rule A2 — If start is still negative after Rule A1, clamp to 0
    start = 0

Rule A3 — If start is greater than length, clamp to length
    start = length
```

Working through each rule with examples:

```python
# Rule A1 — negative start, add length
# s[-2:]
# start = -2
# -2 + 5 = 3   →  start becomes 3
s[-2:]    # same as s[3:]  →  [40, 50]

# Rule A1 then Rule A2 — very negative start, still negative after adding length
# s[-100:]
# start = -100
# -100 + 5 = -95   →  still negative, apply Rule A2
# start = 0
s[-100:]  # same as s[0:]  →  [10, 20, 30, 40, 50]

# Rule A3 — start beyond the list
# s[99:]
# start = 99
# 99 > 5  →  start = 5
# nothing from index 5 onwards
s[99:]    # []
s[5:]     # []   (5 == length, nothing beyond it)
```

### Clamping Rules for Stop (step > 0)

```
Rule B1 — If stop is negative, add length to it
    stop = stop + length

Rule B2 — If stop is still negative after Rule B1, clamp to 0
    stop = 0

Rule B3 — If stop is greater than length, clamp to length
    stop = length
```

```python
# Rule B1 — negative stop, add length
# s[0:-2]
# stop = -2
# -2 + 5 = 3   →  stop becomes 3
s[0:-2]   # same as s[0:3]  →  [10, 20, 30]

# Rule B1 then Rule B2 — very negative stop
# s[0:-100]
# stop = -100
# -100 + 5 = -95  →  still negative, apply Rule B2
# stop = 0
s[0:-100]  # same as s[0:0]  →  []

# Rule B3 — stop beyond the list
# s[1:999]
# stop = 999
# 999 > 5  →  stop = 5
s[1:999]   # same as s[1:5]  →  [20, 30, 40, 50]
s[0:9999]  # same as s[0:5]  →  [10, 20, 30, 40, 50]
```

### Full Table — step > 0

| Slice | start raw | stop raw | start after clamping | stop after clamping | Result |
|---|---|---|---|---|---|
| `s[-1:]` | -1 | not given | -1+5 = 4 | 5 | `[50]` |
| `s[-3:]` | -3 | not given | -3+5 = 2 | 5 | `[30, 40, 50]` |
| `s[-100:]` | -100 | not given | -100+5=-95 → 0 | 5 | `[10,20,30,40,50]` |
| `s[:99]` | not given | 99 | 0 | 99 → 5 | `[10,20,30,40,50]` |
| `s[0:-2]` | 0 | -2 | 0 | -2+5 = 3 | `[10, 20, 30]` |
| `s[99:999]` | 99 | 999 | 99 → 5 | 999 → 5 | `[]` |
| `s[-100:99]` | -100 | 99 | -95 → 0 | 99 → 5 | `[10,20,30,40,50]` |

---

## Path B — When step is Negative (step < 0)

When step is negative, you are moving right to left — going backward through the list. The default values and clamping limits change because the starting point is now the end of the list.

### Default Values When Start or Stop is Not Given

```
If start is not given  →  start = length - 1    (last valid index)
If stop is not given   →  stop  = -(length + 1) (a sentinel meaning "before the beginning")
```

The sentinel `-(length + 1)` is an internal signal that means "go all the way past index 0". You never see or write this value yourself.

```python
s[::-1]    # start defaults to 4 (last index), stop to sentinel
           # goes from index 4 down through index 0
           # [50, 40, 30, 20, 10]

s[4::-1]   # same result explicitly
           # [50, 40, 30, 20, 10]
```

### Clamping Rules for Start (step < 0)

```
Rule C1 — If start is negative, add length to it
    start = start + length

Rule C2 — If start is still negative after Rule C1, clamp to -1
    start = -1
    (-1 here is a sentinel meaning "before index 0" — do not confuse
     with the normal -1 index. The loop will not include index 0.)

Rule C3 — If start is greater than or equal to length, clamp to length - 1
    start = length - 1
```

```python
# Rule C3 — start too large when going backward
# s[99:0:-1]
# start = 99
# 99 >= 5  →  start = length - 1 = 4
s[99:0:-1]    # same as s[4:0:-1]  →  [50, 40, 30, 20]
              # note: stop=0 means go down to but NOT include index 0

# Rule C1 — negative start when going backward
# s[-2::-1]
# start = -2
# -2 + 5 = 3  →  start becomes 3
s[-2::-1]     # same as s[3::-1]  →  [40, 30, 20, 10]
```

### Clamping Rules for Stop (step < 0)

```
Rule D1 — If stop is negative, add length to it
    stop = stop + length

Rule D2 — If stop is still negative after Rule D1, clamp to -1
    stop = -1
    (sentinel — means the loop goes all the way past index 0 and includes it)

Rule D3 — If stop is greater than or equal to length, clamp to length - 1
    stop = length - 1
```

```python
# Rule D2 — very negative stop when going backward, clamp to sentinel -1
# s[4:-100:-1]
# stop = -100
# -100 + 5 = -95  →  still negative, clamp to sentinel -1
# loop goes from index 4 all the way down, including index 0
s[4:-100:-1]   # [50, 40, 30, 20, 10]  — full reverse

# s[::-1] is equivalent — stop defaults to sentinel -1
s[::-1]        # [50, 40, 30, 20, 10]

# Rule D1 — negative stop when going backward
# s[4:-2:-1]
# stop = -2
# -2 + 5 = 3  →  stop becomes 3
# loop goes from index 4 down to but NOT including index 3
s[4:-2:-1]     # same as s[4:3:-1]  →  [50]

# Rule D3 — stop too large when going backward
# s[3:99:-1]
# stop = 99
# 99 >= 5  →  stop = length - 1 = 4
# start=3, stop=4, step=-1 means start < stop going backward → nothing
s[3:99:-1]     # []
```

### Full Table — step < 0

| Slice | start raw | stop raw | start after clamping | stop after clamping | Result |
|---|---|---|---|---|---|
| `s[::-1]` | not given | not given | 4 | sentinel | `[50,40,30,20,10]` |
| `s[3::-1]` | 3 | not given | 3 | sentinel | `[40,30,20,10]` |
| `s[-2::-1]` | -2 | not given | -2+5=3 | sentinel | `[40,30,20,10]` |
| `s[99:0:-1]` | 99 | 0 | 99→4 | 0 | `[50,40,30,20]` |
| `s[4:-100:-1]` | 4 | -100 | 4 | -95→sentinel | `[50,40,30,20,10]` |
| `s[3:99:-1]` | 3 | 99 | 3 | 99→4 | `[]` |

## Complete Rule Summary

```
STAGE 1 — STEP
  If step not given  →  step = 1
  If step == 0       →  ValueError (only error slicing raises)

STAGE 2 — START (depends on sign of step)

  When step > 0:
    If start not given  →  start = 0
    If start < 0        →  start = start + length
    If start still < 0  →  start = 0
    If start > length   →  start = length

  When step < 0:
    If start not given   →  start = length - 1
    If start < 0         →  start = start + length
    If start still < 0   →  start = -1  (sentinel — before index 0)
    If start >= length   →  start = length - 1

STAGE 3 — STOP (depends on sign of step)

  When step > 0:
    If stop not given  →  stop = length
    If stop < 0        →  stop = stop + length
    If stop still < 0  →  stop = 0
    If stop > length   →  stop = length

  When step < 0:
    If stop not given   →  stop = -(length + 1)  (sentinel — before index 0)
    If stop < 0         →  stop = stop + length
    If stop still < 0   →  stop = -1  (sentinel — before index 0)
    If stop >= length   →  stop = length - 1
```

---

## Four Things You Must Know

**Negative indices work because Python adds length to them.**
`s[-1]` → `-1 + 5 = 4` → last element. Always.

**Out-of-bounds single index crashes. Out-of-bounds slice never crashes.**
`s[99]` → `IndexError`. `s[99:200]` → `[]`. No error.

**`s[-100:]` is always safe and returns the full list.**
`-100 + 5 = -95`, still negative, clamped to `0`. Python returns everything.

**`s[0:99999]` on a 5-element list returns the full list.**
`99999 > 5`, clamped to `5`. Python returns everything up to the end.
