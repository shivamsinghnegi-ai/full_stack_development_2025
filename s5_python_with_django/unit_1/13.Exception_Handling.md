# Exception Handling
 
---
 
## What is an Exception
 
An exception is an error that occurs during the execution of a program. When Python encounters a situation it cannot handle — dividing by zero, accessing a missing file, using an undefined variable — it raises an exception. If the exception is not handled, the program stops and prints a traceback.
 
```python
# Division by zero
10 / 0
# ZeroDivisionError: division by zero
 
# Accessing a missing key
d = {"a": 1}
d["b"]
# KeyError: 'b'
 
# Wrong type
"hello" + 5
# TypeError: can only concatenate str (not "int") to str
 
# Missing variable
print(undefined_variable)
# NameError: name 'undefined_variable' is not defined
 
# Index out of range
[1, 2, 3][10]
# IndexError: list index out of range
 
# File not found
open("missing_file.txt")
# FileNotFoundError: [Errno 2] No such file or directory: 'missing_file.txt'
```
 
When Python raises an exception, it creates an exception object that contains the error type and a message. If you handle it, your program continues. If you do not, the program stops.
 
---
 
## try and except
 
The `try` block contains code that might raise an exception. The `except` block contains code that runs if the exception occurs. If no exception occurs, the `except` block is skipped entirely.
 
```python
# Without exception handling
result = 10 / 0    # ZeroDivisionError — program crashes
 
# With exception handling
try:
    result = 10 / 0
except ZeroDivisionError:
    print("Cannot divide by zero!")
    result = 0
 
print(result)    # 0 — program continues
```
 
```python
# Handling missing keys
student = {"name": "David", "grade": 90}
 
try:
    age = student["age"]
except KeyError:
    print("Key 'age' not found in dictionary")
    age = None
 
print(age)    # None
```
 
### Accessing the Exception Object
 
You can access the exception message using `as`:
 
```python
try:
    result = 10 / 0
except ZeroDivisionError as e:
    print(f"Error occurred: {e}")    # Error occurred: division by zero
    print(type(e))                    # <class 'ZeroDivisionError'>
```

## Catching Multiple Exceptions
 
You can have multiple `except` blocks to handle different exception types differently.
 
```python
def safe_divide(a, b):
    try:
        result = a / b
        return result
    except ZeroDivisionError:
        print("Error: Cannot divide by zero")
        return None
    except TypeError:
        print("Error: Both values must be numbers")
        return None
 
safe_divide(10, 2)      # 5.0
safe_divide(10, 0)      # Error: Cannot divide by zero
safe_divide(10, "5")    # Error: Both values must be numbers
```
 
You can also catch multiple exceptions in a single `except` block using a tuple:
 
```python
try:
    data = int(input("Enter a number: "))
    result = 100 / data
except (ValueError, ZeroDivisionError) as e:
    print(f"Invalid input: {e}")
```
 
### Catching All Exceptions
 
Using a bare `except` or `except Exception` catches any exception. Use this carefully — it can hide bugs you should be aware of.
 
```python
# Catching any exception — use sparingly
try:
    risky_operation()
except Exception as e:
    print(f"Something went wrong: {e}")
    print(f"Exception type: {type(e).__name__}")
 
# Bare except — catches everything including system exit — almost never use this
try:
    risky_operation()
except:
    print("Something went wrong")
```

## else and finally
 
### else Clause
 
The `else` block runs only if the `try` block completed without raising any exception. It is for code that should only execute when everything went well.
 
```python
try:
    number = int("42")
except ValueError:
    print("That is not a valid number")
else:
    # Runs only if no exception occurred
    print(f"Successfully converted: {number}")
    print(f"Square: {number ** 2}")
 
# Output:
# Successfully converted: 42
# Square: 1764
```
 
### finally Clause
 
The `finally` block always runs — whether an exception occurred or not, whether it was caught or not. It is used for cleanup operations that must happen regardless of what happened — closing a file, releasing a resource, closing a database connection.
 
```python
try:
    file = open("data.txt", "r")
    content = file.read()
    result = int(content)
except FileNotFoundError:
    print("File not found")
except ValueError:
    print("File does not contain a valid number")
finally:
    # This ALWAYS runs
    print("Cleanup: closing file if it was opened")
    # (In practice, use 'with' statement instead)
```
 
### All Four Clauses Together
 
```python
def read_and_convert(filename):
    try:
        file = open(filename, "r")
        content = file.read().strip()
        number = int(content)
    except FileNotFoundError:
        print(f"File '{filename}' not found")
        return None
    except ValueError:
        print(f"Content is not a valid integer: '{content}'")
        return None
    else:
        print(f"Success! Number is: {number}")
        return number
    finally:
        print("This always runs")
 
read_and_convert("numbers.txt")
```

## Raising Exceptions
 
You can raise exceptions yourself using the `raise` keyword. This is useful for enforcing rules in your functions — if a caller passes an invalid argument, you raise an exception to tell them.
 
```python
def set_age(age):
    if not isinstance(age, int):
        raise TypeError("Age must be an integer")
    if age < 0:
        raise ValueError("Age cannot be negative")
    if age > 150:
        raise ValueError("Age is unrealistically large")
    return age
 
set_age(25)      # 25
set_age(-5)      # ValueError: Age cannot be negative
set_age("old")   # TypeError: Age must be an integer
```
 
### Re-raising an Exception
 
Inside an `except` block, you can re-raise the same exception using `raise` without arguments. This is useful when you want to log the error but still let it propagate.
 
```python
import logging
 
def process_data(data):
    try:
        result = int(data)
        return result
    except ValueError as e:
        print(f"Logging error: {e}")   # log it
        raise                           # re-raise the same exception
```

## Custom Exceptions
 
You can create your own exception classes by inheriting from `Exception` or any of its subclasses. Custom exceptions make your code more expressive — the exception name itself communicates what went wrong.
 
```python
# Basic custom exception
class InsufficientFundsError(Exception):
    pass
 
# Custom exception with additional data
class InvalidAgeError(Exception):
    def __init__(self, age, message="Invalid age provided"):
        self.age = age
        self.message = message
        super().__init__(f"{message}: {age}")
 
# Custom exception hierarchy
class AppError(Exception):
    pass
 
class DatabaseError(AppError):
    pass
 
class NetworkError(AppError):
    pass
 
class TimeoutError(NetworkError):
    pass
 
 
# Using custom exceptions
class BankAccount:
    def __init__(self, balance):
        self.balance = balance
 
    def withdraw(self, amount):
        if amount > self.balance:
            raise InsufficientFundsError(
                f"Cannot withdraw {amount}. Current balance: {self.balance}"
            )
        self.balance -= amount
        return self.balance
 
 
account = BankAccount(1000)
 
try:
    account.withdraw(1500)
except InsufficientFundsError as e:
    print(f"Transaction failed: {e}")
 
try:
    raise InvalidAgeError(250)
except InvalidAgeError as e:
    print(e)          # Invalid age provided: 250
    print(e.age)      # 250
```

## Exception Hierarchy
 
Python's built-in exceptions are organized in a hierarchy. When you catch a parent exception, you also catch all its children.
 
```
BaseException
    SystemExit
    KeyboardInterrupt
    Exception
        ValueError
        TypeError
        NameError
        IndexError
        KeyError
        AttributeError
        ZeroDivisionError
        FileNotFoundError
        IOError
        OSError
        ImportError
        StopIteration
        RuntimeError
            RecursionError
        ArithmeticError
            ZeroDivisionError
            OverflowError
```
 
```python
try:
    result = 10 / 0
except ArithmeticError:
    print("An arithmetic error occurred")
    # catches ZeroDivisionError because it is a child of ArithmeticError
 
try:
    open("missing.txt")
except OSError:
    print("OS error occurred")
    # catches FileNotFoundError because it is a child of OSError
```
 
Always catch more specific exceptions first:
 
```python
try:
    result = 10 / 0
except ZeroDivisionError:     # specific — catches only zero division
    print("Division by zero")
except ArithmeticError:       # less specific — catches other arithmetic errors
    print("Arithmetic error")
except Exception:             # very general — catches everything else
    print("Some other error")
```
