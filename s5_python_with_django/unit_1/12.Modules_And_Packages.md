## What is a Module
 
A module is simply a Python file — any file with a `.py` extension. When a Python file contains functions, classes, and variables that you want to reuse in other programs, that file is a module.
 
Without modules, every program would need to contain all the code it uses. A large program would become a single massive file that is impossible to read or maintain. Modules solve this by letting you split code across multiple files and import what you need.
 
Python comes with hundreds of built-in modules — called the Standard Library — that give you ready-made tools for math, file handling, working with dates, generating random numbers, and much more. You can also install third-party modules (like NumPy, Pandas, Requests) and create your own.

## Importing Modules
 
### import statement
 
The `import` statement loads a module into your program. After importing, you access its contents using dot notation: `module_name.something`.
 
```python
import math
 
# Accessing functions from the module
math.sqrt(25)        # 5.0
math.pow(2, 10)      # 1024.0
math.floor(3.7)      # 3
math.ceil(3.2)       # 4
math.log(100, 10)    # 2.0
math.pi              # 3.141592653589793
math.e               # 2.718281828459045
 
import random
 
random.random()          # random float between 0.0 and 1.0
random.randint(1, 10)    # random integer from 1 to 10 inclusive
random.choice([1, 2, 3, 4, 5])    # picks one randomly
random.shuffle([1, 2, 3, 4, 5])   # shuffles list in place
random.seed(42)          # fixes randomness — same results every run
```
 
### import with alias
 
You can rename a module when importing it using `as`. This is useful when the module name is long or when there is a naming conflict.
 
```python
import math as m
import random as rnd
 
m.sqrt(16)        # 4.0
m.pi              # 3.141592...
 
rnd.randint(1, 6)    # simulating a dice roll
```
 
### from ... import
 
Instead of importing the entire module, you can import specific names from it. After this, you use the name directly without the module prefix.
 
```python
from math import sqrt, pi, floor
 
sqrt(81)     # 9.0  — no math. prefix needed
pi           # 3.14159...
floor(4.9)   # 4
 
from random import randint, choice, seed
 
randint(1, 100)
choice(["apple", "banana", "cherry"])
```
 
### from ... import * (avoid this)
 
This imports everything from a module into the current namespace. It is generally discouraged because it can silently overwrite names in your program without warning.
 
```python
from math import *    # imports everything — sqrt, pi, sin, cos, etc.
 
sqrt(9)    # works — but pollutes the namespace
pi         # works
 
# Problem — if you later define your own sqrt, the original is lost
def sqrt(x):
    return x ** 0.5   # your version overwrites the imported one silently
```
 
### Importing Multiple Things
 
```python
from math import sqrt, pi, log, floor, ceil, factorial
from os import getcwd, listdir, path
from datetime import date, datetime, timedelta
```

## The Standard Library — Important Modules
 
Python's Standard Library is a collection of modules that come with Python. You do not need to install them.
 
### math
 
```python
import math
 
math.sqrt(144)          # 12.0 — square root
math.pow(2, 8)          # 256.0 — power (returns float)
math.log(math.e)        # 1.0 — natural log
math.log(1000, 10)      # 3.0 — log base 10
math.log2(8)            # 3.0 — log base 2
math.sin(math.pi / 2)   # 1.0
math.cos(0)             # 1.0
math.factorial(6)        # 720
math.gcd(48, 18)         # 6 — greatest common divisor
math.floor(3.9)          # 3 — round down
math.ceil(3.1)           # 4 — round up
math.inf                 # infinity — useful for initializing minimum values
math.isnan(float('nan')) # True — check for not-a-number
```
 
### random
 
```python
import random
 
random.seed(42)                  # set seed for reproducibility
random.random()                  # float in [0.0, 1.0)
random.uniform(1.5, 9.5)         # float in [a, b]
random.randint(1, 10)            # integer in [a, b] — both inclusive
random.randrange(0, 100, 5)      # like range() but picks one value
 
data = [1, 2, 3, 4, 5, 6, 7, 8]
random.choice(data)              # pick one element
random.choices(data, k=3)        # pick 3 with replacement
random.sample(data, k=3)         # pick 3 without replacement
random.shuffle(data)             # shuffle in place — modifies original
```
 
### datetime
 
```python
from datetime import date, datetime, timedelta
 
# Current date and time
today = date.today()
print(today)                   # 2025-01-15
 
now = datetime.now()
print(now)                     # 2025-01-15 14:32:10.123456
print(now.year)                # 2025
print(now.month)               # 1
print(now.day)                 # 15
print(now.hour)                # 14
 
# Creating specific dates
birthday = date(2000, 6, 15)
event = datetime(2025, 12, 31, 23, 59, 59)
 
# Date arithmetic
deadline = today + timedelta(days=30)
days_since = today - birthday   # returns a timedelta object
print(days_since.days)          # total days between dates
 
# Formatting dates as strings
now.strftime("%d/%m/%Y")        # "15/01/2025"
now.strftime("%B %d, %Y")       # "January 15, 2025"
now.strftime("%Y-%m-%d %H:%M")  # "2025-01-15 14:32"
 
# Parsing a string into a datetime
datetime.strptime("2025-06-15", "%Y-%m-%d")
```

## Creating Your Own Module
 
Any `.py` file you create is a module. You import it by its filename (without the `.py` extension).
 
Create a file called `calculator.py`:
 
```python
# calculator.py
 
PI = 3.14159
 
def add(a, b):
    return a + b
 
def subtract(a, b):
    return a - b
 
def multiply(a, b):
    return a * b
 
def divide(a, b):
    if b == 0:
        raise ValueError("Cannot divide by zero")
    return a / b
 
def circle_area(radius):
    return PI * radius ** 2
```
 
Now use it in another file in the same folder:
 
```python
# main.py
 
import calculator
 
print(calculator.add(10, 5))         # 15
print(calculator.divide(20, 4))      # 5.0
print(calculator.circle_area(7))     # 153.93...
print(calculator.PI)                 # 3.14159
 
# Or import specific names
from calculator import add, multiply
print(add(3, 4))         # 7
print(multiply(3, 4))    # 12
```

## The __name__ Variable
 
Every Python file has a built-in variable called `__name__`. When a file is run directly, `__name__` is set to `"__main__"`. When a file is imported as a module, `__name__` is set to the module's filename.
 
This lets you write code that only runs when the file is executed directly, not when it is imported.
 
```python
# calculator.py
 
def add(a, b):
    return a + b
 
def multiply(a, b):
    return a * b
 
# This block only runs when you run calculator.py directly
# It does NOT run when you do: import calculator
if __name__ == "__main__":
    print("Running calculator directly")
    print(add(3, 5))       # 8
    print(multiply(4, 6))  # 24
```
 
This pattern is used in every professional Python file. It lets you test your module directly while also making it safe to import.

## What is a Package
 
A package is a folder that contains multiple Python modules (`.py` files) along with a special file called `__init__.py`. The `__init__.py` file marks the folder as a Python package and can be empty or can contain initialization code.
 
Packages let you organize related modules into a directory structure.
 
```
my_package/
    __init__.py
    calculator.py
    string_utils.py
    file_utils.py
```
 
Importing from a package:
 
```python
# Import a module from the package
import my_package.calculator
 
my_package.calculator.add(3, 4)
 
# Import specific function from a module inside a package
from my_package.calculator import add, multiply
 
add(3, 4)       # 7
multiply(2, 5)  # 10
 
# Import the module with an alias
from my_package import calculator as calc
calc.add(1, 2)
```
 
### Nested Packages
 
Packages can contain sub-packages — folders inside folders:
 
```
my_app/
    __init__.py
    utils/
        __init__.py
        math_utils.py
        string_utils.py
    data/
        __init__.py
        loader.py
        cleaner.py
```
 
```python
from my_app.utils.math_utils import normalize
from my_app.data.loader import load_csv
```

## pip — Installing Third-Party Packages
 
pip is Python's package installer. It downloads and installs packages from PyPI (Python Package Index) — a repository of hundreds of thousands of open-source Python packages.
 
```bash
# Install a package
pip install requests
pip install numpy
pip install pandas
 
# Install a specific version
pip install numpy==1.24.0
pip install requests>=2.28.0
 
# Upgrade a package
pip install --upgrade numpy
 
# Uninstall a package
pip uninstall requests
 
# List installed packages
pip list
 
# Show information about a package
pip show numpy
 
# Save all installed packages to a file
pip freeze > requirements.txt
 
# Install all packages from a requirements file
pip install -r requirements.txt
```

## Virtual Environments
 
A virtual environment is an isolated Python environment. Each project gets its own set of installed packages, independent of other projects and the system Python.
 
Without virtual environments, installing a package installs it globally — affecting every project on your machine. If Project A needs `numpy==1.20` and Project B needs `numpy==1.24`, you have a conflict. Virtual environments solve this.
 
```bash
# Create a virtual environment named 'env' in the current folder
python -m venv env
 
# Activate it — Mac/Linux
source env/bin/activate
 
# Activate it — Windows
env\Scripts\activate
 
# Your prompt changes to show (env) when activated
(env) $
 
# Now pip install only installs into this environment
pip install numpy pandas
 
# Deactivate when done
deactivate
```
