# File I/O
 
---
 
## What is File I/O
 
File I/O (Input/Output) is how your program reads data from files and writes data to files. Without file I/O, all data your program creates is lost when the program ends. File I/O allows data to persist on disk.
 
---
 
## Opening a File
 
The `open()` function opens a file and returns a file object. It takes the filename and a mode.
 
```
Mode    Meaning
r       Read (default). File must exist. Cursor at beginning.
w       Write. Creates file if not exists. Overwrites if exists.
a       Append. Creates file if not exists. Adds to end if exists.
x       Exclusive creation. Fails if file already exists.
r+      Read and write. File must exist.
b       Binary mode (used with r, w, a: rb, wb, ab)
t       Text mode (default — used with r, w, a: rt, wt, at)
```
 
```python
# Opening a file — always close it when done
file = open("example.txt", "r")
content = file.read()
file.close()    # must close to release the resource
```
 
Forgetting to close a file can cause data loss or resource leaks. The correct way is to always use the `with` statement.
 
---
 
## The with Statement
 
The `with` statement automatically closes the file when the block ends, even if an exception occurs. This is the correct and recommended way to work with files.
 
```python
with open("example.txt", "r") as file:
    content = file.read()
    print(content)
# File is automatically closed here — even if an exception occurred inside
```

## Reading Files
 
### read() — Read the Entire File
 
```python
with open("example.txt", "r") as file:
    content = file.read()     # reads entire file as one string
    print(content)
    print(type(content))      # <class 'str'>
```
 
```python
# Reading only a specific number of characters
with open("example.txt", "r") as file:
    first_10 = file.read(10)   # reads first 10 characters
    next_5   = file.read(5)    # reads next 5 characters from where it left off
```
 
### readline() — Read One Line at a Time
 
```python
with open("example.txt", "r") as file:
    line1 = file.readline()    # reads first line including the newline \n
    line2 = file.readline()    # reads second line
    line3 = file.readline()    # reads third line
 
# Reading all lines one by one
with open("example.txt", "r") as file:
    while True:
        line = file.readline()
        if not line:           # empty string means end of file
            break
        print(line.strip())    # strip() removes the trailing \n
```
 
### readlines() — Read All Lines into a List
 
```python
with open("example.txt", "r") as file:
    lines = file.readlines()    # returns a list of strings, one per line
 
print(lines)
# ['Hello World\n', 'Second line\n', 'Third line\n']
 
# Each line still has \n at the end — strip when needed
for line in lines:
    print(line.strip())
```
 
### Iterating Over a File — Most Efficient
 
The most memory-efficient way to read a file is to iterate over it directly. Python reads one line at a time without loading the entire file into memory.
 
```python
with open("example.txt", "r") as file:
    for line in file:
        print(line.strip())
 
# This is preferred for large files
# file acts as an iterator — one line per iteration
```

## Writing Files
 
### write() — Write to a File
 
```python
# Mode 'w' creates the file if it does not exist
# If the file already exists, it is COMPLETELY OVERWRITTEN
with open("output.txt", "w") as file:
    file.write("Hello, World!\n")
    file.write("Second line\n")
    file.write("Third line\n")
```
 
`write()` does not add a newline automatically. You must include `\n` explicitly.
 
### writelines() — Write Multiple Lines
 
```python
lines = ["Line one\n", "Line two\n", "Line three\n"]
 
with open("output.txt", "w") as file:
    file.writelines(lines)    # writes each string from the list
                               # does NOT add newlines automatically
```
 
### Appending to a File
 
Mode `'a'` adds content to the end of the file without erasing existing content.
 
```python
# First write — creates the file
with open("log.txt", "w") as file:
    file.write("Program started\n")
 
# Second write — appends to existing content
with open("log.txt", "a") as file:
    file.write("Step 1 completed\n")
 
with open("log.txt", "a") as file:
    file.write("Step 2 completed\n")
 
# Read the final file
with open("log.txt", "r") as file:
    print(file.read())
# Program started
# Step 1 completed
# Step 2 completed
```

## Working with File Paths
 
```python
import os
 
# Relative path — relative to where the script is running
with open("data.txt", "r") as f:
    content = f.read()
 
# Absolute path
with open("/home/user/documents/data.txt", "r") as f:
    content = f.read()
 
# Building paths safely with os.path.join — works on all operating systems
folder = "data"
filename = "students.txt"
full_path = os.path.join(folder, filename)    # "data/students.txt" on Linux/Mac
                                               # "data\\students.txt" on Windows
with open(full_path, "r") as f:
    content = f.read()
 
# Checking if a file exists before opening
if os.path.exists("data.txt"):
    with open("data.txt", "r") as f:
        content = f.read()
else:
    print("File not found")
```

## Handling File Exceptions
 
Always handle file-related exceptions when working with files in production code.
 
```python
def read_file_safe(filename):
    try:
        with open(filename, "r") as file:
            return file.read()
    except FileNotFoundError:
        print(f"File '{filename}' does not exist")
        return None
    except PermissionError:
        print(f"No permission to read '{filename}'")
        return None
    except IOError as e:
        print(f"IO error: {e}")
        return None
 
 
def write_file_safe(filename, content):
    try:
        with open(filename, "w") as file:
            file.write(content)
        print(f"Successfully wrote to '{filename}'")
        return True
    except PermissionError:
        print(f"No permission to write to '{filename}'")
        return False
    except IOError as e:
        print(f"IO error while writing: {e}")
        return False
 
 
content = read_file_safe("students.txt")
if content:
    print(content)
```

## CSV Files
 
CSV (Comma-Separated Values) is the most common format for tabular data. Each row is a line. Each column is separated by a comma.
 
```
name,age,grade,city
David,20,90,Delhi
Alice,22,85,Mumbai
Bob,21,78,Bangalore
```
 
### Reading CSV
 
```python
import csv
 
# Reading as lists
with open("students.csv", "r") as file:
    reader = csv.reader(file)
 
    header = next(reader)    # read the first row — the header
    print(header)            # ['name', 'age', 'grade', 'city']
 
    for row in reader:
        print(row)           # ['David', '20', '90', 'Delhi']
        name = row[0]
        age  = int(row[1])   # csv reads everything as string — convert manually
 
 
# Reading as dictionaries — column names become keys
with open("students.csv", "r") as file:
    reader = csv.DictReader(file)
 
    for row in reader:
        print(row)
        # {'name': 'David', 'age': '20', 'grade': '90', 'city': 'Delhi'}
        print(row["name"])
        print(int(row["age"]))
```
 
### Writing CSV
 
```python
import csv
 
students = [
    ["name", "age", "grade"],    # header row
    ["David", 20, 90],
    ["Alice", 22, 85],
    ["Bob", 21, 78]
]
 
with open("output.csv", "w", newline="") as file:
    writer = csv.writer(file)
 
    for row in students:
        writer.writerow(row)
 
    # Or write all rows at once
    writer.writerows(students)
 
 
# Writing dictionaries
students = [
    {"name": "David", "age": 20, "grade": 90},
    {"name": "Alice", "age": 22, "grade": 85},
]
 
with open("students.csv", "w", newline="") as file:
    fieldnames = ["name", "age", "grade"]
    writer = csv.DictWriter(file, fieldnames=fieldnames)
 
    writer.writeheader()       # writes the header row
    writer.writerows(students) # writes all data rows
```

## JSON Files
 
JSON (JavaScript Object Notation) is a universal format for storing and exchanging structured data. Python dictionaries and lists map directly to JSON objects and arrays.
 
```json
{
    "name": "David",
    "age": 20,
    "grades": [90, 85, 92],
    "address": {
        "city": "Delhi",
        "pincode": "110001"
    }
}
```
 
### Reading JSON
 
```python
import json
 
# Reading JSON from a file
with open("student.json", "r") as file:
    data = json.load(file)    # parses JSON and returns Python dict/list
 
print(data)
print(data["name"])           # David
print(data["grades"][0])      # 90
print(data["address"]["city"])  # Delhi
print(type(data))             # <class 'dict'>
 
# Parsing JSON from a string — common when receiving API responses
json_string = '{"name": "Alice", "score": 95}'
data = json.loads(json_string)    # loads from string
print(data["name"])               # Alice
```
 
### Writing JSON
 
```python
import json
 
student = {
    "name": "David",
    "age": 20,
    "grades": [90, 85, 92],
    "passed": True,
    "address": {
        "city": "Delhi",
        "pincode": "110001"
    }
}
 
# Writing to a file
with open("student.json", "w") as file:
    json.dump(student, file)               # compact — no formatting
    json.dump(student, file, indent=4)     # pretty-printed — readable
 
# Converting to a JSON string
json_string = json.dumps(student)
json_pretty  = json.dumps(student, indent=4, sort_keys=True)
print(json_string)
print(json_pretty)
 
# Reading the file back and verifying
with open("student.json", "r") as file:
    loaded = json.load(file)
 
print(loaded == student)    # True — data survived save and load
```
 
### What JSON Can and Cannot Store
 
JSON handles: strings, numbers (int and float), booleans (`true`/`false`), `null` (maps to Python `None`), lists, and objects (dicts).
 
JSON cannot directly handle: Python `datetime` objects, `set`, `tuple`, or custom class instances without extra work.
 
```python
import json
from datetime import datetime
 
data = {
    "name": "David",
    "birthday": datetime(2000, 6, 15)   # not JSON serializable
}
 
json.dumps(data)   # TypeError: Object of type datetime is not JSON serializable
 
# Fix — convert to string before storing
data["birthday"] = str(data["birthday"])
json.dumps(data)   # works now
```
