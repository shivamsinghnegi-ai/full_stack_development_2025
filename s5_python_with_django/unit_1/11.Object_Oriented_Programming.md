## What is Object-Oriented Programming

Object-Oriented Programming (OOP) is a way of organizing code around objects rather than functions and logic. An object is a unit that combines data (what it knows) and behavior (what it can do) into a single package.

Before OOP, programs were written as a sequence of instructions — one function after another. This works for small programs but becomes difficult to manage as the program grows. OOP solves this by grouping related data and functions together into a class.

Consider a real-world example. A student has data: a name, an age, a roll number, and marks. A student can also do things: attend class, take an exam, receive a grade. In OOP, you would create a `Student` class that holds both the data and the behaviors together.

OOP has four core principles:

**Encapsulation** — bundling data and methods that operate on that data into a single unit (a class), and restricting direct access to some of the object's components.

**Inheritance** — a class can inherit the properties and methods of another class, allowing code reuse and building hierarchies.

**Polymorphism** — different classes can provide different implementations of the same method name. The same function call behaves differently depending on which object it is called on.

**Abstraction** — hiding complex implementation details and exposing only what is necessary through a clean interface.

## Classes and Objects

A class is a blueprint. It defines the structure and behavior that objects of that class will have. An object is an instance of a class — a specific thing created from the blueprint.

Think of a class as the blueprint for a house and an object as an actual house built from that blueprint. Many houses can be built from the same blueprint — each house is separate but shares the same structure.

```python
# Defining a class — class keyword, then name in PascalCase
class Student:
    pass    # pass means the class body is empty for now

# Creating objects (instances) from the class
student1 = Student()
student2 = Student()

# student1 and student2 are two separate objects of the same class
print(type(student1))    # <class '__main__.Student'>
print(student1)          # <__main__.Student object at 0x...>
```

## The __init__ Method

`__init__` is the constructor method. Python calls it automatically every time a new object is created from the class. It is used to set up the initial state of the object by assigning values to its attributes.

The double underscores on both sides mark it as a special method that Python calls in specific situations. These are called dunder methods (short for double underscore).

```python
class Student:
    def __init__(self, name, age, roll_number):
        # self refers to the object being created
        # These lines create instance attributes
        self.name = name
        self.age = age
        self.roll_number = roll_number

# Creating objects — arguments go to __init__
s1 = Student("David", 20, 101)
s2 = Student("Alice", 22, 102)
s3 = Student("Bob", 21, 103)

# Accessing attributes using dot notation
print(s1.name)          # David
print(s2.age)           # 22
print(s3.roll_number)   # 103
```

### The self Parameter

`self` is a reference to the current object — the specific instance that the method is being called on. It is always the first parameter of any method in a class.

When you call `s1.name`, Python internally translates this to `Student.name(s1)`. The object before the dot is automatically passed as `self`.

You must write `self` as the first parameter, but you never pass it yourself when calling a method. Python passes it automatically.

```python
class Dog:
    def __init__(self, name, breed):
        self.name = name      # self.name is an attribute of this specific dog
        self.breed = breed

    def bark(self):
        print(f"{self.name} says: Woof!")    # self.name refers to this dog's name

d1 = Dog("Bruno", "Labrador")
d2 = Dog("Max", "Pug")

d1.bark()    # Bruno says: Woof!
d2.bark()    # Max says: Woof!
# same method, but self refers to d1 in the first call and d2 in the second
```

## Instance Variables and Class Variables

### Instance Variables

Instance variables are defined inside `__init__` using `self.variable_name`. Each object has its own separate copy of instance variables. Changing one object's instance variable does not affect any other object.

```python
class Circle:
    def __init__(self, radius):
        self.radius = radius    # instance variable — unique to each object

c1 = Circle(5)
c2 = Circle(10)

print(c1.radius)    # 5
print(c2.radius)    # 10

c1.radius = 7       # modifying c1's radius
print(c1.radius)    # 7
print(c2.radius)    # 10 — c2 is not affected
```

### Class Variables

Class variables are defined directly inside the class body, outside any method. They are shared across all objects of the class. Changing a class variable through the class name changes it for every object.

```python
class Student:
    school_name = "Python Academy"    # class variable — shared by all students
    student_count = 0                 # class variable — tracks total objects

    def __init__(self, name, age):
        self.name = name              # instance variable
        self.age = age                # instance variable
        Student.student_count += 1   # increment class variable each time

s1 = Student("David", 20)
s2 = Student("Alice", 22)
s3 = Student("Bob", 21)

# Accessing class variable through class name
print(Student.school_name)      # Python Academy
print(Student.student_count)    # 3

# Accessing class variable through object — works but not recommended
print(s1.school_name)           # Python Academy

# Changing class variable through class name — affects all objects
Student.school_name = "Advanced Python Academy"
print(s1.school_name)    # Advanced Python Academy
print(s2.school_name)    # Advanced Python Academy
```

### Key Difference

```python
class Example:
    class_var = "I am shared"

    def __init__(self, value):
        self.instance_var = value

e1 = Example("first")
e2 = Example("second")

# Instance variables are independent
e1.instance_var = "changed"
print(e1.instance_var)    # changed
print(e2.instance_var)    # second — not affected

# Class variable is shared
Example.class_var = "changed for all"
print(e1.class_var)    # changed for all
print(e2.class_var)    # changed for all
```

## Instance Methods

Instance methods are functions defined inside a class. They always take `self` as the first parameter and operate on the data of a specific object.

```python
class Rectangle:
    def __init__(self, length, width):
        self.length = length
        self.width = width

    def area(self):
        return self.length * self.width

    def perimeter(self):
        return 2 * (self.length + self.width)

    def is_square(self):
        return self.length == self.width

    def scale(self, factor):
        self.length = self.length * factor
        self.width = self.width * factor

r1 = Rectangle(5, 3)
r2 = Rectangle(4, 4)

print(r1.area())         # 15
print(r1.perimeter())    # 16
print(r1.is_square())    # False

print(r2.is_square())    # True

r1.scale(2)
print(r1.length)    # 10
print(r1.width)     # 6
```

## Encapsulation

Encapsulation means bundling data and methods together and controlling access to the internal data. In Python, this is done by convention using naming prefixes.

### Public Attributes (default)

Any attribute without a prefix is public — it can be accessed and modified from anywhere.

```python
class Student:
    def __init__(self, name):
        self.name = name    # public — accessible everywhere

s = Student("David")
print(s.name)     # David
s.name = "Bob"    # modifiable from outside
print(s.name)     # Bob
```

### Protected Attributes — Single Underscore

A single underscore prefix `_attribute` is a convention that signals "this is intended for internal use". Python does not enforce this — it is just a warning to other developers that they should not access it directly.

```python
class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner
        self._balance = balance    # protected — do not access directly

    def get_balance(self):
        return self._balance

    def deposit(self, amount):
        if amount > 0:
            self._balance += amount

account = BankAccount("David", 5000)
print(account._balance)    # 5000 — accessible but convention says don't
print(account.get_balance())    # 5000 — correct way to access
```

### Private Attributes — Double Underscore

A double underscore prefix `__attribute` triggers name mangling. Python renames the attribute internally to `_ClassName__attribute`, making it harder (but not impossible) to access from outside.

```python
class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner
        self.__balance = balance    # private

    def get_balance(self):
        return self.__balance    # accessible inside the class

    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount

    def withdraw(self, amount):
        if 0 < amount <= self.__balance:
            self.__balance -= amount
        else:
            print("Insufficient funds")

account = BankAccount("David", 5000)
print(account.get_balance())    # 5000 — correct way

print(account.__balance)        # AttributeError — name mangling hides it
print(account._BankAccount__balance)    # 5000 — still accessible this way
                                         # but should never be done
account.deposit(1000)
print(account.get_balance())    # 6000

account.withdraw(8000)    # Insufficient funds
print(account.get_balance())    # 6000
```

### The @property Decorator

`@property` lets you access a method like an attribute. This is the Pythonic way to implement getters and setters — it gives you clean attribute-like syntax while still allowing you to add validation logic.

```python
class Temperature:
    def __init__(self, celsius):
        self._celsius = celsius

    @property
    def celsius(self):
        return self._celsius

    @celsius.setter
    def celsius(self, value):
        if value < -273.15:
            raise ValueError("Temperature below absolute zero is not possible")
        self._celsius = value

    @property
    def fahrenheit(self):
        return (self._celsius * 9/5) + 32    # computed from celsius, read-only


t = Temperature(25)

# Access like an attribute, not like a method call
print(t.celsius)       # 25    — calls the getter
print(t.fahrenheit)    # 77.0  — computed property

# Set like an attribute — calls the setter with validation
t.celsius = 100
print(t.celsius)       # 100

t.celsius = -300       # ValueError: Temperature below absolute zero

t.fahrenheit = 50      # AttributeError — no setter defined for fahrenheit
```

## Inheritance

Inheritance allows a class (called a child class or subclass) to inherit all the attributes and methods of another class (called the parent class or superclass). The child class can then add new attributes and methods, or override existing ones.

Inheritance models an "is-a" relationship. A `Dog` is an `Animal`. A `SavingsAccount` is a `BankAccount`. A `Teacher` is a `Person`.

```python
# Parent class
class Animal:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def eat(self):
        print(f"{self.name} is eating.")

    def sleep(self):
        print(f"{self.name} is sleeping.")

    def speak(self):
        print("Some sound")


# Child class — inherits from Animal
class Dog(Animal):
    def __init__(self, name, age, breed):
        super().__init__(name, age)    # call parent's __init__
        self.breed = breed             # new attribute specific to Dog

    def speak(self):                   # override parent's speak
        print(f"{self.name} says: Woof!")

    def fetch(self):                   # new method specific to Dog
        print(f"{self.name} is fetching the ball.")


class Cat(Animal):
    def speak(self):                   # override parent's speak
        print(f"{self.name} says: Meow!")


d = Dog("Bruno", 3, "Labrador")
c = Cat("Whiskers", 5)

# Inherited methods from Animal
d.eat()      # Bruno is eating.
d.sleep()    # Bruno is sleeping.

# Overridden method
d.speak()    # Bruno says: Woof!
c.speak()    # Whiskers says: Meow!

# New method only in Dog
d.fetch()    # Bruno is fetching the ball.

# Checking inheritance
print(isinstance(d, Dog))      # True
print(isinstance(d, Animal))   # True — d is both a Dog and an Animal
print(isinstance(c, Dog))      # False
```

### super()

`super()` gives you access to the parent class. It is used most commonly in `__init__` to call the parent's constructor so you do not have to repeat the parent's initialization code.

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age
        print(f"Person created: {self.name}")

    def introduce(self):
        print(f"My name is {self.name} and I am {self.age} years old.")


class Teacher(Person):
    def __init__(self, name, age, subject):
        super().__init__(name, age)    # runs Person's __init__ first
        self.subject = subject          # then adds Teacher-specific attribute
        print(f"Teacher created: {self.name}, teaches {self.subject}")

    def introduce(self):
        super().introduce()            # call parent's introduce method
        print(f"I teach {self.subject}.")


t = Teacher("Dr. Smith", 40, "Physics")
# Person created: Dr. Smith
# Teacher created: Dr. Smith, teaches Physics

t.introduce()
# My name is Dr. Smith and I am 40 years old.
# I teach Physics.
```

### Multi-Level Inheritance

A child can itself be a parent of another class.

```python
class Animal:
    def breathe(self):
        print("Breathing")

class Mammal(Animal):
    def feed_young_with_milk(self):
        print("Feeding young with milk")

class Dog(Mammal):
    def bark(self):
        print("Woof!")

d = Dog()
d.breathe()                 # inherited from Animal
d.feed_young_with_milk()    # inherited from Mammal
d.bark()                    # defined in Dog

# Method Resolution Order — the order Python searches for a method
print(Dog.__mro__)
# (<class 'Dog'>, <class 'Mammal'>, <class 'Animal'>, <class 'object'>)
```

### Multiple Inheritance

A class can inherit from more than one parent class.

```python
class Flyable:
    def fly(self):
        print(f"{self.name} is flying.")

class Swimmable:
    def swim(self):
        print(f"{self.name} is swimming.")

class Duck(Animal, Flyable, Swimmable):
    def __init__(self, name, age):
        Animal.__init__(self, name, age)

    def speak(self):
        print(f"{self.name} says: Quack!")

d = Duck("Donald", 3)
d.speak()    # Donald says: Quack!
d.fly()      # Donald is flying.
d.swim()     # Donald is swimming.
```

## Polymorphism

Polymorphism means "many forms". In OOP, it means the same method name can behave differently depending on the class of the object it is called on.

You have already seen this with method overriding. But polymorphism goes further — you can write code that works on any object that has a particular method, without caring what type of object it is.

```python
class Dog:
    def __init__(self, name):
        self.name = name
    def speak(self):
        return f"{self.name} says Woof!"

class Cat:
    def __init__(self, name):
        self.name = name
    def speak(self):
        return f"{self.name} says Meow!"

class Duck:
    def __init__(self, name):
        self.name = name
    def speak(self):
        return f"{self.name} says Quack!"

# Polymorphic function — works with any object that has a speak() method
def make_animal_speak(animal):
    print(animal.speak())    # Python calls the correct speak() based on object type

animals = [Dog("Bruno"), Cat("Whiskers"), Duck("Donald")]

for animal in animals:
    make_animal_speak(animal)
# Bruno says Woof!
# Whiskers says Meow!
# Donald says Quack!
```

### Duck Typing

Python uses duck typing — "if it walks like a duck and quacks like a duck, it is a duck." Python does not check the type of an object. It only checks whether the object has the method or attribute you are trying to use.

```python
class Robot:
    def __init__(self, name):
        self.name = name
    def speak(self):
        return f"{self.name} says: BEEP BOOP"

# Robot is not an Animal. It does not inherit from Animal.
# But it has a speak() method, so make_animal_speak still works.
r = Robot("R2D2")
make_animal_speak(r)    # R2D2 says: BEEP BOOP
```

## The __str__ Method

When you print an object without `__str__`, Python shows something unreadable like `<__main__.Student object at 0x7f3a9c>`. By defining `__str__` in your class, you control what gets printed.

`__str__` is a special method Python calls automatically whenever you use `print()` on your object. It must return a string.

```python
class Student:
    def __init__(self, name, age, grade):
        self.name = name
        self.age = age
        self.grade = grade

    def __str__(self):
        return f"Student: {self.name}, Age: {self.age}, Grade: {self.grade}"


s1 = Student("David", 20, 90)
s2 = Student("Alice", 22, 85)

# Without __str__ — ugly output
# <__main__.Student object at 0x7f3a9c12>

# With __str__ — clean, readable output
print(s1)    # Student: David, Age: 20, Grade: 90
print(s2)    # Student: Alice, Age: 22, Grade: 85
```

```python
class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner
        self._balance = balance

    def deposit(self, amount):
        if amount > 0:
            self._balance += amount

    def withdraw(self, amount):
        if 0 < amount <= self._balance:
            self._balance -= amount
        else:
            print("Insufficient funds")

    def get_balance(self):
        return self._balance

    def __str__(self):
        return f"Account[{self.owner}] — Balance: {self._balance}"


acc = BankAccount("David", 5000)
acc.deposit(2000)
print(acc)    # Account[David] — Balance: 7000
```

## A Complete Example

This example brings all the concepts covered in these notes together — classes, objects, `__init__`, instance methods, class variables, encapsulation, inheritance, `super()`, method overriding, polymorphism, and `__str__`.

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def introduce(self):
        print(f"Hi, I am {self.name} and I am {self.age} years old.")

    def __str__(self):
        return f"Person({self.name}, {self.age})"


class Student(Person):
    school_name = "Python Academy"    # class variable — shared by all students

    def __init__(self, name, age, roll_number, grade):
        super().__init__(name, age)       # call Person's __init__
        self.roll_number = roll_number    # Student-specific attributes
        self._grade = grade               # protected — use get_grade() to access

    def get_grade(self):
        return self._grade

    def set_grade(self, grade):
        if 0 <= grade <= 100:
            self._grade = grade
        else:
            print("Grade must be between 0 and 100")

    def introduce(self):
        super().introduce()    # call Person's introduce first
        print(f"I study at {Student.school_name}. My roll number is {self.roll_number}.")

    def is_passing(self):
        return self._grade >= 40

    def __str__(self):
        return f"Student({self.name}, Roll: {self.roll_number}, Grade: {self._grade})"


class Teacher(Person):
    def __init__(self, name, age, subject):
        super().__init__(name, age)
        self.subject = subject

    def introduce(self):
        super().introduce()
        print(f"I teach {self.subject} at {Student.school_name}.")

    def __str__(self):
        return f"Teacher({self.name}, Subject: {self.subject})"


# Creating objects
s1 = Student("David", 20, 101, 85)
s2 = Student("Alice", 22, 102, 35)
t1 = Teacher("Dr. Smith", 40, "Physics")

# __str__ in action
print(s1)    # Student(David, Roll: 101, Grade: 85)
print(t1)    # Teacher(Dr. Smith, Subject: Physics)

# Calling methods
s1.introduce()
# Hi, I am David and I am 20 years old.
# I study at Python Academy. My roll number is 101.

t1.introduce()
# Hi, I am Dr. Smith and I am 40 years old.
# I teach Physics at Python Academy.

# Encapsulation — using getter and setter
print(s1.get_grade())    # 85
s1.set_grade(90)
print(s1.get_grade())    # 90
s1.set_grade(150)        # Grade must be between 0 and 100

# Class variable
print(Student.school_name)    # Python Academy

# Checking pass/fail
print(s1.is_passing())    # True
print(s2.is_passing())    # False

# Polymorphism — same method name, different behavior
people = [s1, s2, t1]
for person in people:
    print(person)    # calls __str__ of each class
```
