## 1. Use of Server and Types of Servers

### What is a Server

A server is a dedicated computer (hardware) or a program (software) that provides services, data, or resources to other computers (clients) over a network.

### Key Definition

> "A server is a computer program or device that provides a service to another computer program (client) and its user, often over a network."

### Use of Server in a Network

- Centralizes data storage, making it easier to manage, back up, and secure.
- Provides controlled access to resources (files, printers, applications) to multiple clients simultaneously.
- Enables centralized authentication and authorization (login credentials, permissions).
- Reduces redundancy — a single server can serve many clients instead of duplicating resources on every machine.
- Improves security by allowing centralized monitoring and control of network activity.

### Types of Servers and Their Roles

| Server Type | Role / Function |
|---|---|
| File Server | Stores and manages files centrally; clients access, upload, and download shared files |
| Print Server | Manages print requests from multiple clients and controls access to shared printers |
| Web Server | Hosts websites and delivers web pages to clients over HTTP/HTTPS |
| Mail Server | Handles sending, receiving, and storing of email messages (e.g., SMTP, POP3, IMAP based) |
| Database Server | Stores and manages databases; processes queries sent by client applications |
| DNS Server | Translates domain names (e.g., www.example.com) into IP addresses |
| DHCP Server | Automatically assigns IP addresses to devices on a network |
| Application Server | Hosts and runs application software, allowing clients to run applications without local installation |
| Proxy Server | Acts as an intermediary between client and other servers, providing caching, filtering, and security |
| FTP Server | Enables file transfer between client and server using File Transfer Protocol |

## 2. Internet, Intranet, and Extranet

These three terms describe the **scope of access** of a network — who can use it and to what extent.

### Internet

- A global system of interconnected networks that uses standard protocols (TCP/IP) to link devices worldwide.
- Publicly accessible; anyone with a connection and appropriate device can access it.
- Not owned by a single organization; it is a network of networks.

### Key Definition

> "The Internet is a worldwide, publicly accessible network of interconnected computer networks that transmit data using the standard Internet Protocol (IP)."

### Intranet

- A private network based on Internet technologies (TCP/IP, web browsers) but restricted to use within a single organization.
- Accessible only to authorized members of the organization (employees, students, staff).
- Used for internal communication, sharing internal documents, internal portals.

### Key Definition

> "An Intranet is a private network, accessible only to an organization's staff, that uses Internet-based technologies to share information securely within the organization."

### Extranet

- An extension of an Intranet that allows controlled access to selected external users, such as business partners, vendors, or clients.
- Uses authentication (usernames, passwords, VPN) to allow limited outside access to specific parts of the organization's network.

### Key Definition

> "An Extranet is a controlled private network that allows access to partners, vendors, or authorized outside users, while keeping the rest of the organization's internal network private."

### Comparison Table (Exam Important)

| Parameter | Internet | Intranet | Extranet |
|---|---|---|---|
| Accessibility | Public, open to everyone | Private, restricted to organization members | Private, restricted to organization + selected outsiders |
| Ownership | No single owner (network of networks) | Owned and managed by a single organization | Owned by organization, shared access given to partners |
| Security | Least secure (open access) | Highly secure (internal only) | Secure, but slightly more exposed due to external access |
| Purpose | Global information sharing and communication | Internal organizational communication and resource sharing | Sharing selected information with external stakeholders |
| Example | Browsing any public website | A company's internal HR portal | A supplier accessing a company's order management system |

### Exam Points

- Intranet is essentially a "mini Internet" restricted to one organization.
- Extranet = Intranet + limited external access (this relationship is a common exam one-liner).
- All three use the same underlying technology (TCP/IP, web-based access) — the difference lies in scope of accessibility, not the technology itself.

## 3. Connection Oriented and Connectionless Service

These describe two different approaches a network can use to deliver data between sender and receiver.

### Connection Oriented Service

- A logical connection (session) is established between sender and receiver **before** data transfer begins.
- Data is delivered in the same order it was sent (sequencing maintained).
- Involves three phases: **connection establishment, data transfer, connection termination**.
- More reliable, but has higher overhead and setup time.
- Example protocol: TCP (Transmission Control Protocol).
- Analogy: Similar to a telephone call — you must first establish a connection (dial and connect) before you can talk, and you end the call when done.

### Key Definition 

> "Connection-oriented service is a network communication method in which a dedicated logical connection is established between sender and receiver before data transmission begins."

### Connectionless Service

- No prior connection is established; data (in the form of packets) is sent independently, each carrying full addressing information.
- Packets may arrive out of order, may be lost, or duplicated — no guarantee of delivery or order.
- Lower overhead, faster, but less reliable.
- Example protocol: UDP (User Datagram Protocol).
- Analogy: Similar to sending postal letters — each letter is independent, may take a different route, and may arrive in a different order than sent, with no guarantee of delivery.

### Key Definition

> "Connectionless service is a network communication method in which data is transmitted without establishing a prior connection, and each packet is routed independently."

### Comparison Table

| Parameter | Connection Oriented | Connectionless |
|---|---|---|
| Connection Setup | Required before data transfer | Not required |
| Reliability | High (acknowledgment-based, guaranteed delivery) | Low (best-effort delivery, no guarantee) |
| Order of Delivery | Maintained (sequenced) | Not guaranteed |
| Overhead | Higher (due to connection setup/teardown) | Lower |
| Speed | Comparatively slower | Comparatively faster |
| Example Protocol | TCP | UDP |
| Real-world Analogy | Telephone call | Postal mail |
| Suitable For | Applications needing reliability (file transfer, email) | Applications needing speed (video streaming, online gaming, VoIP) |
