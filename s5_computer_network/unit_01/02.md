## 1. Network Hardware

Network hardware refers to the physical devices required to establish and operate a computer network.

### Key Definition

> "Network hardware consists of the physical devices that connect computers and other electronic devices for the purpose of sending and receiving data."

### Major Hardware Components

| Hardware | Function |
|---|---|
| NIC (Network Interface Card) | Enables a device to physically connect to a network and communicate |
| Modem | Converts digital signals to analog and vice versa for transmission over telephone/cable lines |
| Hub | Basic device to connect multiple computers in a LAN, broadcasts data to all ports |
| Switch | Connects devices in a LAN and forwards data only to the intended destination |
| Router | Connects different networks and forwards data packets between them based on IP address |
| Cables/Connectors | Physical medium for wired data transmission (twisted pair, coaxial, fiber) |
| Access Point | Allows wireless devices to connect to a wired network |

### Classification Basis

Network hardware can be classified based on:
- **Function**: Connecting device (switch, router) vs transmission medium (cable).
- **Network type used in**: LAN devices (hub, switch) vs WAN devices (router, modem).

### Exam Points

- NIC is essential for any device to become part of a network; every node must have one.
- Hub works at the physical layer (broadcasts to all), Switch works more intelligently (forwards selectively) —

## 2. Network Software

Network software refers to the programs and protocols that manage, control, and enable communication and resource sharing across a network.

### Key Definition 

> "Network software is a set of programs that helps in establishing, monitoring, and managing communication between devices connected in a network."

### Categories of Network Software

| Category | Description | Example |
|---|---|---|
| Network Operating System (NOS) | OS designed to support networking functions, manage shared resources | Windows Server, Linux |
| Protocol Software | Implements rules for data communication | TCP/IP stack |
| Network Management Software | Monitors and manages network performance, security, faults | SNMP-based tools, Nagios |
| Client Software | Requests services/resources from server | Web browsers, email clients |
| Server Software | Provides services/resources to clients | Web server software (Apache), Mail server software |

### Functions of Network Software

- Managing user access and permissions (security).
- Enabling communication between hardware devices using protocols.
- Providing network services (file sharing, printing, email).
- Monitoring network traffic and performance.

### Exam Points

- Network hardware provides the physical means of connection; network software provides the logical control and management.
- NOS (Network Operating System) is a critical exam term — must be able to define and give examples.

## 3. Network Simulator

A network simulator is a software tool used to model, design, and analyze the behavior of a computer network without requiring actual physical hardware.

### Key Definition

> "A network simulator is a software application that predicts the behavior of a computer network by calculating the interaction between different network entities."

### Why Network Simulators are Used

- Testing network design before actual physical implementation (cost saving).
- Studying protocol behavior under different conditions.
- Training and educational purposes (learning networking concepts practically).
- Analyzing network performance (throughput, delay, packet loss) under simulated traffic.

### Common Network Simulators

| Simulator | Use |
|---|---|
| Cisco Packet Tracer | Designing and simulating LAN/WAN topologies, widely used for CCNA-level learning |
| NS2 / NS3 (Network Simulator 2/3) | Research-level simulation of protocols and network behavior |
| GNS3 | Emulates real network hardware for advanced testing |
| OPNET | Used for network performance modeling in industry |

### Exam Points

- Network simulators allow testing without real hardware, reducing cost and risk before deployment.
- Cisco Packet Tracer is the most commonly used tool at the academic/learning level.

## 4. Peer-to-Peer Network vs Client-Server Network

### Peer-to-Peer (P2P) Network

- All computers (nodes) have equal status; each can act as both client and server.
- No centralized control or dedicated server.
- Resources (files, printers) are shared directly between nodes.
- Suitable for small networks (home, small office).

### Client-Server Network

- Dedicated server(s) provide services and resources; clients request and use them.
- Centralized control, management, and security.
- Suitable for medium to large organizations.

### Comparison Table (Exam Important)

| Parameter | Peer-to-Peer (P2P) | Client-Server |
|---|---|---|
| Architecture | All nodes are equal | Dedicated server + multiple clients |
| Control | Decentralized | Centralized |
| Cost | Low (no dedicated server needed) | High (requires dedicated server hardware/software) |
| Security | Weaker, managed individually by each node | Stronger, managed centrally |
| Scalability | Poor for large networks | Good, scales well for large networks |
| Performance | Degrades as more nodes are added | Server can be optimized to handle more clients |
| Maintenance | Difficult as network grows (no central admin) | Easier, centralized administration |
| Example | Home network sharing files between 2-3 PCs | Organizational network with a file/database server |

### Exam Points

- P2P networks are simple and cheap but not scalable or secure for large use cases.
- Client-Server networks are the standard architecture for organizational/business networks due to centralized control and security.
- A node in P2P can act as both requester and provider of services; in Client-Server, roles are fixed (client only requests, server only provides).

## 5. Line Configuration

Line configuration refers to the way two or more communication devices are attached to a communication link (the physical arrangement of devices on a link).

### Key Definition 

> "Line configuration refers to the method of connecting two or more communication devices to a link for the purpose of data transmission."

### Types of Line Configuration

#### 5.1 Point-to-Point Configuration

- A dedicated link is established between exactly two devices.
- The entire capacity of the link is reserved for transmission between these two devices only.
- Example: Two computers connected directly via a cable, or a leased line connection.

#### 5.2 Multipoint (Multidrop) Configuration

- More than two devices share a single link.
- Capacity of the channel is shared, either spatially (all devices use it simultaneously) or temporally (devices take turns).
- Example: A LAN where multiple computers share the same communication medium (e.g., a bus topology network).

### Comparison Table

| Parameter | Point-to-Point | Multipoint |
|---|---|---|
| Number of Devices | Exactly 2 | More than 2 |
| Link Capacity | Fully dedicated to the two devices | Shared among multiple devices |
| Example | Leased line between two offices | LAN with shared bus/hub |

### Exam Points

- Line configuration is different from topology — configuration is about how many devices attach to a single link, while topology is about the overall physical/logical layout.
- Point-to-point gives dedicated bandwidth; multipoint requires sharing, which can lead to contention/collisions.
