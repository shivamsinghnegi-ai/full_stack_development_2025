## 1. Protocols

### What is a Protocol

A protocol is a set of rules and conventions that govern how data is formatted, transmitted, and received between devices on a network, ensuring that communication happens correctly and consistently.

### Key Definition (Quotable)

> "A protocol is a set of rules that govern data communication, defining what is communicated, how it is communicated, and when it is communicated."

### Why Protocols are Needed

- Devices from different manufacturers, running different software/hardware, need a common language to communicate — protocols provide this standardization.
- Without protocols, data sent by one device may not be understood or correctly interpreted by another.
- Protocols ensure reliable, ordered, and error-checked communication.
- Enable interoperability between heterogeneous systems (different OS, hardware, vendors).

### Three Key Elements of a Protocol

| Element | Description |
|---|---|
| Syntax | Structure/format of data — how the data is arranged (e.g., which bits represent address, which represent data) |
| Semantics | Meaning of each section of bits — what action or response is expected |
| Timing | When data should be sent and at what speed (synchronization between sender and receiver) |

### Functions Performed by Protocols

- Data sequencing (breaking data into sequenced packets and reassembling correctly).
- Data routing (finding the best path from sender to receiver).
- Flow control (ensuring sender does not overwhelm receiver).
- Error control (detecting and correcting transmission errors).
- Addressing (identifying sender and receiver uniquely on the network).

### Examples of Common Protocols

| Protocol | Full Form / Purpose |
|---|---|
| TCP | Transmission Control Protocol — reliable, connection-oriented data delivery |
| IP | Internet Protocol — addressing and routing of packets |
| HTTP | HyperText Transfer Protocol — used for web page transfer |
| FTP | File Transfer Protocol — used for transferring files |
| SMTP | Simple Mail Transfer Protocol — used for sending emails |
| UDP | User Datagram Protocol — fast, connectionless data delivery |

### Exam Points

- Syntax, Semantics, and Timing are the three core components of any protocol — this is a frequently asked definition-based question.
- A protocol is essentially the "rulebook" of communication; without agreed rules, devices cannot understand each other regardless of how good the hardware is.
