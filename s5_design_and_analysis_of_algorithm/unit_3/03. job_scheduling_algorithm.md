# Job Scheduling Algorithm

# 1. Introduction

Before learning the **Job Scheduling Algorithm**, let us first understand the type of problem that this algorithm is designed to solve.

Suppose you own a company and you receive several projects from different clients.

Each project offers a different amount of profit, and every project has a deadline before which it must be completed.

However, there is one limitation.

You have only **one machine**, **one employee**, or **one processor**, which means you can complete only **one job at a time**.

Now imagine that there are more jobs than the available time slots.

Since it is impossible to complete every job, you must decide **which jobs should be selected** and **which jobs should be rejected**.

Naturally, your objective is not simply to complete as many jobs as possible.

Instead, your objective is to complete those jobs that will earn the **maximum possible profit**.

This is exactly the problem solved by the **Job Scheduling Algorithm**.

The Job Scheduling Algorithm is one of the most popular applications of the **Greedy Algorithm** because at every step it tries to make the best possible decision that leads towards the maximum overall profit.

Instead of selecting jobs randomly, the algorithm follows a systematic approach to determine the most profitable schedule.

Before understanding the implementation of this algorithm, let us first understand the Job Scheduling Problem in detail.

---

# 2. What is the Job Scheduling Problem?

Suppose there are several jobs waiting to be executed.

Every job has three important pieces of information associated with it.

- A unique Job ID
- A Deadline
- A Profit

Each job requires exactly **one unit of time** to complete.

This is one of the most important assumptions of the Job Scheduling Problem.

It means that regardless of whether the job offers a profit of **₹10** or **₹10,000**, every job still requires only **one time slot** for its execution.

Now suppose the available time slots are limited.

Since every job cannot be completed, we need to decide which jobs should be performed so that the total profit becomes maximum.

This optimization problem is known as the **Job Scheduling Problem**.

---

## Definition

> The **Job Scheduling Problem** is the problem of selecting and scheduling jobs in such a way that every selected job is completed before its deadline while maximizing the total profit.

In simple words,

our objective is

> **Complete the most profitable set of jobs without missing their deadlines.**

---

Every job consists of three values.

| Field | Description |
|--------|-------------|
| Job ID | A unique name used to identify the job |
| Deadline | The latest time slot in which the job can be completed |
| Profit | The amount earned after successfully completing the job |

| J2 | 1 | 19 |
| J3 | 2 | 27 |
| J4 | 1 | 25 |
| J5 | 3 | 15 |

```cpp
    {'J2', 1, 19},
    {'J3', 2, 27},
    {'J4', 1, 25},
    {'J5', 3, 15}

```

Here,

For example,


means

- Job ID = J1
- Deadline = 2

Similarly,

```text

```

means

- Job ID = J3
- Deadline = 2
- Profit = 27

Notice that,

every job has its own deadline as well as its own profit.

Some jobs offer higher profit, while others offer lower profit.

Similarly, some jobs must be completed earlier, while others can be completed later.

Therefore, choosing the correct jobs becomes very important.

# 4. What Does the Deadline Mean?

Many beginners get confused about the meaning of a deadline.

Let us understand it carefully.

The deadline of a job tells us the **latest time slot** in which that particular job can be completed.

Suppose a job has

```text
Deadline = 3
```

This does **not** mean that the job requires three units of time.

Remember, every job requires only **one unit of time**.

It simply means that the job must be completed **on or before the third time slot**.

For example,

if

```text
Deadline = 3
```

then the job may be scheduled in

```text
Time Slot 1
```

or

```text
Time Slot 2
```

or

```text
Time Slot 3
```

However,

it **cannot** be scheduled in

```text
Time Slot 4
```

because doing so would miss the deadline.

Similarly,

if

```text
Deadline = 1
```

then that job can only be performed during

```text
Time Slot 1
```

If Time Slot 1 is already occupied, then that job can no longer be selected.

This concept plays a very important role in the Job Scheduling Algorithm.

# 5. Understanding the Problem Through a Detailed Example

Now let us understand the complete problem using the given input.

Suppose we have the following jobs.

| Job ID | Deadline | Profit |
|---------|---------:|-------:|
| J1 | 2 | 100 |
| J2 | 1 | 19 |
| J3 | 2 | 27 |
| J4 | 1 | 25 |
| J5 | 3 | 15 |

The maximum deadline among all the jobs is

```text
3
```

Therefore,

we have only three available time slots.

```text
Time Slot 1

Time Slot 2

Time Slot 3
```

Since every job requires exactly one unit of time, only one job can be placed inside one time slot.

This means

```text
Time Slot 1 → One Job

Time Slot 2 → One Job

Time Slot 3 → One Job
```

Now count the total number of jobs.

```text
5 Jobs
```

But how many time slots do we actually have?

```text
3 Time Slots
```

Immediately,

we can observe one important fact.

```text
Total Jobs > Available Time Slots
```

This means, it is impossible to perform every job.

Some jobs must be rejected.

Now the next question becomes,

> **Which jobs should we select so that the total profit becomes maximum?**

Should we select the jobs in the same order in which they are given?

Should we always select the job having the earliest deadline?

Or,

should we always select the job having the highest profit?

These are the questions that the Job Scheduling Algorithm answers.

Instead of making random decisions, the algorithm follows a greedy strategy that always tries to maximize the total profit while ensuring that every selected job finishes before its deadline.

# 6. Why Can't We Select Jobs Randomly?

Now that we understand the input format of the Job Scheduling Problem,

let us think about another important question.

> **Can we simply select the jobs randomly?**

At first glance, this may seem like a reasonable idea.

Suppose we start selecting jobs one by one in the order in which they are given.

Let us see what happens.

Given Jobs

```text

```

which is **after its deadline**.

Therefore,

Job **J2** cannot be completed successfully.

Similarly,

Job **J3** has

```text
Deadline = 2
```

but we are executing it in

```text
Time Slot 3
```

Again,

its deadline is missed.

Therefore, simply selecting jobs randomly does **not** guarantee a valid schedule.

## Important Observation

Random selection may

- Miss deadlines.
- Reject high-profit jobs.
- Produce a very small total profit.

Therefore, we need a better strategy.

The Job Scheduling Algorithm uses a **Greedy Approach** to solve this problem efficiently.

# 7. Why Do We Sort the Jobs According to Profit?

After understanding that random selection is not a good idea, the next question naturally becomes,

> **Which job should we select first?**

Think about this carefully.

Suppose you have only one available time slot.

Would you like to perform

```text
Profit = 20
```

or

```text
Profit = 200
```

Obviously,

you would prefer the second job because it earns much more profit.

This is the main intuition behind the Job Scheduling Algorithm.

Instead of considering low-profit jobs first, the algorithm always tries to schedule the highest-profit jobs first.

This greedy decision helps maximize the overall profit.

---

|---------|---------:|-------:|

## Why Don't We Sort According to Deadline?

Many beginners ask,

> **Why don't we sort the jobs according to their deadlines?**

Suppose we sort by deadline.

The order becomes

| Job | Deadline | Profit |
|------|---------:|-------:|
| J2 | 1 | 19 |
| J4 | 1 | 25 |
| J1 | 2 | 100 |
| J3 | 2 | 27 |
| J5 | 3 | 15 |

Now,

we may end up scheduling

```text
J2
```

before

```text
J1
```

Although this satisfies the deadline, we lose the opportunity to earn a much larger profit.

Therefore, sorting by deadline alone does **not** maximize the total profit.

This is why the greedy strategy always sorts the jobs according to **profit**.

---

## Important Observation

Sorting by profit ensures that

- High-profit jobs are considered first.
- Expensive jobs are less likely to be rejected.
- The final schedule has the maximum possible profit.

# 8. Why Do We Need a Slot Vector?

After sorting the jobs,

the next question becomes,

> **Where should we place the selected jobs?**

Since every job requires exactly **one unit of time**,

we need a way to represent the available time slots.

For this purpose,

we create a vector called the **Slot Vector**.

The Slot Vector represents all the available time slots.

Initially,

every slot is empty.

As we schedule jobs,

these empty slots gradually become occupied.

---

## Finding the Maximum Deadline

Consider the given jobs.

| Job ID | Deadline |
|---------|----------|
| J1 | 2 |
| J2 | 1 |
| J3 | 2 |
| J4 | 1 |
| J5 | 3 |

The maximum deadline is

```text
3
```

Therefore,

we need exactly

```text
3 Time Slots
```

---

## Creating the Slot Vector

In C++,

we create it as

```cpp
vector<int> slot(maxDeadline + 1, -1);
```

---

## Why Do We Use `maxDeadline + 1`?

Suppose

```text
Maximum Deadline = 3
```

The valid time slots are

```text
1

2

3
```

If we create

```cpp
vector<int> slot(3);
```

then the indices become

```text
0

1

2
```

which does not match the actual deadline numbers.

Instead,

we create

```cpp
vector<int> slot(4);
```

Now the indices become

```text
0

1

2

3
```

Notice that,

we simply ignore index

```text
0
```

and directly use

```text
Index 1

↓

Time Slot 1

Index 2

↓

Time Slot 2

Index 3

↓

Time Slot 3
```

This makes the implementation much simpler because the deadline value directly corresponds to the vector index.

---

## Why Is Every Value Initialized with `-1`?

Initially,

no job has been scheduled.

Therefore,

every slot is empty.

To indicate that a slot is empty,

we store

```text
-1
```

The initial Slot Vector becomes

| Index | Value |
|------:|------:|
| 0 | -1 |
| 1 | -1 |
| 2 | -1 |
| 3 | -1 |

Here,

```text
-1
```

simply means

> **No job has been assigned to this time slot yet.**

Later,

whenever a job is scheduled,

that `-1` value is replaced by the corresponding job index.

For example,

after scheduling

```text
J1
```

the Slot Vector may become

| Index | Value |
|------:|------:|
| 0 | -1 |
| 1 | -1 |
| 2 | J1 |
| 3 | -1 |

This indicates that

Time Slot 2 is now occupied by Job J1,

while the remaining slots are still available.

---

## Important Observation

The Slot Vector helps us answer one very important question.

> **Is this particular time slot already occupied?**

If the value is

```text
-1
```

the slot is free.

Otherwise, it already contains a scheduled job.

This simple idea allows the algorithm to efficiently schedule jobs while respecting their deadlines.

# Why Does the Algorithm Search Backwards?

Many students wonder,

> Why do we start checking from the deadline and move backwards instead of checking from Time Slot 1?

The answer is very simple.

Suppose

```text
J1
```

has

```text
Deadline = 3
```

It can be performed in

```text
Time Slot 1

or

Time Slot 2

or

Time Slot 3
```

Now suppose we immediately place it in

```text
Time Slot 1
```

Then,

a future job having

```text
Deadline = 1
```

will have no place to execute.

However,

if we place

```text
J1
```

in its **latest possible slot**,

we leave the earlier slots free for jobs having smaller deadlines.

This greedy decision increases the possibility of scheduling more jobs,

which ultimately leads to a larger total profit.

Therefore,

the algorithm always checks

```text
Deadline

↓

Deadline - 1

↓

Deadline - 2

↓

...

↓

1
```

instead of moving in the forward direction.

---

# Important Observations

After completing the dry run,

remember the following points.

- The jobs are first sorted according to **profit** in descending order.
- The algorithm always considers the highest-profit job first.
- Every job is placed in the **latest available slot** before its deadline.
- If the deadline slot is occupied, the algorithm searches backwards.
- If no free slot exists before the deadline, the job is rejected.
- This greedy strategy helps maximize the total profit while ensuring that every selected job finishes before its deadline.

# Cpp Code:

```cpp
#include <iostream>      // Used for input and output
#include <vector>        // Used to store all jobs
#include <algorithm>     // Used for sort()
using namespace std;

//------------------------------------------------------------
// Structure to represent one Job
//------------------------------------------------------------
struct Job
{
    char id;         // Unique Job ID
    int deadline;    // Deadline of the job
    int profit;      // Profit earned after completing the job
};

//------------------------------------------------------------
// Comparator Function
//
// Purpose:
// Sort all jobs in descending order of profit.
//
// Highest profit job should come first because
// Job Scheduling follows the Greedy Approach.
//------------------------------------------------------------
bool compare(Job a, Job b)
{
    return a.profit > b.profit;
}

//------------------------------------------------------------
// Job Scheduling Function
//------------------------------------------------------------
void jobScheduling(vector<Job> &jobs)
{
    //--------------------------------------------------------
    // Step 1
    //
    // Sort all jobs according to maximum profit.
    //
    // After sorting,
    // Highest Profit
    //        ↓
    // Lowest Profit
    //--------------------------------------------------------
    sort(jobs.begin(), jobs.end(), compare);

    //--------------------------------------------------------
    // Step 2
    //
    // Find the maximum deadline among all jobs.
    //
    // Why?
    //
    // Because the number of available time slots
    // depends upon the largest deadline.
    //--------------------------------------------------------
    int maxDeadline = 0;

    for (Job job : jobs)
    {
        maxDeadline = max(maxDeadline, job.deadline);
    }

    //--------------------------------------------------------
    // Step 3
    //
    // Create Slot Vector
    //
    // Size = Maximum Deadline + 1
    //
    // Why +1?
    //
    // Because deadlines start from 1.
    //
    // Index 0 is ignored.
    //
    // Every value is initialized with -1.
    //
    // -1 means
    //
    // "No Job Assigned Yet"
    //--------------------------------------------------------
    vector<int> slot(maxDeadline + 1, -1);

    //--------------------------------------------------------
    // Variable to store total profit.
    //--------------------------------------------------------
    int totalProfit = 0;

    //--------------------------------------------------------
    // Variable to count scheduled jobs.
    //--------------------------------------------------------
    int totalJobs = 0;

    //--------------------------------------------------------
    // Step 4
    //
    // Traverse every job one by one.
    //
    // Since jobs are already sorted,
    // highest profit jobs are processed first.
    //--------------------------------------------------------
    for (int i = 0; i < jobs.size(); i++)
    {
        //----------------------------------------------------
        // Start checking from the job's deadline.
        //
        // If that slot is occupied,
        // move backwards.
        //
        // Example
        //
        // Deadline = 3
        //
        // Check
        //
        // 3
        // ↓
        // 2
        // ↓
        // 1
        //----------------------------------------------------
        for (int j = jobs[i].deadline; j >= 1; j--)
        {
            //------------------------------------------------
            // If the slot is empty,
            // schedule the current job.
            //------------------------------------------------
            if (slot[j] == -1)
            {
                //--------------------------------------------
                // Store the index of the job.
                //
                // We store the index instead of the job
                // itself because later we can easily
                // access the complete job information.
                //--------------------------------------------
                slot[j] = i;

                //--------------------------------------------
                // Add current job profit.
                //--------------------------------------------
                totalProfit += jobs[i].profit;

                //--------------------------------------------
                // Increase scheduled job count.
                //--------------------------------------------
                totalJobs++;

                //--------------------------------------------
                // Stop searching because the job
                // has already been scheduled.
                //--------------------------------------------
                break;
            }
        }
    }

    //--------------------------------------------------------
    // Print Scheduled Jobs
    //--------------------------------------------------------
    cout << "Selected Jobs:\n\n";

    //--------------------------------------------------------
    // Traverse every slot.
    //
    // Ignore slot 0.
    //--------------------------------------------------------
    for (int i = 1; i <= maxDeadline; i++)
    {
        //----------------------------------------------------
        // If slot contains a job
        //----------------------------------------------------
        if (slot[i] != -1)
        {
            cout << "Time Slot "
                 << i
                 << " -> "
                 << jobs[slot[i]].id
                 << endl;
        }
    }

    cout << "\nTotal Jobs Completed : "
         << totalJobs
         << endl;

    cout << "Maximum Profit : "
         << totalProfit
         << endl;
}

//------------------------------------------------------------
// Driver Function
//------------------------------------------------------------
int main()
{
    //--------------------------------------------------------
    // Input Example
    //
    // Job ID
    // Deadline
    // Profit
    //--------------------------------------------------------
    vector<Job> jobs =
    {
        {'J', 2, 100},   // J1
        {'K', 1, 19},    // J2
        {'L', 2, 27},    // J3
        {'M', 1, 25},    // J4
        {'N', 3, 15}     // J5
    };

    //--------------------------------------------------------
    // Call Job Scheduling Algorithm
    //--------------------------------------------------------
    jobScheduling(jobs);

    return 0;
}
```
