# Fractional Knapsack Algorithm

# 1. Introduction

Before learning the **Fractional Knapsack Algorithm**, let us first understand the type of problem that this algorithm is designed to solve.

Suppose you own a bag that has a limited carrying capacity.

You are standing in front of several valuable items.

Every item has

- A Weight
- A Profit (or Value)

Your objective is to place items inside the bag in such a way that the total profit becomes as large as possible.

However, there is one important limitation.

The bag cannot carry unlimited weight.

It can carry only a fixed maximum weight, which is known as the **Knapsack Capacity**.

Now imagine that the total weight of all the available items is much larger than the capacity of the bag.

Since every item cannot fit inside the bag,

you must decide

- Which items should be selected?
- Which items should be ignored?
- Whether an item should be taken completely or only partially?

These are exactly the questions answered by the **Fractional Knapsack Algorithm**.

The Fractional Knapsack Algorithm is one of the most popular applications of the **Greedy Algorithm** because, at every step, it makes the locally best decision that eventually leads to the maximum overall profit.

Unlike the 0/1 Knapsack Problem, the Fractional Knapsack Problem allows us to take **only a fraction of an item** whenever the complete item cannot fit inside the remaining capacity of the bag.

This single difference makes the Fractional Knapsack Problem much easier to solve using a greedy approach.

Before understanding the implementation, let us first understand the complete problem in detail.

---

# 2. What is the Fractional Knapsack Problem?

Suppose there are several items available.

Every item has two important properties.

- Profit (or Value)
- Weight

We are also given a bag having a fixed maximum capacity.

Our objective is to place items inside the bag in such a way that

- The total weight never exceeds the capacity of the bag.
- The total profit becomes as large as possible.

The most important characteristic of this problem is that

> **An item can be divided into smaller parts.**

This means

if an item cannot fit completely,

we are allowed to take only the required fraction of that item.

This property is the reason why this problem is called the **Fractional Knapsack Problem**.

---

## Definition

> The **Fractional Knapsack Problem** is the problem of selecting items in such a way that the total profit becomes maximum while ensuring that the total weight never exceeds the capacity of the knapsack. If required, an item may be taken partially.

In simple words, our objective is

> **Fill the knapsack with the most valuable items while staying within the given weight limit.**

---

# 3. Understanding the Input

Every item consists of two values.

| Field | Description |
|--------|-------------|
| Profit | Profit earned if the complete item is selected |
| Weight | Weight occupied by the item inside the knapsack |

Suppose we are given the following items.

| Item | Profit | Weight |
|------|-------:|-------:|
| I1 | 60 | 10 |
| I2 | 100 | 20 |
| I3 | 120 | 30 |

Similarly,

```text

```

contains

- Profit = 120
- Weight = 30

Notice that, every item has a different profit as well as a different weight.

Some items are heavier.

Some items are lighter.

Some items give larger profit, while others give smaller profit.

Therefore, choosing the correct items becomes very important.

---

Apart from the list of items,

we are also given another input known as the **Knapsack Capacity**.

Suppose

```cpp
int capacity = 50;
```

This means

```text

=

```

In other words,

the total weight of all the selected items must never become greater than

```text
50
```

For example,

suppose the current weight inside the bag is

```text
35
```

Then,

the remaining capacity becomes

```text
50 - 35

=

15
```

This means

the bag can now carry only

```text
15 more units of weight.
```

Any decision taken by the algorithm must always satisfy this weight constraint.

---

# 5. Understanding the Output

After processing all the items, the algorithm should determine

- Which items are selected completely.
- Which item is selected partially (if required).
- The total weight inside the knapsack.
- The maximum profit obtained.

For the given example, the final answer will not simply be

> "Select Item 1, Item 2 and Item 3."

Instead, the algorithm may decide to

- Take Item 1 completely.
- Take Item 2 completely.
- Take only a fraction of Item 3.

This happens because the remaining capacity may not be sufficient to accommodate the complete third item.

This is the most important feature of the Fractional Knapsack Problem.

---

# 6. Important Assumption

Before moving further,

remember one very important assumption.

In the Fractional Knapsack Problem,

every item is divisible.

This means

if an item weighs

```text
30 kg
```

and the remaining capacity is only

```text
20 kg
```

then we are allowed to take only

```text
20 kg
```

of that item.

Similarly,

if only

```text
5 kg
```

of capacity is available,

then we can take only

```text
5 kg
```

from that item.

Because of this property, the algorithm is capable of producing the maximum possible profit using a greedy strategy.

# 8. Why Can't We Simply Choose the Highest Profit Item?

After understanding that random selection is not a good idea, another question naturally arises.

> **Why don't we simply choose the item having the highest profit?**

At first, this idea seems perfectly correct.

After all, our objective is to maximize profit.

So,

why not simply select the most profitable item first?

Let us understand this through an example.

---

## Example

Suppose we have the following items.

| Item | Profit | Weight |
|------|-------:|-------:|




```text

```

units of weight.

Therefore, this item cannot even fit inside the knapsack.

Even if we take only a fraction, we still need to compare whether another combination can generate more profit for every unit of weight.

Simply looking at the total profit is not enough.

---

## Important Observation

A large profit does **not** necessarily mean that the item is the best choice.

Sometimes, a very profitable item may also be extremely heavy.

Such an item occupies a large portion of the knapsack, leaving very little space for other items.

Therefore, profit alone cannot determine the best item.

# 9. Why Can't We Simply Choose the Lightest Item?

Some students think of another strategy.

> **Why don't we always select the lightest item first?**

Again,

| Item | Profit | Weight |
|------|-------:|-------:|

```text

```


On the other hand,

Item


Therefore, choosing items based only on their weight also does **not** produce the optimal solution.

---

## Important Observation

Weight alone is also not sufficient.

A lighter item may generate very little profit, whereas a heavier item may generate much larger profit.

Therefore, we need a criterion that considers **both** profit and weight together.

---

# 10. The Main Idea Behind the Fractional Knapsack Algorithm

Until now,

we have seen that

- Random selection fails.
- Selecting the highest-profit item fails.
- Selecting the lightest item also fails.

Now the question becomes,

> **What should we compare?**

The answer is

```text
Profit Per Unit Weight
```

which is also known as

```text
Profit / Weight Ratio
```

Instead of asking

> "Which item gives the highest total profit?"

the algorithm asks

> **"Which item gives the highest profit for every one unit of weight?"**

This is the greedy decision made by the Fractional Knapsack Algorithm.

---

# 11. Understanding Profit Per Unit Weight

The Profit/Weight Ratio tells us how much profit we earn for every one unit of weight occupied inside the knapsack.

The formula is

```text
Profit Per Unit Weight

=

Profit

/

Weight
```

Now,

let us calculate the ratio for every item.

| Item | Profit | Weight | Profit/Weight |
|------|-------:|-------:|--------------:|
| I1 | 60 | 10 | 6 |
| I2 | 100 | 20 | 5 |
| I3 | 120 | 30 | 4 |

Now observe the result carefully.

Although

```text
I3
```

has the largest total profit, its Profit/Weight Ratio is actually the smallest.

On the other hand,

```text
I1
```

generates

```text
6
```

units of profit

for every

```text
1
```

unit of weight.

Therefore,

Item

```text
I1
```

is actually the most valuable item.

---

# 12. Why Do We Sort According to Profit/Weight Ratio?

After calculating the ratio, the next step of the algorithm is to sort all the items in **descending order** of Profit/Weight Ratio.

Our original input is

| Item | Profit | Weight | Ratio |
|------|-------:|-------:|------:|
| I1 | 60 | 10 | 6 |
| I2 | 100 | 20 | 5 |
| I3 | 120 | 30 | 4 |

After sorting,

the order becomes

```text
Highest Ratio

↓

Lowest Ratio
```

```text
I1

↓

I2

↓

I3
```

Notice that, the algorithm no longer cares about

- highest profit,

or

- lowest weight.

Instead,

it gives priority to the item that provides the **maximum profit for every unit of weight**.

This greedy decision ensures that every kilogram of the knapsack is utilized in the most efficient manner.

---

## Important Observations

While studying the Fractional Knapsack Algorithm, remember the following points.

- Random selection does not guarantee maximum profit.
- Selecting the highest-profit item is not always correct.
- Selecting the lightest item is also not always correct.
- The correct greedy choice is based on the **Profit/Weight Ratio**.
- Items are sorted in **descending order** of this ratio.
- The algorithm always considers the item having the highest ratio first.

# Why Does the Algorithm Stop After Taking a Fraction?

Many beginners ask,

> **Why don't we continue checking the remaining items after taking a fraction?**

The answer is very simple.

After taking the required fraction, the knapsack becomes completely full.

The remaining capacity becomes

```text
0
```

Once there is no space left inside the knapsack, it is impossible to place any additional item.

Therefore, the algorithm immediately terminates.

---

# Important Observations

After completing the dry run, remember the following points.

- The algorithm first calculates the **Profit/Weight Ratio** for every item.
- The items are sorted in descending order of this ratio.
- Every item is checked one by one.
- If the complete item fits, it is selected completely.
- If the complete item does not fit, only the required fraction is selected.
- After taking a fraction of an item, the knapsack becomes full, and the algorithm stops.
- This greedy strategy guarantees the maximum possible profit for the Fractional Knapsack Problem.

# CPP code:

```cpp
#include <iostream>      // Used for input and output operations
#include <vector>        // Used to store all the items
#include <algorithm>     // Used for sorting the items
#include <iomanip>       // Used to print decimal values

using namespace std;

//------------------------------------------------------------
// Structure to represent one Item
//------------------------------------------------------------
struct Item
{
    int profit;      // Profit (Value) of the item
    int weight;      // Weight of the item
};

//------------------------------------------------------------
// Comparator Function
//
// Purpose:
// Sort all the items according to their
// Profit Per Unit Weight (Profit / Weight Ratio)
//
// Highest Ratio
//          ↓
// Lowest Ratio
//
// Why?
//
// Because Fractional Knapsack follows the
// Greedy Approach.
//
// At every step,
// we always select the item that gives the
// highest profit for every one unit of weight.
//------------------------------------------------------------
bool compare(Item a, Item b)
{
    double ratio1 = (double)a.profit / a.weight;
    double ratio2 = (double)b.profit / b.weight;

    return ratio1 > ratio2;
}

//------------------------------------------------------------
// Fractional Knapsack Function
//------------------------------------------------------------
double fractionalKnapsack(vector<Item> &items, int capacity)
{
    //--------------------------------------------------------
    // Step 1
    //
    // Sort all the items according to
    // Profit / Weight Ratio.
    //--------------------------------------------------------
    sort(items.begin(), items.end(), compare);

    //--------------------------------------------------------
    // Variable to store the total profit.
    //
    // Why double?
    //
    // Because while taking a fraction of an item,
    // the obtained profit may be in decimal.
    //--------------------------------------------------------
    double totalProfit = 0;

    //--------------------------------------------------------
    // Variable to store the current weight
    // already occupied inside the knapsack.
    //--------------------------------------------------------
    int currentWeight = 0;

    //--------------------------------------------------------
    // Traverse every item one by one.
    //
    // Since items are already sorted,
    // the item having the highest
    // Profit / Weight Ratio
    // will be processed first.
    //--------------------------------------------------------
    for (Item item : items)
    {
        //----------------------------------------------------
        // Check whether the complete item
        // can fit inside the remaining capacity.
        //----------------------------------------------------
        if (currentWeight + item.weight <= capacity)
        {
            //------------------------------------------------
            // Take the complete item.
            //------------------------------------------------
            currentWeight += item.weight;

            //------------------------------------------------
            // Add its complete profit.
            //------------------------------------------------
            totalProfit += item.profit;

            //------------------------------------------------
            // Display the selected item.
            //------------------------------------------------
            cout << "Complete Item Selected\n";
            cout << "Profit : " << item.profit << endl;
            cout << "Weight : " << item.weight << endl;
            cout << "Current Profit : "
                 << totalProfit << endl;

            cout << "Current Weight : "
                 << currentWeight << endl;

            cout << "--------------------------\n";
        }

        //----------------------------------------------------
        // Otherwise,
        // the complete item cannot fit.
        //
        // Since this is Fractional Knapsack,
        // take only the required fraction.
        //----------------------------------------------------
        else
        {
            //------------------------------------------------
            // Calculate the remaining capacity.
            //------------------------------------------------
            int remainingWeight =
                capacity - currentWeight;

            //------------------------------------------------
            // Calculate the fraction of the item
            // that can fit.
            //
            // Formula
            //
            // Remaining Capacity
            // -------------------
            // Item Weight
            //------------------------------------------------
            double fraction =
                (double)remainingWeight / item.weight;

            //------------------------------------------------
            // Profit obtained from the selected fraction.
            //
            // Formula
            //
            // Item Profit × Fraction
            //------------------------------------------------
            totalProfit +=
                item.profit * fraction;

            //------------------------------------------------
            // Knapsack becomes completely full.
            //------------------------------------------------
            currentWeight = capacity;

            //------------------------------------------------
            // Display the selected fraction.
            //------------------------------------------------
            cout << "Fraction of Item Selected\n";

            cout << "Original Profit : "
                 << item.profit << endl;

            cout << "Original Weight : "
                 << item.weight << endl;

            cout << "Fraction Taken : "
                 << fraction << endl;

            cout << "Current Profit : "
                 << totalProfit << endl;

            cout << "Current Weight : "
                 << currentWeight << endl;

            cout << "--------------------------\n";

            //------------------------------------------------
            // No capacity is left.
            //
            // Therefore,
            // stop the algorithm.
            //------------------------------------------------
            break;
        }
    }

    //--------------------------------------------------------
    // Return the maximum profit.
    //--------------------------------------------------------
    return totalProfit;
}

//------------------------------------------------------------
// Driver Function
//------------------------------------------------------------
int main()
{
    //--------------------------------------------------------
    // Input Items
    //
    // Profit
    // Weight
    //--------------------------------------------------------
    vector<Item> items =
    {
        {60,10},
        {100,20},
        {120,30}
    };

    //--------------------------------------------------------
    // Maximum Capacity of the Knapsack.
    //--------------------------------------------------------
    int capacity = 50;

    //--------------------------------------------------------
    // Call Fractional Knapsack Algorithm.
    //--------------------------------------------------------
    double maximumProfit =
        fractionalKnapsack(items, capacity);

    //--------------------------------------------------------
    // Display the final answer.
    //--------------------------------------------------------
    cout << fixed << setprecision(2);

    cout << "\nMaximum Profit : "
         << maximumProfit
         << endl;

    return 0;
}
```
