## Numerical Problems on Address Translation

### Problem Type 1: Basic Address Translation

**Problem:**
Given:
- Logical Address = 3500
- Page Size = 512 bytes
- Page 6 maps to Frame 9

Find: Physical Address

**Solution:**
```
Step 1: Extract Page Number and Offset
Page Number = 3500 / 512 = 6 (quotient)
Page Offset = 3500 % 512 = 436 (remainder)

Step 2: Get Frame Number from Page Table
Page 6 → Frame 9 (given)

Step 3: Calculate Physical Address
Physical Address = (Frame Number × Page Size) + Offset
                = (9 × 512) + 436
                = 4608 + 436
                = 5044
```

### Problem Type 2: Address Space Calculation

**Problem:**
Given:
- Logical Address = 32 bits
- Page Size = 4 KB
- Physical Memory = 64 MB

Find:
a) Number of pages
b) Number of frames
c) Size of page table

**Solution:**
```
Step 1: Calculate Logical Address Space
Logical Address Space = 2^32 bytes = 4 GB

Step 2: Number of Pages
Page Size = 4 KB = 2^12 bytes
Number of Pages = 4 GB / 4 KB = 2^32 / 2^12 = 2^20 = 1,048,576 pages

Step 3: Number of Frames
Physical Memory = 64 MB = 2^26 bytes
Number of Frames = 64 MB / 4 KB = 2^26 / 2^12 = 2^14 = 16,384 frames

Step 4: Size of Page Table
Each entry needs to store frame number = 14 bits (to address 2^14 frames)
Assume each entry = 4 bytes (including control bits)
Page Table Size = Number of Pages × Entry Size
                = 2^20 × 4 bytes
                = 4 MB
```

### Problem Type 3: Page Number and Offset Bits

**Problem:**
Given:
- Logical Address Space = 2^16 bytes
- Page Size = 2^10 bytes

Find: Bits for page number and offset

**Solution:**
```
Step 1: Offset Bits
Page Size = 2^10 bytes
Offset Bits = 10 bits

Step 2: Page Number Bits
Logical Address = 16 bits total
Page Number Bits = 16 - 10 = 6 bits

Step 3: Number of Pages
Number of Pages = 2^6 = 64 pages

Logical Address Structure:
┌──────────────┬──────────────┐
│ Page Number  │ Page Offset  │
│   6 bits     │   10 bits    │
└──────────────┴──────────────┘
```

### Problem Type 4: Effective Access Time (EAT)

**Problem:**
Given:
- TLB Hit Ratio = 85%
- TLB access time = 10 ns
- Memory access time = 80 ns

Find: Effective Access Time

**Solution:**
```
Method 1: Using Formula
EAT = α(ε + m) + (1-α)(ε + 2m)

Where:
α = 0.85 (hit ratio)
ε = 10 ns (TLB time)
m = 80 ns (memory time)

EAT = 0.85(10 + 80) + 0.15(10 + 80 + 80)
    = 0.85(90) + 0.15(170)
    = 76.5 + 25.5
    = 102 ns

Method 2: Step by Step
TLB Hit: 85% of time → 10 + 80 = 90 ns
TLB Miss: 15% of time → 10 + 80 + 80 = 170 ns

EAT = (0.85 × 90) + (0.15 × 170)
    = 76.5 + 25.5
    = 102 ns
```

### Problem Type 5: Multi-Level Paging

**Problem:**
Given:
- Logical Address = 32 bits
- Page Size = 4 KB
- Two-level paging
- Outer page table has 1024 entries

Find: Structure of logical address

**Solution:**
```
Step 1: Offset Bits
Page Size = 4 KB = 2^12 bytes
Offset = 12 bits

Step 2: Remaining Bits for Page Numbers
Total bits = 32
Remaining = 32 - 12 = 20 bits for page numbers

Step 3: Outer Page Table
1024 entries = 2^10 entries
Outer page bits = 10 bits

Step 4: Inner Page Table
Inner page bits = 20 - 10 = 10 bits

Logical Address Structure:
┌──────────┬──────────┬──────────┐
│  Outer   │  Inner   │  Offset  │
│ 10 bits  │ 10 bits  │ 12 bits  │
└──────────┴──────────┴──────────┘
```

### Problem Type 6: Internal Fragmentation

**Problem:**
Given:
- Process size = 72,766 bytes
- Page Size = 2,048 bytes

Find: Internal fragmentation

**Solution:**
```
Step 1: Number of Pages Needed
Number of Pages = ⌈72,766 / 2,048⌉ = ⌈35.53⌉ = 36 pages

Step 2: Total Memory Allocated
Total Allocated = 36 × 2,048 = 73,728 bytes

Step 3: Internal Fragmentation
Internal Fragmentation = Allocated - Used
                      = 73,728 - 72,766
                      = 962 bytes

Alternative Method:
Last page usage = 72,766 % 2,048 = 1,086 bytes
Internal Fragmentation = 2,048 - 1,086 = 962 bytes
```

### Problem Type 7: Page Table Size

**Problem:**
Given:
- Logical Address Space = 4 GB
- Physical Memory = 512 MB
- Page Size = 8 KB
- Each page table entry = 4 bytes

Find: Size of page table

**Solution:**
```
Step 1: Number of Pages
Logical Space = 4 GB = 2^32 bytes
Page Size = 8 KB = 2^13 bytes
Number of Pages = 2^32 / 2^13 = 2^19 = 524,288 pages

Step 2: Page Table Size
Page Table Size = Number of Pages × Entry Size
                = 524,288 × 4 bytes
                = 2,097,152 bytes
                = 2 MB
```

### Problem Type 8: Complete Address Translation

**Problem:**
Given:
- Logical Address Space = 64 pages
- Physical Memory = 32 frames
- Page Size = 1 KB
- Logical Address = 12,345

Page Table:
```
Page 0 → Frame 5
Page 1 → Frame 2
...
Page 12 → Frame 28
```

Find: Physical address for logical address 12,345

**Solution:**
```
Step 1: Calculate Page Number and Offset
Page Size = 1 KB = 1024 bytes
Page Number = 12,345 / 1024 = 12
Page Offset = 12,345 % 1024 = 57

Step 2: Look up Page Table
Page 12 → Frame 28

Step 3: Calculate Physical Address
Physical Address = (Frame Number × Page Size) + Offset
                = (28 × 1024) + 57
                = 28,672 + 57
                = 28,729
```
