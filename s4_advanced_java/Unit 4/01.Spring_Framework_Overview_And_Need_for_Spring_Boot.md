# Unit 4 — Introduction to Spring Boot Framework
## Spring Framework Overview & Need for Spring Boot

---

## 1. Quick Recap — What You Already Know

Before Spring, you built Java web apps using:

| Technology | What it does |
|---|---|
| **JDBC** | Connect to database, run SQL queries |
| **Servlet** | Handle HTTP requests (like GET, POST) |
| **JSP** | Show dynamic HTML pages to the user |

A typical flow you already know:

```
User → Browser → Servlet → JDBC → Database
                    ↓
                   JSP (response shown to user)
```

This worked. But it had **many problems** — and that's exactly why Spring Framework was created.

---

## 2. Problems With the Old Way (Servlet + JSP + JDBC)

You may have already felt these problems while coding:

### Problem 1 — Too Much Boilerplate Code
Every time you wrote JDBC code, you repeated the same lines:

```java
// You wrote this again and again in EVERY servlet
Class.forName("com.mysql.jdbc.Driver");
Connection con = DriverManager.getConnection(url, user, pass);
PreparedStatement ps = con.prepareStatement("SELECT * FROM students");
ResultSet rs = ps.executeQuery();
// ... close connection, handle exceptions ...
```

Same code. Every. Single. Time.

---

### Problem 2 — Tight Coupling
In your Servlet, you directly created objects of other classes:

```java
public class StudentServlet extends HttpServlet {
    StudentDAO dao = new StudentDAO(); // directly creating object
}
```

If tomorrow you want to replace `StudentDAO` with `StudentDAOv2`, you have to **go inside every Servlet and change the code**. This is called **tight coupling** — classes are too dependent on each other.

---

### Problem 3 — Manual web.xml Configuration
Every Servlet you wrote had to be registered in `web.xml`:

```xml
<servlet>
    <servlet-name>StudentServlet</servlet-name>
    <servlet-class>com.example.StudentServlet</servlet-class>
</servlet>
<servlet-mapping>
    <servlet-name>StudentServlet</servlet-name>
    <url-pattern>/student</url-pattern>
</servlet-mapping>
```

For 10 servlets → 10 blocks of XML. Very tedious and error-prone.

---

### Problem 4 — No Built-in Dependency Management
You had to manually download JAR files (Servlet API, MySQL connector, etc.) and add them to your project. Getting the **right versions** that work together was a headache.

---

### Problem 5 — External Server Required
To run your Servlet/JSP app, you needed to:
1. Install **Apache Tomcat** separately
2. Build a **WAR file**
3. Deploy the WAR to Tomcat
4. Start Tomcat manually

This was a lot of steps just to test your app.

---

## 3. What is Spring Framework?

Spring Framework is an **open-source Java framework** created by **Rod Johnson** in **2003** to fix exactly the problems listed above.

**Simple Definition:**
> Spring Framework is a collection of tools and libraries that makes building Java applications easier by handling repetitive tasks like object creation, database connection, and configuration automatically.

### What Spring Gave Us:

| Problem (Old Way) | Spring's Solution |
|---|---|
| Repeated JDBC boilerplate | Spring JDBC Template handles it |
| Tight coupling | Dependency Injection (DI) |
| Manual web.xml config | Annotation-based configuration |
| Manual object creation | IoC Container manages objects |
| Complex setup | Modular, pick only what you need |

> **Note:** DI and IoC are covered in detail on **Day 3**. Today just understand what problems they solve.

---

## 4. Spring Framework — Core Modules (Know for Exam)

Spring is not one thing — it is a collection of modules. You pick what you need:

| Module | What it does |
|---|---|
| **Spring Core** | IoC Container + Dependency Injection (most fundamental) |
| **Spring MVC** | Replaces Servlets for handling web requests |
| **Spring JDBC** | Replaces boilerplate JDBC code |
| **Spring Data JPA** | Even easier database access using JPA/Hibernate |
| **Spring Security** | Login, authentication, authorization |
| **Spring Boot** | Makes all of the above easy to set up |

> **Exam Tip:** Be ready to list 4–5 Spring modules with one-line descriptions.

---

## 5. Spring MVC vs Your Old Servlet Approach

You used `HttpServlet` to handle requests. Spring MVC replaces that with a much cleaner approach:

**Old way — Servlet:**
```java
@WebServlet("/student")
public class StudentServlet extends HttpServlet {
    protected void doGet(HttpServletRequest req, HttpServletResponse res)
        throws ServletException, IOException {
        res.getWriter().print("Hello " + req.getParameter("name"));
    }
}
```

**Spring MVC way:**
```java
@RestController
public class StudentController {
    @GetMapping("/student")
    public String getStudent(@RequestParam String name) {
        return "Hello " + name;
    }
}
```

Much shorter, much cleaner. Spring handles all the `HttpServletRequest` / `HttpServletResponse` boilerplate for you.

---

## 6. But Spring Framework Still Had Problems...

Even with Spring Framework, setting up a project was complex:

- Had to configure `DispatcherServlet` manually in `web.xml`
- Had to download and manage dependency JARs carefully
- Still needed to deploy to an **external Tomcat server**
- A lot of XML configuration was needed (`applicationContext.xml`, etc.)
- Getting the right **version** of every Spring library was difficult

This is where **Spring Boot** came in.

---

## 7. What is Spring Boot?

Spring Boot is a **project built on top of Spring Framework** that removes all the complex setup and configuration.

- Released by **Pivotal Team** in **2014**
- Built on top of Spring Framework (everything Spring can do, Spring Boot can do — faster)
- Goal: Get a working app running in minutes, not hours

**Simple Definition for Exam:**
> Spring Boot is an extension of Spring Framework that provides auto-configuration, embedded server, and starter dependencies to build stand-alone Spring applications with minimal setup.

---

## 8. Key Features of Spring Boot (Most Important for Exam)

### 8.1 Auto Configuration 
Spring Boot **automatically configures** your application based on what libraries are present.

Example: If Spring Boot sees `spring-boot-starter-web` in your project, it automatically:
- Sets up `DispatcherServlet` (which you had to do manually before)
- Configures error pages
- Starts embedded Tomcat

You don't write a single line of XML for this.

---

### 8.2 Embedded Server 
Spring Boot comes with **Tomcat built-in**. You don't install Tomcat separately.

**Old way (Servlet/JSP):**
```
1. Install Tomcat separately
2. Build WAR file
3. Copy WAR to Tomcat's webapps folder
4. Start Tomcat
5. Open browser
```

**Spring Boot way:**
```
1. Run the main() method
2. Open browser 
```

The app starts on its own. No separate server needed.

---

### 8.3 Spring Boot Starters 
Instead of manually downloading individual JARs, Spring Boot gives you **starter packs** — one dependency that includes everything needed for a feature.

| Starter | What's Included |
|---|---|
| `spring-boot-starter-web` | Spring MVC + Embedded Tomcat + Jackson (JSON) |
| `spring-boot-starter-data-jpa` | Hibernate + JPA + Spring Data |
| `spring-boot-starter-security` | Spring Security |
| `spring-boot-starter-test` | JUnit + Mockito |
| `spring-boot-starter-jdbc` | Spring JDBC + HikariCP connection pool |

Compare this to your old approach where you downloaded `servlet-api.jar`, `mysql-connector.jar`, etc. one by one and hoped they were compatible.

---

### 8.4 No XML Configuration 
Everything is done using **annotations** and one `application.properties` file.

**Old Spring XML config:**
```xml
<bean id="dataSource" class="org.springframework.jdbc.datasource.DriverManagerDataSource">
    <property name="driverClassName" value="com.mysql.jdbc.Driver"/>
    <property name="url" value="jdbc:mysql://localhost/mydb"/>
    <property name="username" value="root"/>
    <property name="password" value="1234"/>
</bean>
```

**Spring Boot — application.properties:**
```properties
spring.datasource.url=jdbc:mysql://localhost/mydb
spring.datasource.username=root
spring.datasource.password=1234
```

That's it. Spring Boot reads this and sets up the entire database connection automatically.

---

### 8.5 Production-Ready Features (Actuator)
Spring Boot includes **Spring Actuator** which gives built-in health checks, metrics, and environment info via endpoints like `/actuator/health`. Useful when your app is live.

---

## 9. Spring Boot Architecture — Mapped to What You Know

```
                     [ Browser / Client ]
                              |
                              ▼
           [ Embedded Tomcat — built into Spring Boot ]
           (you used to install this separately)
                              |
                              ▼
                   [ DispatcherServlet ]
           (auto-configured — replaces your web.xml)
                              |
                              ▼
                   [ @RestController ]
           (replaces your HttpServlet class)
                              |
                              ▼
                   [ @Service — Business Logic ]
                              |
                              ▼
                   [ @Repository — Database Layer ]
           (replaces your DAO class + JDBC boilerplate)
                              |
                              ▼
                        [ Database ]
```

**Easy way to remember the mapping:**

| What you used | Spring Boot equivalent |
|---|---|
| `HttpServlet` class | `@RestController` |
| `web.xml` mappings | `@GetMapping`, `@PostMapping` annotations |
| DAO class + JDBC code | `@Repository` + Spring Data JPA |
| JDBC `getConnection()` | `application.properties` datasource config |
| External Tomcat | Embedded Tomcat (auto-started) |

---

## 10. Comparison Table — Old Stack vs Spring Boot

> **This comparison table is very frequently asked in exams.**

| Feature | Servlet + JSP + JDBC | Spring Boot |
|---|---|---|
| Server | External Tomcat required | Embedded Tomcat (built-in) |
| Configuration | web.xml + XML files | application.properties only |
| Database code | Manual JDBC boilerplate | Auto-configured via starter |
| Request handling | HttpServlet class | @RestController annotation |
| Dependency management | Manual JAR downloads | Starter dependencies |
| Object management | Manual (`new` keyword) | IoC Container (automatic) |
| Deployment | WAR file on Tomcat | JAR file (self-contained) |
| Code length | Long and repetitive | Short and clean |
| Testing | Harder due to coupling | Easier |

---

## 11. Advantages of Spring Boot

1. **Faster development** — No boilerplate, minimal configuration
2. **Embedded server** — No separate Tomcat installation needed
3. **Auto configuration** — Smart defaults, works out of the box
4. **Starter dependencies** — No version conflict issues
5. **No XML configuration** — Annotations + properties file is enough
6. **Great for REST APIs** — Industry standard for modern web development
7. **Microservice-friendly** — Widely used for microservices architecture

---

## 12. Limitations of Spring Boot

1. Overkill for very small or simple applications
2. Auto-configuration can be difficult to debug when something goes wrong
3. JAR size is large because it bundles many libraries

> **Exam Tip:** Exams sometimes ask for "limitations" — always keep 2–3 points ready.

## 13. Questions

**Q1. What are the problems with traditional Servlet-based web development?**

**Ans:** The traditional Servlet + JSP + JDBC approach had the following problems:
1. Repeated boilerplate code, especially in JDBC (open connection, execute query, close connection — every time)
2. Tight coupling between classes (Servlet directly creating DAO objects using `new`)
3. Manual `web.xml` configuration for every Servlet
4. External Tomcat server installation and WAR deployment required
5. Manual JAR management with frequent version conflicts

---

**Q2. What is Spring Framework? Why was it introduced?**

**Ans:** Spring Framework is an open-source, lightweight Java framework introduced by Rod Johnson in 2003. It was introduced to solve the problems of traditional Java EE and Servlet-based development such as tight coupling, boilerplate code, and complex configuration. Spring introduced concepts like Dependency Injection (DI) and Inversion of Control (IoC) to make Java applications easier to develop and test.

---

**Q3. What is Spring Boot? How is it different from Spring Framework?**

**Ans:** Spring Boot is a project built on top of Spring Framework that provides auto-configuration, embedded server support, and starter dependencies to build stand-alone applications quickly. While Spring Framework reduced complexity compared to plain Servlets, it still required manual XML configuration and an external server. Spring Boot removes all of this.

| Feature | Spring Framework | Spring Boot |
|---|---|---|
| Configuration | Manual XML | Auto Configuration |
| Server | External Tomcat | Embedded Tomcat |
| Setup time | Slow | Very fast |

---

**Q4. What is Auto Configuration in Spring Boot? Give an example.**

**Ans:** Auto Configuration is a Spring Boot feature where it automatically detects the libraries (JARs/starters) present in the project and configures the required beans without any manual setup.

Example: If `spring-boot-starter-web` is added, Spring Boot automatically configures `DispatcherServlet`, sets up embedded Tomcat on port 8080, and enables JSON support — without any XML or Java configuration code.

---

**Q5. What are Spring Boot Starters? List any four.**

**Ans:** Spring Boot Starters are pre-packaged groups of dependencies that include all libraries required for a specific feature.

| Starter | Purpose |
|---|---|
| `spring-boot-starter-web` | Build REST APIs and web applications |
| `spring-boot-starter-data-jpa` | Database access using JPA and Hibernate |
| `spring-boot-starter-security` | Authentication and authorization |
| `spring-boot-starter-test` | Unit testing with JUnit and Mockito |

---

**Q6. Compare Servlet-based development with Spring Boot. (Any 5 points)**

**Ans:**

| Feature | Servlet + JSP | Spring Boot |
|---|---|---|
| Server | External Tomcat | Embedded Tomcat |
| Config file | web.xml | application.properties |
| Request handling | HttpServlet class | @RestController |
| DB connection | Manual JDBC code | Auto-configured |
| Deployment | WAR file | Executable JAR file |
