# API Rate Limiter + Analytics System

## Overview

This project is a backend-heavy system built using: - Next.js
(Frontend) - Spring Boot (Backend) - MySQL (Database)

It provides: - API rate limiting per user - Request logging - Usage
analytics dashboard - CRUD operations for users and rate limit rules

------------------------------------------------------------------------

## Architecture

Frontend (Next.js) → Backend (Spring Boot) → Database (MySQL)

Flow: Request → Filter → Controller → Service → Repository → DB

## Core Logic

### Rate Limiting Algorithm (Fixed Window)

-   Count requests in last time window
-   If count \>= limit → block request (429)
-   Else → allow

------------------------------------------------------------------------

## RateLimiterFilter Flow

1.  Extract API key
2.  Identify user
3.  Fetch rate limit rule
4.  Count requests in DB
5.  Compare with limit
6.  Allow or block request
7.  Log request
