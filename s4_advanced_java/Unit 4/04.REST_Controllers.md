## 2. What is REST?

**REST = Representational State Transfer**

REST is a way to build APIs (Application Programming Interfaces) that can be used by:
- Mobile apps
- Web browsers
- Other servers
- Any client

In REST, everything is treated as a **Resource** (Student, Product, Order, etc.) and you perform operations on it using **HTTP Methods**.

| HTTP Method | Operation | Real Life Example |
|---|---|---|
| **GET** | Read / Fetch data | View student list |
| **POST** | Create new data | Add new student |
| **PUT** | Update existing data | Edit student details |
| **DELETE** | Delete data | Remove student |

> **Exam Tip:** REST uses HTTP methods to perform CRUD operations.
> GET = Read, POST = Create, PUT = Update, DELETE = Delete

## 3. Your Old Way vs Spring Boot Way

You used to handle HTTP requests using `HttpServlet`:

**Old Servlet way:**
```java
@WebServlet("/student")
public class StudentServlet extends HttpServlet {

    protected void doGet(HttpServletRequest req, HttpServletResponse res)
            throws ServletException, IOException {
        // manually read request
        // manually write response
        // manually set content type
        res.setContentType("application/json");
        res.getWriter().print("{\"name\":\"Ravi\"}");
    }

    protected void doPost(HttpServletRequest req, HttpServletResponse res)
            throws ServletException, IOException {
        String name = req.getParameter("name");
        // manually read, manually process, manually respond
    }
}
```

**Spring Boot way:**
```java
@RestController
public class StudentController {

    @GetMapping("/student")
    public Student getStudent() {
        return new Student(1, "Ravi", "ravi@gmail.com");
        // Spring automatically converts to JSON!
    }

    @PostMapping("/student")
    public String addStudent(@RequestBody Student student) {
        return "Added: " + student.getName();
    }
}
```

Spring handles ALL the boilerplate — content type, JSON conversion, request reading, response writing — everything automatic.

## 4. @RestController — The Main Annotation

```java
@RestController
public class StudentController {
    // your methods here
}
```

`@RestController` is a combination of **2 annotations**:

| Annotation | What it does |
|---|---|
| `@Controller` | Marks class as Spring Bean + handles HTTP requests |
| `@ResponseBody` | Return value is directly written to HTTP response (as JSON/text) |

**Without `@ResponseBody`** — Spring tries to find a JSP/HTML page with that name.
**With `@ResponseBody`** — Spring directly sends the return value as response.

Since REST APIs always return data (not pages), we always use `@RestController`.

## 5. @RequestMapping — Base URL for Controller

You can give a **common base URL** to all methods in a controller:

```java
@RestController
@RequestMapping("/api/students")   // all URLs will start with /api/students
public class StudentController {

    @GetMapping("")           // → /api/students
    public String getAll() { return "all students"; }

    @GetMapping("/{id}")      // → /api/students/1
    public String getById() { return "one student"; }
}
```

This avoids repeating `/api/students` in every method.

## 6. All HTTP Method Annotations

| Annotation | HTTP Method | Used For |
|---|---|---|
| `@GetMapping` | GET | Fetch / Read data |
| `@PostMapping` | POST | Create new data |
| `@PutMapping` | PUT | Update existing data |
| `@DeleteMapping` | DELETE | Delete data |
| `@PatchMapping` | PATCH | Partial update |

These are all **shortcuts** for `@RequestMapping(method = RequestMethod.GET)` etc.

## 7. How to Read Data from Request — 4 Ways

### 7.1 @PathVariable — Read from URL path

Use when the value is **part of the URL**.

```java
// URL: http://localhost:8080/students/3
@GetMapping("/students/{id}")
public String getStudent(@PathVariable int id) {
    return "Student ID: " + id;
}
```

`{id}` in the URL → `@PathVariable int id` in the method.

**Multiple path variables:**
```java
// URL: http://localhost:8080/students/3/courses/Java
@GetMapping("/students/{studentId}/courses/{courseName}")
public String getStudentCourse(
        @PathVariable int studentId,
        @PathVariable String courseName) {
    return "Student " + studentId + " - Course: " + courseName;
}
```

---

### 7.2 @RequestParam — Read from URL query string

Use when the value is sent as `?key=value` in the URL.

```java
// URL: http://localhost:8080/students?course=Java
@GetMapping("/students")
public String getStudentsByCourse(@RequestParam String course) {
    return "Course: " + course;
}
```

**With default value** (if param not sent):
```java
// URL: http://localhost:8080/students         → page=1 (default)
// URL: http://localhost:8080/students?page=3  → page=3
@GetMapping("/students")
public String getStudents(@RequestParam(defaultValue = "1") int page) {
    return "Page: " + page;
}
```

**Optional param:**
```java
@GetMapping("/students")
public String search(@RequestParam(required = false) String name) {
    if (name == null) return "All students";
    return "Searching: " + name;
}
```

---

### 7.3 @RequestBody — Read JSON body from POST/PUT

Use when data is sent in the **request body** as JSON (from mobile app / Postman).

```java
@PostMapping("/students")
public String addStudent(@RequestBody Student student) {
    return "Added: " + student.getName();
}
```

Client sends this JSON:
```json
{
    "id": 5,
    "name": "Rahul",
    "email": "rahul@gmail.com"
}
```

Spring **automatically converts JSON → Student object**. This is called **Deserialization**.

> **Old way (Servlet):** You had to manually read `request.getParameter()` for each field.
> **Spring way:** Just write `@RequestBody Student student` — Spring handles the rest.

---

### 7.4 @RequestHeader — Read HTTP headers (Less common)

```java
@GetMapping("/students")
public String getStudents(@RequestHeader("Authorization") String token) {
    return "Token received: " + token;
}
```

## 8. ResponseEntity — Control Your HTTP Response

By default, Spring returns **HTTP 200 OK** for every successful response.

But sometimes you need to return:
- `201 Created` when a new student is added
- `404 Not Found` when student doesn't exist
- `400 Bad Request` when input is wrong

Use `ResponseEntity` for this:

```java
@GetMapping("/students/{id}")
public ResponseEntity<Student> getStudent(@PathVariable int id) {

    Student student = studentService.getById(id);

    if (student == null) {
        return ResponseEntity.status(404).build();      // 404 Not Found
    }

    return ResponseEntity.ok(student);                  // 200 OK + student data
}
```

### Common ResponseEntity Methods

| Method | HTTP Status |
|---|---|
| `ResponseEntity.ok(data)` | 200 OK |
| `ResponseEntity.created(uri).body(data)` | 201 Created |
| `ResponseEntity.noContent().build()` | 204 No Content |
| `ResponseEntity.notFound().build()` | 404 Not Found |
| `ResponseEntity.badRequest().body(msg)` | 400 Bad Request |
| `ResponseEntity.status(500).body(msg)` | 500 Internal Server Error |

### Student.java (Model)

```java
package com.college.studentapp.model;

public class Student {

    private int id;
    private String name;
    private String email;
    private String course;

    // Default constructor (needed for @RequestBody JSON conversion)
    public Student() {}

    // Parameterized constructor
    public Student(int id, String name, String email, String course) {
        this.id     = id;
        this.name   = name;
        this.email  = email;
        this.course = course;
    }

    // Getters and Setters (needed for JSON conversion)
    public int    getId()              { return id; }
    public void   setId(int id)        { this.id = id; }

    public String getName()            { return name; }
    public void   setName(String name) { this.name = name; }

    public String getEmail()               { return email; }
    public void   setEmail(String email)   { this.email = email; }

    public String getCourse()              { return course; }
    public void   setCourse(String course) { this.course = course; }
}
```

> **Important:** `@RequestBody` needs a **default constructor + setters** to convert JSON to object.
> `@GetMapping` return needs **getters** to convert object to JSON.

### StudentService.java (Service Layer)

```java
package com.college.studentapp.service;

import com.college.studentapp.model.Student;
import org.springframework.stereotype.Service;

import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;
import java.util.Optional;

@Service
public class StudentService {

    // In-memory list as fake database (no DB setup needed)
    private List<Student> students = new ArrayList<>(Arrays.asList(
        new Student(1, "Ravi Sharma",  "ravi@gmail.com",  "Java"),
        new Student(2, "Priya Patel",  "priya@gmail.com", "Python"),
        new Student(3, "Amit Verma",   "amit@gmail.com",  "Web Dev")
    ));

    // GET all students
    public List<Student> getAll() {
        return students;
    }

    // GET student by ID
    public Optional<Student> getById(int id) {
        return students.stream()
                .filter(s -> s.getId() == id)
                .findFirst();
    }

    // GET students by course
    public List<Student> getByCourse(String course) {
        List<Student> result = new ArrayList<>();
        for (Student s : students) {
            if (s.getCourse().equalsIgnoreCase(course)) {
                result.add(s);
            }
        }
        return result;
    }

    // POST — add new student
    public Student add(Student student) {
        students.add(student);
        return student;
    }

    // PUT — update existing student
    public Optional<Student> update(int id, Student updatedStudent) {
        for (Student s : students) {
            if (s.getId() == id) {
                s.setName(updatedStudent.getName());
                s.setEmail(updatedStudent.getEmail());
                s.setCourse(updatedStudent.getCourse());
                return Optional.of(s);
            }
        }
        return Optional.empty();  // student not found
    }

    // DELETE — remove student by ID
    public boolean delete(int id) {
        return students.removeIf(s -> s.getId() == id);
    }
}
```

### StudentController.java (Controller Layer) — MAIN FILE

```java
package com.college.studentapp.controller;

import com.college.studentapp.model.Student;
import com.college.studentapp.service.StudentService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

import java.util.List;
import java.util.Optional;

@RestController
@RequestMapping("/api/students")   // base URL for all endpoints
public class StudentController {

    private final StudentService studentService;

    @Autowired
    public StudentController(StudentService studentService) {
        this.studentService = studentService;
    }

    // ================================================
    // 1. GET ALL STUDENTS
    // URL    : GET http://localhost:8080/api/students
    // Returns: List of all students as JSON
    // ================================================
    @GetMapping
    public ResponseEntity<List<Student>> getAllStudents() {
        List<Student> students = studentService.getAll();
        return ResponseEntity.ok(students);   // 200 OK + JSON list
    }

    // ================================================
    // 2. GET STUDENT BY ID
    // URL    : GET http://localhost:8080/api/students/1
    // Returns: One student or 404
    // ================================================
    @GetMapping("/{id}")
    public ResponseEntity<Student> getStudentById(@PathVariable int id) {
        Optional<Student> student = studentService.getById(id);

        if (student.isPresent()) {
            return ResponseEntity.ok(student.get());       // 200 OK
        } else {
            return ResponseEntity.notFound().build();      // 404 Not Found
        }
    }

    // ================================================
    // 3. GET STUDENTS BY COURSE
    // URL    : GET http://localhost:8080/api/students?course=Java
    // Returns: Students in that course
    // ================================================
    @GetMapping("/search")
    public ResponseEntity<List<Student>> getStudentsByCourse(
            @RequestParam String course) {

        List<Student> students = studentService.getByCourse(course);
        return ResponseEntity.ok(students);
    }

    // ================================================
    // 4. POST — ADD NEW STUDENT
    // URL    : POST http://localhost:8080/api/students
    // Body   : JSON { "id":4, "name":"...", ... }
    // Returns: Added student + 201 Created
    // ================================================
    @PostMapping
    public ResponseEntity<Student> addStudent(@RequestBody Student student) {
        Student added = studentService.add(student);
        return ResponseEntity.status(201).body(added);    // 201 Created
    }

    // ================================================
    // 5. PUT — UPDATE EXISTING STUDENT
    // URL    : PUT http://localhost:8080/api/students/1
    // Body   : JSON { "name":"...", "email":"...", ... }
    // Returns: Updated student or 404
    // ================================================
    @PutMapping("/{id}")
    public ResponseEntity<Student> updateStudent(
            @PathVariable int id,
            @RequestBody Student updatedStudent) {

        Optional<Student> result = studentService.update(id, updatedStudent);

        if (result.isPresent()) {
            return ResponseEntity.ok(result.get());        // 200 OK
        } else {
            return ResponseEntity.notFound().build();      // 404 Not Found
        }
    }

    // ================================================
    // 6. DELETE — REMOVE STUDENT
    // URL    : DELETE http://localhost:8080/api/students/1
    // Returns: Success message or 404
    // ================================================
    @DeleteMapping("/{id}")
    public ResponseEntity<String> deleteStudent(@PathVariable int id) {
        boolean deleted = studentService.delete(id);

        if (deleted) {
            return ResponseEntity.ok("Student deleted successfully!");  // 200 OK
        } else {
            return ResponseEntity.notFound().build();                   // 404 Not Found
        }
    }
}
```

## 12. JSON Conversion — How Spring Does It Automatically

When your method returns a Java object, Spring automatically converts it to JSON.
When `@RequestBody` receives JSON, Spring automatically converts it to Java object.

This is done by a library called **Jackson** which comes bundled with `spring-boot-starter-web`.

```
Java Object → JSON   (called Serialization)   → happens on @GetMapping return
JSON → Java Object   (called Deserialization) → happens on @RequestBody
```

**Example:**
```java
// Java Object
Student s = new Student(1, "Ravi", "ravi@gmail.com", "Java");

// Spring converts to JSON automatically:
// {
//     "id": 1,
//     "name": "Ravi",
//     "email": "ravi@gmail.com",
//     "course": "Java"
// }
```

> **For JSON conversion to work, your model class needs:**
> - Default (no-arg) constructor
> - Getters (for serialization — object to JSON)
> - Setters (for deserialization — JSON to object)

## 13. @PathVariable vs @RequestParam — Difference

| Feature | @PathVariable | @RequestParam |
|---|---|---|
| Where in URL | Inside the path `/students/{id}` | After `?` in URL `/students?course=Java` |
| Example URL | `/students/3` | `/students?course=Java` |
| Used for | Identifying a specific resource | Filtering / searching |
| Required? | Always required | Can be optional |
| Example code | `@PathVariable int id` | `@RequestParam String course` |

---

## 14. @Controller vs @RestController — Difference

| Feature | @Controller | @RestController |
|---|---|---|
| Used for | Web apps that return HTML pages | REST APIs that return JSON/data |
| Returns | View name (JSP/Thymeleaf page) | Data directly (JSON, String, Object) |
| Needs @ResponseBody? | Yes, on every method | No — already included |
| Old equivalent | Servlet + JSP | Servlet that returns JSON |
| Use in | MVC web apps | REST APIs, mobile backends |

> **For REST APIs — always use `@RestController`**

## 15. HTTP Status Codes — Important for Exam

| Code | Name | When to use |
|---|---|---|
| **200** | OK | Successful GET, PUT, DELETE |
| **201** | Created | Successful POST (new resource created) |

| **404** | Not Found | Resource doesn't exist |

## 16. Questions

**Q1. What is @RestController? How is it different from @Controller?**

**Ans:** `@RestController` is a combination of `@Controller` and `@ResponseBody`. It is used to create REST APIs where the return value is directly written to the HTTP response as JSON or text. `@Controller` is used for web applications that return view names (like JSP pages). In REST API development, `@RestController` is always preferred since we return data, not pages.

---

**Q2. What is the difference between @PathVariable and @RequestParam?**

**Ans:**
- `@PathVariable` reads a value from the URL path itself (e.g., `/students/3` → `id = 3`). It is used to identify a specific resource.
- `@RequestParam` reads a value from the query string (e.g., `/students?course=Java` → `course = "Java"`). It is used for filtering or searching.

---

**Q3. What is @RequestBody? Why is it used?**

**Ans:** `@RequestBody` is used to read the JSON data sent in the HTTP request body and automatically convert it into a Java object. This conversion is done by Jackson library (included in `spring-boot-starter-web`). It is used in `@PostMapping` and `@PutMapping` where the client sends data in JSON format. The model class needs a default constructor and setters for this conversion to work.

---

**Q4. What is ResponseEntity in Spring Boot?**

**Ans:** `ResponseEntity` is a class in Spring Boot that allows you to control the complete HTTP response — including status code, headers, and body. By default, Spring returns 200 OK for every successful response. Using `ResponseEntity`, we can return appropriate status codes like 201 (Created), 404 (Not Found), 400 (Bad Request), etc. Example: `ResponseEntity.status(201).body(student)` returns 201 Created with student data.

---

**Q5. List all HTTP methods used in REST API and their purpose.**

**Ans:**

| HTTP Method | Spring Annotation | Purpose |
|---|---|---|
| GET | @GetMapping | Read/Fetch data |
| POST | @PostMapping | Create new resource |
| PUT | @PutMapping | Update existing resource |
| DELETE | @DeleteMapping | Delete a resource |
| PATCH | @PatchMapping | Partial update of resource |

---

**Q6. What is @RequestMapping? How is it used in Spring Boot?**

**Ans:** `@RequestMapping` is used to map HTTP requests to controller methods. It can be placed on the class (to set a base URL for all methods) or on individual methods. Spring provides shortcut annotations like `@GetMapping`, `@PostMapping`, etc. which are specialized versions of `@RequestMapping` for specific HTTP methods. Example: `@RequestMapping("/api/students")` on a class means all method URLs will start with `/api/students`.
