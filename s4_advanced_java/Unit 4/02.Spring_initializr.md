## 2. How Did You Create Projects Before?

When you built Servlet/JSP projects, you:
1. Created a **Dynamic Web Project** in Eclipse manually
2. Downloaded JAR files one by one (servlet-api.jar, mysql-connector.jar, etc.)
3. Set up `web.xml` manually
4. Configured Tomcat in your IDE manually

**With Spring Boot**, all of this is replaced by a single tool called **Spring Initializr**.

---

## 3. What is Spring Initializr?

Spring Initializr is a **web-based project generator** for Spring Boot applications.

- Website: **https://start.spring.io**
- You select your project options, click Generate, and download a ready-to-use project as a ZIP file
- No manual folder creation, no manual JAR downloading

**Think of it like this:**  
Just like you used Eclipse's "New Dynamic Web Project" wizard for Servlets, Spring Initializr is the wizard for Spring Boot — except much more powerful.

---

## 4. Spring Initializr — Step by Step

When you open **https://start.spring.io**, you see these options:

### Step 1 — Project Type
Choose between **Maven** or **Gradle** (build tool — explained in the next section).  
**Choose: Maven** (most common, easier for beginners)

### Step 2 — Language
Choose **Java**

### Step 3 — Spring Boot Version
Choose the latest **stable version** (avoid SNAPSHOT versions — those are under development)

### Step 4 — Project Metadata

| Field | What to enter | Example |
|---|---|---|
| **Group** | Your organization/package prefix | `com.college` |
| **Artifact** | Your project name | `studentapp` |
| **Name** | Display name | `studentapp` |
| **Description** | Short description | `Student Management App` |
| **Package Name** | Auto-filled from Group + Artifact | `com.college.studentapp` |
| **Packaging** | JAR or WAR | **JAR** (default for Spring Boot) |
| **Java Version** | Your installed Java version | `17` or `21` |

> **Exam Tip:** Know the difference between JAR and WAR packaging.  
> - **JAR** — Includes embedded Tomcat. Just run `java -jar`. Used in Spring Boot.  
> - **WAR** — Needs external Tomcat. Used in traditional Servlet/JSP apps.

### Step 5 — Add Dependencies
Click **"Add Dependencies"** and search for what you need.

For a basic web project, add:
- **Spring Web** → gives you `spring-boot-starter-web` (Spring MVC + embedded Tomcat)

### Step 6 — Generate
Click the **"Generate"** button → a ZIP file is downloaded.

### Step 7 — Import into IDE
- Unzip the downloaded file
- Open **IntelliJ IDEA** or **Eclipse**
- Import as **Maven Project** (File → Import → Existing Maven Projects)

## 5. Build Tools — Maven vs Gradle

When you did Servlet projects, you manually managed JAR files. Build tools automate this.

### What is a Build Tool?
A build tool is software that:
- Automatically **downloads dependencies** (JAR files) from the internet
- **Compiles** your code
- **Packages** your code into a JAR/WAR file
- **Runs tests** automatically

### Maven (Most Common — Focus Here)

Maven uses a file called **`pom.xml`** (Project Object Model) to manage everything.

You just write **what** you need — Maven figures out **how** to get it.

```xml
<!-- Instead of downloading mysql-connector.jar manually, you just write: -->
<dependency>
    <groupId>mysql</groupId>
    <artifactId>mysql-connector-java</artifactId>
    <version>8.0.33</version>
</dependency>
```

Maven downloads it automatically from the internet (from **Maven Central Repository**).

### Gradle

Gradle uses a file called **`build.gradle`** and uses a scripting syntax (Groovy or Kotlin).

```groovy
// Same dependency in Gradle
implementation 'mysql:mysql-connector-java:8.0.33'
```

### Maven vs Gradle Comparison

| Feature | Maven | Gradle |
|---|---|---|
| Config file | `pom.xml` (XML) | `build.gradle` (Groovy/Kotlin) |
| Syntax | XML — verbose but clear | Scripting — shorter but complex |
| Speed | Slower | Faster |
| Popularity | Very widely used | Growing, used in Android |
| For beginners | Easier | Slightly harder |
| Spring Boot default | Yes | Also supported |

> **For this unit: Focus on Maven and `pom.xml`**

## 6. Understanding pom.xml (Very Important)

`pom.xml` is the heart of a Maven project. When Spring Initializr generates your project, it creates this file automatically.

Here is a typical Spring Boot `pom.xml` with explanation:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project>

    <!-- 1. Maven coordinates of YOUR project -->
    <groupId>com.college</groupId>
    <artifactId>studentapp</artifactId>
    <version>0.0.1-SNAPSHOT</version>
    <packaging>jar</packaging>

    <!-- 2. Spring Boot Parent — gives default configs for all Spring Boot apps -->
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>3.2.0</version>
    </parent>

    <!-- 3. Java version -->
    <properties>
        <java.version>17</java.version>
    </properties>

    <!-- 4. Dependencies — libraries your project needs -->
    <dependencies>

        <!-- Spring Web — for REST controllers, embedded Tomcat -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>

        <!-- Spring Boot Test — for unit testing -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-test</artifactId>
            <scope>test</scope>
        </dependency>

    </dependencies>

    <!-- 5. Build plugin — packages app as executable JAR -->
    <build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>

</project>
```

### Key Sections of pom.xml Explained

| Section | Purpose |
|---|---|
| `<groupId>` | Reverse domain of your organization |
| `<artifactId>` | Your project/module name |
| `<version>` | Version of your project |
| `<packaging>` | jar or war |
| `<parent>` | Inherits Spring Boot default configuration |
| `<dependencies>` | List of all libraries your project needs |
| `<build>` + plugin | Packages the project as a runnable JAR |

> **Exam Tip:** `spring-boot-starter-parent` is very important — it manages versions of all Spring libraries automatically so you don't get version conflicts.

## 7. Spring Boot Project Structure

When you unzip the generated project and open it, the folder structure looks like this:

```
studentapp/
│
├── src/
│   ├── main/
│   │   ├── java/
│   │   │   └── com/
│   │   │       └── college/
│   │   │           └── studentapp/
│   │   │               └── StudentappApplication.java  ← MAIN CLASS
│   │   │
│   │   └── resources/
│   │       ├── application.properties   ← CONFIG FILE
│   │       ├── static/                  ← CSS, JS, images
│   │       └── templates/               ← HTML templates (if using Thymeleaf)
│   │
│   └── test/
│       └── java/
│           └── com/college/studentapp/
│               └── StudentappApplicationTests.java  ← TEST CLASS
│
├── pom.xml                              ← MAVEN CONFIG
└── mvnw / mvnw.cmd                      ← Maven wrapper scripts
```

### Understanding Each Folder

| Folder / File | Purpose | Old equivalent |
|---|---|---|
| `src/main/java` | All your Java code goes here | `src` folder in Eclipse |
| `src/main/resources` | Config files, static files | `WebContent` in Servlet project |
| `application.properties` | App configuration (port, DB, etc.) | JDBC connection code + `web.xml` |
| `src/test/java` | Test classes | — |
| `pom.xml` | Dependency management | Manual JAR files |
| `static/` | CSS, JavaScript, images | `WebContent/` folder |
| `templates/` | HTML templates (Thymeleaf) | JSP files |

> **Exam Tip:** Know what goes in each folder. A common exam question is "Explain the folder structure of a Spring Boot project."

## 8. The Main Class — @SpringBootApplication

When you open the generated project, you'll find one Java file already created:

```java
package com.college.studentapp;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication                          // ← Most important annotation
public class StudentappApplication {

    public static void main(String[] args) {
        SpringApplication.run(StudentappApplication.class, args);  // ← Starts the app
    }

}
```

This is the **entry point** of every Spring Boot application.

### @SpringBootApplication — What Does It Do?

`@SpringBootApplication` is actually a combination of **3 annotations**:

| Annotation | What it does |
|---|---|
| `@SpringBootConfiguration` | Marks this class as a configuration class (like Spring XML config, but in Java) |
| `@EnableAutoConfiguration` | Tells Spring Boot to auto-configure beans based on classpath |
| `@ComponentScan` | Scans all classes in the same package and sub-packages for Spring components |

> **Exam Tip:** "What does @SpringBootApplication consist of?" is a frequently asked question. Remember all 3.

### SpringApplication.run()

```java
SpringApplication.run(StudentappApplication.class, args);
```

This single line:
1. Creates the **Spring IoC Container** (ApplicationContext)
2. Performs **Auto Configuration**
3. Starts the **embedded Tomcat server** on port **8080**
4. Scans for components (`@Controller`, `@Service`, `@Repository`, etc.)

**Compare to old Tomcat startup:** Earlier you started Tomcat manually and deployed your WAR. Here, one line of code does everything.

## 9. application.properties — The Config File

Located at: `src/main/resources/application.properties`

This file replaces all your old configuration — JDBC connection strings, Tomcat port, etc.

### Common Properties

```properties
# Change server port (default is 8080)
server.port=9090

# Application name
spring.application.name=studentapp

# Database configuration (replaces your JDBC getConnection() code)
spring.datasource.url=jdbc:mysql://localhost:3306/studentdb
spring.datasource.username=root
spring.datasource.password=1234
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver

# JPA / Hibernate settings
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true

# Logging level
logging.level.org.springframework=INFO
```

> **Exam Tip:** Know the property key names. Especially `server.port`, `spring.datasource.url`, `spring.datasource.username`, `spring.datasource.password`.

### Old Way vs application.properties

**Old JDBC connection code (you wrote this in every DAO):**
```java
String url = "jdbc:mysql://localhost:3306/studentdb";
String user = "root";
String pass = "1234";
Connection con = DriverManager.getConnection(url, user, pass);
```

**Spring Boot application.properties:**
```properties
spring.datasource.url=jdbc:mysql://localhost:3306/studentdb
spring.datasource.username=root
spring.datasource.password=1234
```

Spring Boot reads this and creates the connection automatically. You never write `getConnection()` again.

## 11. Adding Your First REST Controller (Quick Preview)

After the project is running, let's add one controller to verify everything works:

```java
package com.college.studentapp;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {

    @GetMapping("/hello")
    public String sayHello() {
        return "Hello from Spring Boot!";
    }
}
```

Open browser → `http://localhost:8080/hello`  
You will see: **Hello from Spring Boot!** 

This is the Spring Boot version of writing a `doGet()` method in a Servlet — but much simpler.

> **Controllers, @GetMapping, @PostMapping, etc. are covered in detail on Day 4.**

## 12. JAR vs WAR Packaging — Important Difference

| Feature | JAR | WAR |
|---|---|---|
| Full form | Java ARchive | Web ARchive |
| Used in | **Spring Boot** (default) | Traditional Servlet/JSP |
| Includes | Embedded Tomcat | No server included |
| How to run | `java -jar app.jar` | Deploy on external Tomcat |
| Portability | Self-contained, runs anywhere | Needs Tomcat to be set up |
| Spring Boot default | **Yes** | Optional |

> **Exam Tip:** Spring Boot defaults to **JAR** packaging because it includes embedded Tomcat. WAR is used only when you need to deploy to an existing external Tomcat server.

## 13. How Spring Boot Finds Your Classes — @ComponentScan

When `SpringApplication.run()` is called, Spring Boot automatically scans for classes annotated with:

- `@RestController`
- `@Service`
- `@Repository`
- `@Component`

**Important rule:** These classes must be in the **same package** or **sub-packages** of the main class.

```
com.college.studentapp                  ← Main class is here
com.college.studentapp.controller       ← @RestController here — FOUND 
com.college.studentapp.service          ← @Service here — FOUND 
com.college.studentapp.repository       ← @Repository here — FOUND 
com.other.package                       ← @Component here — NOT FOUND 
```

> **Exam Tip:** A very common exam/viva question — "Why is my controller not working?" Answer: Because it's outside the base package of the main class, so `@ComponentScan` doesn't detect it.

## 14. Summary — What Happens When Spring Boot Starts

```
main() method is called
        ↓
SpringApplication.run() executes
        ↓
Spring IoC Container (ApplicationContext) is created
        ↓
@EnableAutoConfiguration reads classpath → auto-configures beans
        ↓
@ComponentScan scans packages → finds @Controller, @Service, @Repository
        ↓
Embedded Tomcat starts on port 8080
        ↓
App is ready to handle requests 
```

## 15. Exam Questions & Model Answers

**Q1. What is Spring Initializr? How is it used?**

**Ans:** Spring Initializr is a web-based project generator available at `https://start.spring.io`. It is used to create a Spring Boot project by selecting project type (Maven/Gradle), language (Java), Spring Boot version, metadata (groupId, artifactId), and required dependencies. It generates a ZIP file containing a fully configured, ready-to-run Spring Boot project structure.

---

**Q2. Explain the folder structure of a Spring Boot project.**

**Ans:** A Spring Boot Maven project has the following structure:
- `src/main/java` — Contains all Java source files (controllers, services, repositories)
- `src/main/resources` — Contains `application.properties`, static files (CSS/JS), and templates
- `src/test/java` — Contains test classes
- `pom.xml` — Maven configuration file that manages dependencies
- `application.properties` — Central configuration file for server port, database settings, etc.

---

**Q3. What is @SpringBootApplication? What annotations does it combine?**

**Ans:** `@SpringBootApplication` is the main annotation placed on the entry-point class of a Spring Boot application. It is a combination of three annotations:
1. `@SpringBootConfiguration` — Marks the class as a Spring configuration class
2. `@EnableAutoConfiguration` — Enables auto-configuration of Spring beans based on classpath
3. `@ComponentScan` — Scans the current package and sub-packages for Spring components

---

**Q4. What is pom.xml? Explain its important sections.**

**Ans:** `pom.xml` (Project Object Model) is the Maven configuration file used to manage project dependencies, build settings, and project metadata. Important sections:
- `<groupId>`, `<artifactId>`, `<version>` — Identify the project
- `<parent>` — Inherits default Spring Boot dependency versions from `spring-boot-starter-parent`
- `<dependencies>` — Lists all required libraries (starters, etc.)
- `<build>` — Contains the Maven plugin to package the app as a runnable JAR

---

**Q5. What is the difference between JAR and WAR packaging in Spring Boot?**

**Ans:**

| JAR | WAR |
|---|---|
| Default in Spring Boot | Used for traditional web apps |
| Includes embedded Tomcat | Needs external Tomcat |
| Run with `java -jar` | Deploy on Tomcat server |
| Self-contained | Depends on server |

Spring Boot uses JAR by default because the embedded Tomcat is already bundled inside.

---

**Q6. What does SpringApplication.run() do?**

**Ans:** `SpringApplication.run()` is the starting point of a Spring Boot application. When called, it:
1. Creates the Spring IoC Container (ApplicationContext)
2. Performs Auto Configuration based on classpath dependencies
3. Starts the embedded Tomcat server on port 8080 (by default)
4. Scans for Spring components using `@ComponentScan`
5. Makes the application ready to handle HTTP requests

---

**Q7. What is Maven? How does it help in Spring Boot development?**

**Ans:** Maven is a build tool that automates dependency management, compilation, and packaging. In Spring Boot development, Maven helps by:
- Automatically downloading required JAR files (starters, libraries) from Maven Central
- Avoiding version conflicts using `spring-boot-starter-parent`
- Packaging the application into a runnable JAR using the Spring Boot Maven plugin
- Running the app using `./mvnw spring-boot:run`
