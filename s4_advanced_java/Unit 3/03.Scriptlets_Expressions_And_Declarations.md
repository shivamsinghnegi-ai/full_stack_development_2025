# Scriptlets, Expressions & Declarations

---

## What are JSP Scripting Elements?

JSP Scripting Elements are **special tags** that allow you to write Java code inside an HTML page.

There are **3 types:**

| Type | Tag | Purpose |
|------|-----|---------|
| Declaration | `<%! %>` | Declare variables and methods |
| Scriptlet | `<% %>` | Write Java logic / statements |
| Expression | `<%= %>` | Print / display a value |

> Simple rule to remember:
> - `<%! %>` → Declare (with `!`)
> - `<% %>` → Do (logic)
> - `<%= %>` → Display (with `=`)

---

---

# 1. Declaration Tag `<%! %>`

---

## What is it?

A Declaration is used to **declare variables and methods** that belong to the entire JSP class.

## Syntax

```jsp
<%! variable or method declaration here %>
```

## Where does it go in the generated Servlet?

Declarations are placed **outside** the `_jspService()` method — meaning they become **instance variables or methods** of the Servlet class.

```
Generated Servlet Class
├── <%! %>  ← Declaration goes HERE (class level)
└── _jspService() {
        <% %>   ← Scriptlet goes here
        <%= %>  ← Expression goes here
    }
```

---

## Example 1 — Declaring a Variable

```jsp
<%@ page language="java" contentType="text/html; charset=UTF-8" %>

<%! int counter = 0; %>          <!-- declared at class level -->

<!DOCTYPE html>
<html>
<body>
    <h2>Declaration Example</h2>
    <p>Counter value: <%= counter %></p>
</body>
</html>
```

**Output:**
```
Counter value: 0
```

---

## Example 2 — Declaring a Method

```jsp
<%@ page language="java" contentType="text/html; charset=UTF-8" %>

<%!
    // declaring a method using Declaration tag
    String greet(String name) {
        return "Hello, " + name + "! Welcome to JSP.";
    }
%>

<!DOCTYPE html>
<html>
<body>
    <h2>Method Declaration Example</h2>
    <p><%= greet("Vikas") %></p>
    <p><%= greet("Rahul") %></p>
</body>
</html>
```

**Output:**
```
Hello, Vikas! Welcome to JSP.
Hello, Rahul! Welcome to JSP.
```

---

## Key Points about Declaration

- Uses `<%! %>` tag
- Variables declared here are **shared across all requests** (instance variables)
- Methods declared here can be **called anywhere** in the same JSP
- Placed **outside** `_jspService()` in the generated Servlet
- Declared only **once** when JSP is loaded

# 2. Scriptlet Tag `<% %>`

---

## What is it?

A Scriptlet is used to write **any Java code / logic** inside a JSP page.

## Syntax

```jsp
<% java code here %>
```

## Where does it go in the generated Servlet?

Scriptlets go **inside** the `_jspService()` method — meaning they run on **every request**.

---

## Example 1 — Basic Scriptlet

```jsp
<%@ page language="java" contentType="text/html; charset=UTF-8" %>
<!DOCTYPE html>
<html>
<body>

<%
    String name = "Vikas";
    int age = 21;
    out.println("<p>Name: " + name + "</p>");
    out.println("<p>Age: " + age + "</p>");
%>

</body>
</html>
```

**Output:**
```
Name: Vikas
Age: 21
```

---

## Example 2 — if/else in Scriptlet

```jsp
<%@ page language="java" contentType="text/html; charset=UTF-8" %>
<!DOCTYPE html>
<html>
<body>

<%
    int marks = 75;
%>

<h2>Result Card</h2>

<%
    if (marks >= 40) {
%>
        <p style="color:green;">Result: PASS </p>
<%
    } else {
%>
        <p style="color:red;">Result: FAIL </p>
<%
    }
%>

<p>Marks: <%= marks %></p>

</body>
</html>
```

**Output:**
```
Result: PASS 
Marks: 75
```

> Notice how the `if/else` is split across HTML — the HTML between `{` and `}` is also part of the output. This is a very common JSP pattern.

---

## Example 3 — for loop in Scriptlet

```jsp
<%@ page language="java" contentType="text/html; charset=UTF-8" %>
<!DOCTYPE html>
<html>
<body>

<h2>Multiplication Table of 5</h2>

<table border="1">
<%
    for (int i = 1; i <= 10; i++) {
%>
    <tr>
        <td>5 × <%= i %></td>
        <td><%= 5 * i %></td>
    </tr>
<%
    }
%>
</table>

</body>
</html>
```

**Output:** A table showing 5×1=5, 5×2=10 ... 5×10=50

---

## Example 4 — Reading Request Parameters

```jsp
<%@ page language="java" contentType="text/html; charset=UTF-8" %>
<!DOCTYPE html>
<html>
<body>

<%
    String username = request.getParameter("username");
    if (username == null) {
        username = "Guest";
    }
%>

<h2>Welcome, <%= username %>!</h2>

</body>
</html>
```

**Test URL:** `http://localhost:8080/jsp-unit3-1.0-SNAPSHOT/index.jsp?username=Vikas`

**Output:**
```
Welcome, Vikas!
```

---

## Key Points about Scriptlet

- Uses `<% %>` tag
- Can write **any valid Java code** inside it
- Runs **every time** a request comes in
- Has access to JSP **implicit objects** like `request`, `response`, `out`, `session`
- Placed **inside** `_jspService()` in the generated Servlet

# 3. Expression Tag `<%= %>`

---

## What is it?

An Expression is used to **print / display a value** directly in the HTML output.

## Syntax

```jsp
<%= expression here %>
```

> **Important rules:**
> - No semicolon `;` at the end inside `<%= %>`
> - Only **one expression** per tag
> - The expression must return a value (String, int, etc.)

---

## Example 1 — Print a simple value

```jsp
<%@ page language="java" contentType="text/html; charset=UTF-8" %>
<!DOCTYPE html>
<html>
<body>
    <p>Today's date: <%= new java.util.Date() %></p>
    <p>2 + 3 = <%= 2 + 3 %></p>
    <p>Uppercase: <%= "hello".toUpperCase() %></p>
</body>
</html>
```

**Output:**
```
Today's date: Fri Mar 06 10:30:00 IST 2026
2 + 3 = 5
Uppercase: HELLO
```

---

## Example 2 — Expression with variables

```jsp
<%@ page language="java" contentType="text/html; charset=UTF-8" %>

<%
    String city = "Bhavnagar";
    int population = 600000;
%>

<!DOCTYPE html>
<html>
<body>
    <h2>City Info</h2>
    <p>City: <%= city %></p>
    <p>Population: <%= population %></p>
    <p>City length: <%= city.length() %> characters</p>
</body>
</html>
```

**Output:**
```
City: Bhavnagar
Population: 600000
City length: 9 characters
```

---

## Expression vs out.println()

Both do the same thing — print output. But Expression is cleaner:

```jsp
<!-- Using Scriptlet with out.println -->
<% out.println("<p>Hello World</p>"); %>

<!-- Using Expression (cleaner way) -->
<p><%= "Hello World" %></p>
```

> Always prefer `<%= %>` over `out.println()` for displaying values — it is cleaner and easier to read.

---

## Key Points about Expression

- Uses `<%= %>` tag
- **No semicolon** at the end
- Automatically converts the value to **String** and prints it
- Used only to **display** values — not for logic
- Equivalent to `out.print(expression)` in the generated Servlet

# All 3 Together — Combined Example

```jsp
<%@ page language="java" contentType="text/html; charset=UTF-8" %>

<%-- Declaration: define a method --%>
<%!
    String getGrade(int marks) {
        if (marks >= 90) return "A+";
        else if (marks >= 75) return "A";
        else if (marks >= 60) return "B";
        else if (marks >= 40) return "C";
        else return "F";
    }
%>

<!DOCTYPE html>
<html>
<head><title>Student Report</title></head>
<body>

<h2>Student Report Card</h2>

<%-- Scriptlet: write logic --%>
<%
    String[] students = {"Vikas", "Rahul", "Priya", "Anjali"};
    int[]    marks    = {92, 78, 45, 55};
%>

<table border="1" cellpadding="8">
    <tr>
        <th>Name</th>
        <th>Marks</th>
        <th>Grade</th>
        <th>Result</th>
    </tr>

    <%
        for (int i = 0; i < students.length; i++) {
    %>
    <tr>
        <td><%= students[i] %></td>               <%-- Expression --%>
        <td><%= marks[i] %></td>                  <%-- Expression --%>
        <td><%= getGrade(marks[i]) %></td>        <%-- Expression calling Declaration method --%>
        <td>
            <% if (marks[i] >= 40) { %>
                <span style="color:green;">PASS</span>
            <% } else { %>
                <span style="color:red;">FAIL</span>
            <% } %>
        </td>
    </tr>
    <%
        }
    %>
</table>

</body>
</html>
```

**Output in browser:**

| Name | Marks | Grade | Result |
|------|-------|-------|--------|
| Vikas | 92 | A+ | PASS |
| Rahul | 78 | A | PASS |
| Priya | 45 | C | PASS |
| Anjali | 55 | C | PASS |

# JSP Comment Tag

JSP has its own comment tag that is **different from HTML comments**:

```jsp
<%-- This is a JSP comment — NOT sent to browser --%>

<!-- This is HTML comment — IS sent to browser (user can see in View Source) -->
```

> Always use `<%-- --%>` for comments in JSP — they are more secure because they never reach the browser.

# JSP Implicit Objects

Inside Scriptlets and Expressions, you get these objects **for free** — no need to create them:

| Object | Type | Purpose |
|--------|------|---------|
| `request` | HttpServletRequest | Get data sent by user |
| `response` | HttpServletResponse | Send data back to user |
| `out` | JspWriter | Print output to page |
| `session` | HttpSession | Store user session data |
| `application` | ServletContext | App-wide shared data |
| `config` | ServletConfig | Servlet config info |
| `pageContext` | PageContext | Access all other objects |
| `page` | Object | Current JSP page itself |
| `exception` | Throwable | Error handling (error pages only) |

**Most commonly used:**

```jsp
<%
    // request - get data from URL or form
    String name = request.getParameter("name");

    // session - store login info
    session.setAttribute("loggedInUser", name);

    // out - print to page
    out.println("<p>Welcome " + name + "</p>");

    // application - count total visitors
    Integer visits = (Integer) application.getAttribute("visits");
    if (visits == null) visits = 0;
    application.setAttribute("visits", ++visits);
%>

<p>Total site visits: <%= application.getAttribute("visits") %></p>
```

# Important Points for Exam

1. Three scripting elements: **Declaration** (`<%! %>`), **Scriptlet** (`<% %>`), **Expression** (`<%= %>`)
2. Declaration → placed **outside** `_jspService()` → instance level
3. Scriptlet → placed **inside** `_jspService()` → runs every request
4. Expression → **no semicolon** → automatically prints the value
5. Expression is equivalent to `out.print()` in Servlet
6. JSP comment `<%-- --%>` is **not sent** to browser
7. HTML comment `<!-- -->` **is sent** to browser
8. JSP has **9 implicit objects** available inside scriptlets and expressions

---

## Quick Difference Table

| Feature | Declaration `<%! %>` | Scriptlet `<% %>` | Expression `<%= %>` |
|---------|---------------------|-------------------|---------------------|
| Purpose | Declare variable/method | Write Java logic | Display a value |
| Semicolon |  Required |  Required |  Not allowed |
| Position in Servlet | Outside `_jspService()` | Inside `_jspService()` | Inside `_jspService()` |
| Runs | Once (class load) | Every request | Every request |
| Can write if/for/while |  No |  Yes |  No |
| Returns output |  No |  No |  Yes |
