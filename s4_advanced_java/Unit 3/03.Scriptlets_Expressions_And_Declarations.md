# Scriptlets, Expressions & Declarations

---

## 1. JSP Scripting Elements

There are **3 types** of scripting elements in JSP:

| Element | Syntax | Purpose |
|---|---|---|
| **Scriptlet** | `<% ... %>` | Write Java **statements** (logic) |
| **Expression** | `<%= ... %>` | **Print/display** a value |
| **Declaration** | `<%! ... %>` | Declare **variables and methods** |

---

### 1.1 Scriptlets `<% ... %>`

Used to write **Java code (statements)** inside a JSP page.

**Syntax:**
```jsp
<% Java statements here %>
```

**Example 1 — Simple loop:**
```jsp
<html>
<body>
    <h2>Numbers 1 to 5:</h2>
    <%
        for (int i = 1; i <= 5; i++) {
    %>
        <p>Number: <%= i %></p>
    <%
        }
    %>
</body>
</html>
```

**Example 2 — Variables and conditions:**
```jsp
<%
    String name = "Vikas";
    int marks = 85;
%>

<h1>Welcome, <%= name %></h1>

<%
    if (marks >= 75) {
%>
    <p>Result: Distinction </p>
<%
    } else {
%>
    <p>Result: Pass</p>
<%
    }
%>
```

> [!NOTE]
> Scriptlet code goes inside the `_jspService()` method of the generated Servlet. So variables declared in scriptlets are **local variables**.

---

### 1.2 Expressions `<%= ... %>`

Used to **print/output** a value directly to the browser. No semicolon needed!

**Syntax:**
```jsp
<%= Java expression %>
```

**Example:**
```jsp
<html>
<body>
    <p>Today's Date: <%= new java.util.Date() %></p>
    <p>Sum of 10 + 20 = <%= 10 + 20 %></p>
    <p>Your Name: <%= request.getParameter("name") %></p>
</body>
</html>
```

> [!IMPORTANT]
> **Exam Tip:** `<%= x %>` is equivalent to `out.print(x)` in a Servlet. **Never put a semicolon** inside an expression tag!

**Comparison:**
```jsp
<!-- These two are EQUIVALENT -->
<%= "Hello World" %>
<% out.print("Hello World"); %>
```

---

### 1.3 Declarations `<%! ... %>`

Used to declare **instance variables** and **methods** that belong to the Servlet class (not inside `_jspService()`).

**Syntax:**
```jsp
<%! variable or method declaration %>
```

**Example 1 — Instance variable:**
```jsp
<%! int count = 0; %>

<html>
<body>
    <h2>Page visited: <%= ++count %> times</h2>
</body>
</html>
```

> This `count` persists across requests because it's an **instance variable** of the Servlet class.

**Example 2 — Method declaration:**
```jsp
<%!
    public int square(int n) {
        return n * n;
    }
%>

<html>
<body>
    <p>Square of 5 = <%= square(5) %></p>
    <p>Square of 9 = <%= square(9) %></p>
</body>
</html>
```

---

### Scriptlet vs Declaration — Key Difference (Exam Favorite!)

| Feature | Scriptlet `<% %>` | Declaration `<%! %>` |
|---|---|---|
| Where in generated Servlet? | Inside `_jspService()` method | **Outside** `_jspService()`, at class level |
| Variable scope | **Local** to each request | **Instance** variable (shared across requests) |
| Can define methods? | No | Yes |
| Use case | Writing logic/statements | Declaring variables & helper methods |

**Example to understand the difference:**
```jsp
<!-- SCRIPTLET: local variable — resets every request -->
<% int a = 0; %>
<p>Scriptlet count: <%= ++a %></p>
<!-- Always prints 1 -->

<!-- DECLARATION: instance variable — persists across requests -->
<%! int b = 0; %>
<p>Declaration count: <%= ++b %></p>
<!-- Prints 1, then 2, then 3... on each refresh -->
```

## 2. JSP Implicit Objects

JSP provides **9 built-in objects** that you can use without declaring them:

| Object | Type | Purpose |
|---|---|---|
| `request` | HttpServletRequest | Read form data, parameters |
| `response` | HttpServletResponse | Send response to client |
| `out` | JspWriter | Print output to browser |
| `session` | HttpSession | Store session data |
| `application` | ServletContext | Application-wide data |
| `config` | ServletConfig | Servlet configuration |
| `pageContext` | PageContext | Access to all scopes |
| `page` | Object (this) | Current Servlet instance |
| `exception` | Throwable | Error handling (error pages only) |

## 3. Quick Revision — Exam Q&A

### Q1: What is JSP? How is it different from Servlet?
> JSP is a server-side technology for creating dynamic web pages using HTML embedded with Java code. Unlike Servlets where you write Java with HTML inside `out.println()`, JSP allows you to write HTML directly with Java code embedded using special tags. JSP is internally converted to a Servlet by the JSP engine.

### Q2: Explain the JSP lifecycle.
> 1) **Translation** — JSP file is converted to a Servlet (.java)
> 2) **Compilation** — The .java file is compiled to .class
> 3) **Class Loading** — The .class is loaded into memory
> 4) **Instantiation** — A Servlet object is created
> 5) **Initialization** — `jspInit()` is called once
> 6) **Request Processing** — `_jspService()` handles each request
> 7) **Destruction** — `jspDestroy()` is called when server shuts down

### Q3: Differentiate between Scriptlet and Declaration.
> Scriptlet (`<% %>`) code is placed inside `_jspService()` and creates local variables. Declaration (`<%! %>`) code is placed at the class level and creates instance variables/methods that persist across requests.

### Q4: What are JSP implicit objects? Name any 5.
> Implicit objects are pre-defined objects available in every JSP page without declaration. Five important ones: `request`, `response`, `session`, `out`, `application`.

### Q5: Write a JSP page that displays numbers 1 to 10.
```jsp
<html>
<body>
    <h2>Numbers 1 to 10</h2>
    <% for (int i = 1; i <= 10; i++) { %>
        <p><%= i %></p>
    <% } %>
</body>
</html>
```
