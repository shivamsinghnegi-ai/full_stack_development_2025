# Expression Language (EL) & JSTL

---

## SECTION A: Expression Language (EL)

---

### 1. What is Expression Language (EL)?

EL is a **simpler way to access data** (variables, beans, request params, etc.) in JSP **without writing Java code**.

> **One-line definition:** EL = a shortcut to access and display data in JSP without scriptlets.

**Syntax:**
```
${ expression }
```

**Why EL?** — Without EL you'd write:
```jsp
<!-- Old way (ugly) -->
<% String name = (String) request.getAttribute("name"); %>
<p><%= name %></p>

<!-- With EL (clean) -->
<p>${name}</p>
```

---

### 2. EL — Accessing Different Scopes

EL automatically searches these scopes **in order**: `page → request → session → application`

| EL Expression | What it accesses |
|---|---|
| `${name}` | Auto-searches all scopes for "name" |
| `${requestScope.name}` | Only request scope |
| `${sessionScope.user}` | Only session scope |
| `${applicationScope.count}` | Only application scope |
| `${pageScope.x}` | Only page scope |

---

### 3. EL — Accessing Request Parameters

```jsp
<!-- URL: login.jsp?username=Vikas&age=21 -->

<p>Name: ${param.username}</p>       <!-- prints: Vikas -->
<p>Age:  ${param.age}</p>            <!-- prints: 21 -->

<!-- For multiple values (checkboxes) -->
<p>First hobby: ${paramValues.hobby[0]}</p>
```

---

### 4. EL — Accessing JavaBean Properties

If a `Student` bean is stored in request scope with property `name` and `marks`:

```jsp
<!-- Stored in servlet: request.setAttribute("s", studentObj); -->

<p>Name: ${s.name}</p>       <!-- calls s.getName() -->
<p>Marks: ${s.marks}</p>     <!-- calls s.getMarks() -->
```

EL uses **dot notation** (`.`) or **bracket notation** (`[]`):
```jsp
${student.name}       <!-- same as -->
${student["name"]}
```

---

### 5. EL — Operators

| Type | Operators | Example |
|---|---|---|
| Arithmetic | `+ - * / %` | `${10 + 5}` → 15 |
| Relational | `== != < > <= >=` | `${age >= 18}` → true |
| Logical | `&& \|\| !` | `${a && b}` |
| Ternary | `? :` | `${marks >= 33 ? 'Pass' : 'Fail'}` |
| Empty | `empty` | `${empty name}` → true if null/empty |

**Examples:**
```jsp
<p>${10 * 20}</p>                          <!-- 200 -->
<p>${empty username ? "Guest" : username}</p>   <!-- Guest if null -->
<p>${marks >= 75 ? "Distinction" : "Pass"}</p>
```

---

### 6. EL — Implicit Objects

| EL Object | Purpose |
|---|---|
| `param` | Request parameters (`param.id`) |
| `paramValues` | Multiple values (`paramValues.hobby[0]`) |
| `requestScope` | Attributes in request scope |
| `sessionScope` | Attributes in session scope |
| `applicationScope` | Attributes in application scope |
| `pageScope` | Attributes in page scope |
| `header` | HTTP request headers |
| `cookie` | Cookie values |
| `initParam` | Context init params |

---

### 7. EL — Quick Exam Example

**Servlet stores data:**
```java
request.setAttribute("username", "Anjali");
request.setAttribute("score", 88);
request.getRequestDispatcher("result.jsp").forward(request, response);
```

**result.jsp using EL:**
```jsp
<html><body>
    <h2>Welcome, ${username}!</h2>
    <p>Your Score: ${score}</p>
    <p>Result: ${score >= 40 ? "PASS" : "FAIL"}</p>
    <p>Name given?: ${empty username ? "No" : "Yes"}</p>
</body></html>
```

>  **Exam Tip:** EL expressions are **read-only** — they can only display data, not modify it. To modify data, use scriptlets or JSTL.

## SECTION B: Core JSTL Tags

---

### 1. What is JSTL?

**JSTL (JavaServer Pages Standard Tag Library)** is a collection of **custom tags** that replace Java scriptlets in JSP with clean, readable tags.

> **One-line definition:** JSTL = ready-made tags for common tasks like loops, conditions, and output — so you avoid Java code in JSP.

**Why JSTL?**
- Cleaner code (no scriptlets)
- Easier to maintain
- Works perfectly with EL

---

### 2. JSTL Libraries Overview

| Library | Prefix | Purpose |
|---|---|---|
| **Core** | `c` | Variables, loops, conditions, redirect |
| Formatting | `fmt` | Date/number formatting, i18n |
| SQL | `sql` | DB queries (rarely used now) |
| XML | `x` | XML processing |
| Functions | `fn` | String functions |

> **Exam Focus:** Only **Core tags** (`c:`) are important for your exam.

---

### 3. How to Include JSTL in JSP

Add this **taglib directive** at the top of every JSP using JSTL:

```jsp
<%@ taglib uri="http://java.sun.com/jsp/jstl/core" prefix="c" %>
```

Also add the JSTL jar (`jstl-1.2.jar`) to your project's `WEB-INF/lib/`.

---

### 4. Core JSTL Tags — All Important Tags

---

#### 4.1 `<c:out>` — Output a value

Safely prints a value (handles null, escapes HTML).

```jsp
<c:out value="${username}" default="Guest" />
<!-- If username is null → prints "Guest" -->
<!-- If username = "Anjali" → prints "Anjali" -->
```

| Attribute | Purpose |
|---|---|
| `value` | Value to display |
| `default` | Shown if value is null |
| `escapeXml` | Escape HTML chars (default: true) |

---

#### 4.2 `<c:set>` — Set a variable

Creates/sets a variable in a scope.

```jsp
<c:set var="name" value="Rohit" scope="session" />
<c:set var="price" value="${basePrice * 1.18}" />

<p>Name: ${name}</p>
<p>Price with GST: ${price}</p>
```

| Attribute | Purpose |
|---|---|
| `var` | Variable name |
| `value` | Value to assign |
| `scope` | `page / request / session / application` (default: page) |

---

#### 4.3 `<c:remove>` — Remove a variable

```jsp
<c:remove var="name" scope="session" />
```

---

#### 4.4 `<c:if>` — Conditional (if only, no else)

```jsp
<c:if test="${marks >= 40}">
    <p>Student has PASSED!</p>
</c:if>

<c:if test="${empty username}">
    <p>Please login first.</p>
</c:if>
```

> **Note:** `<c:if>` has **no else**. Use `<c:choose>` for if-else.

---

#### 4.5 `<c:choose>` — if / else-if / else

```jsp
<c:choose>
    <c:when test="${marks >= 75}">
        <p>Distinction</p>
    </c:when>
    <c:when test="${marks >= 60}">
        <p>First Class</p>
    </c:when>
    <c:when test="${marks >= 40}">
        <p>Pass</p>
    </c:when>
    <c:otherwise>
        <p>Fail</p>
    </c:otherwise>
</c:choose>
```

| Tag | Role |
|---|---|
| `<c:choose>` | Wrapper (like switch) |
| `<c:when test="...">` | Each condition (like if/else-if) |
| `<c:otherwise>` | Default (like else) |

---

#### 4.6 `<c:forEach>` — Loop (Most Important!)

**Loop over a range (like for loop):**
```jsp
<c:forEach var="i" begin="1" end="5">
    <p>Number: ${i}</p>
</c:forEach>
<!-- Prints 1 2 3 4 5 -->
```

**Loop with step:**
```jsp
<c:forEach var="i" begin="0" end="10" step="2">
    <p>${i}</p>   <!-- 0 2 4 6 8 10 -->
</c:forEach>
```

**Loop over a List (from servlet):**
```java
// In Servlet:
List<String> fruits = Arrays.asList("Apple", "Mango", "Banana");
request.setAttribute("fruits", fruits);
```
```jsp
<!-- In JSP: -->
<c:forEach var="fruit" items="${fruits}">
    <p>${fruit}</p>
</c:forEach>
```

**Loop over a List of Objects:**
```java
// Servlet stores: request.setAttribute("students", studentList);
```
```jsp
<table>
  <tr><th>Name</th><th>Marks</th></tr>
  <c:forEach var="s" items="${students}">
    <tr>
        <td>${s.name}</td>
        <td>${s.marks}</td>
    </tr>
  </c:forEach>
</table>
```

**`varStatus` — loop counter/info:**
```jsp
<c:forEach var="item" items="${list}" varStatus="status">
    <p>${status.index} : ${item}</p>   <!-- 0-based index -->
    <p>${status.count} : ${item}</p>   <!-- 1-based count -->
    <p>First? ${status.first}</p>
    <p>Last? ${status.last}</p>
</c:forEach>
```

---

#### 4.7 `<c:forTokens>` — Split and loop a string

```jsp
<c:forTokens var="city" items="Mumbai,Delhi,Pune" delims=",">
    <p>${city}</p>
</c:forTokens>
<!-- Prints: Mumbai, Delhi, Pune -->
```

---

#### 4.8 `<c:redirect>` — Redirect to another page

```jsp
<c:redirect url="login.jsp" />
<c:redirect url="http://www.google.com" />
```

---

#### 4.9 `<c:import>` — Include external content

```jsp
<c:import url="header.jsp" />
<c:import url="https://example.com/data.txt" />
```

---

#### 4.10 `<c:url>` — Build URLs properly

```jsp
<c:url value="profile.jsp" var="link">
    <c:param name="id" value="101" />
</c:url>
<a href="${link}">View Profile</a>
<!-- Output: profile.jsp?id=101 -->
```

### 5. JSTL Core Tags — Quick Reference Table

| Tag | Use | Key Attributes |
|---|---|---|
| `<c:out>` | Display value | `value`, `default` |
| `<c:set>` | Set variable | `var`, `value`, `scope` |
| `<c:remove>` | Delete variable | `var`, `scope` |
| `<c:if>` | If condition | `test` |
| `<c:choose>` | If-else block | — |
| `<c:when>` | Condition in choose | `test` |
| `<c:otherwise>` | Default in choose | — |
| `<c:forEach>` | Loop | `var`, `items`, `begin`, `end`, `step` |
| `<c:forTokens>` | Loop over split string | `var`, `items`, `delims` |
| `<c:redirect>` | Redirect | `url` |
| `<c:import>` | Include content | `url` |
| `<c:url>` | Build URL | `value`, `var` |

### 6. Full Exam Example — EL + JSTL Together

**Servlet (StudentServlet.java):**
```java
List<Student> list = new ArrayList<>();
list.add(new Student("Ankit", 82));
list.add(new Student("Priya", 55));
list.add(new Student("Raj", 35));
request.setAttribute("students", list);
request.setAttribute("subject", "Advanced Java");
request.getRequestDispatcher("students.jsp").forward(request, response);
```

**students.jsp:**
```jsp
<%@ taglib uri="http://java.sun.com/jsp/jstl/core" prefix="c" %>
<html><body>

<h2>Results — ${subject}</h2>

<table border="1">
  <tr><th>#</th><th>Name</th><th>Marks</th><th>Result</th></tr>

  <c:forEach var="s" items="${students}" varStatus="st">
    <tr>
      <td>${st.count}</td>
      <td>${s.name}</td>
      <td>${s.marks}</td>
      <td>
        <c:choose>
          <c:when test="${s.marks >= 75}">Distinction</c:when>
          <c:when test="${s.marks >= 40}">Pass</c:when>
          <c:otherwise>Fail</c:otherwise>
        </c:choose>
      </td>
    </tr>
  </c:forEach>
</table>

</body></html>
```

## Quick Revision Q&A

**Q1: What is Expression Language in JSP?**
> EL (`${ }`) is a simplified syntax to access data (request params, session vars, beans) in JSP without Java scriptlets.

**Q2: What does `${empty name}` do?**
> Returns `true` if `name` is null, empty string, or empty collection. Useful for null checks without Java code.

**Q3: What is JSTL? Why is it used?**
> JSTL is a standard tag library providing tags for common tasks (loops, conditions, output). It replaces Java scriptlets in JSP, making pages cleaner and easier to maintain.

**Q4: Difference between `<c:if>` and `<c:choose>`?**
> `<c:if>` handles a single condition with no else. `<c:choose>` with `<c:when>` and `<c:otherwise>` handles if/else-if/else chains.

**Q5: How do you iterate over a list using JSTL?**
```jsp
<c:forEach var="item" items="${list}">
    <p>${item}</p>
</c:forEach>
```

**Q6: What is `varStatus` in `<c:forEach>`?**
> It provides loop metadata: `index` (0-based), `count` (1-based), `first` (boolean), `last` (boolean).
