# JSP Architecture & Lifecycle

---

## 1. What is JSP?

**JSP (Java Server Pages)** is a server-side technology that allows you to create **dynamic web pages** using HTML + Java code.

> **Think of it simply:** JSP = HTML + Java code embedded inside it.

| Feature | Servlet | JSP |
|---|---|---|
| Writing style | Java code with HTML inside | HTML with Java code inside |
| Ease of use | Harder (lots of `out.println`) | Easier (write HTML directly) |
| File extension | `.java` | `.jsp` |
| Compilation | Developer compiles | **Server auto-compiles** |
| Best for | Business logic (Controller) | Presentation (View) |

> [!IMPORTANT]
> **Exam Tip:** JSP is internally converted into a Servlet by the server. So JSP = Servlet under the hood!

## 2. JSP Architecture

```
┌──────────┐    HTTP Request    ┌──────────────┐
│  Browser │ ──────────────────▶│  Web Server  │
│ (Client) │                    │  (Tomcat)    │
└──────────┘                    └──────┬───────┘
      ▲                                │
      │         HTTP Response          │
      └────────────────────────────────┤
                                       ▼
                                ┌──────────────┐
                                │  JSP Engine  │
                                │(JSP →Servlet)│
                                └──────┬───────┘
                                       │
                                       ▼
                                ┌──────────────┐
                                │  Servlet     │
                                │ (Generated)  │
                                └──────┬───────┘
                                       │
                                       ▼
                                ┌──────────────┐
                                │  Database    │
                                │  (if needed) │
                                └──────────────┘
```

### How it works (step by step):

1. **Client** sends an HTTP request to the server (e.g., `http://localhost:8080/hello.jsp`)
2. **Web Server (Tomcat)** receives the request and identifies it as a `.jsp` file
3. **JSP Engine** translates the `.jsp` file into a **Servlet** (`.java` file)
4. The Servlet is **compiled** into a `.class` file
5. The Servlet is **executed**, and the output (HTML) is sent back to the browser

> [!NOTE]
> **Translation happens only once** — the first time the JSP is requested (or when the JSP file is modified). After that, the compiled Servlet is reused for better performance.

## 3. JSP Lifecycle

The JSP lifecycle has **7 phases**:

```
 ┌─────────────────────────────────────────────────┐
 │  1. Translation  (JSP → Servlet .java file)     │
 │  2. Compilation  (.java → .class file)          │
 │  3. Class Loading (load .class into memory)     │
 │  4. Instantiation (create Servlet object)       │
 │  5. Initialization — jspInit()                  │
 │  6. Request Processing — _jspService()          │  ← Called for EVERY request
 │  7. Destruction — jspDestroy()                  │
 └─────────────────────────────────────────────────┘
```

### Key Lifecycle Methods:

| Method | When Called | Purpose |
|---|---|---|
| `jspInit()` | Once, when JSP is first loaded | Initialize resources (DB connection, etc.) |
| `_jspService(request, response)` | **Every time** a request comes | Process the request and generate response |
| `jspDestroy()` | Once, when server shuts down or JSP is removed | Cleanup resources |

### Lifecycle Diagram:

```
First Request
     │
     ▼
 Translation ──▶ Compilation ──▶ Loading ──▶ Instantiation
                                                    │
                                                    ▼
                                              jspInit()  ← called ONCE
                                                    │
                                                    ▼
                                            _jspService() ← called EVERY request
                                                    │
                                         (on shutdown)
                                                    ▼
                                             jspDestroy() ← called ONCE
```

> [!IMPORTANT]
> **Exam Tip:** The lifecycle is similar to Servlet lifecycle. Key difference: Steps 1-2 (Translation & Compilation) are **extra** in JSP because JSP must first be converted to a Servlet.
