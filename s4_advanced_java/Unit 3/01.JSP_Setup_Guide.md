# JSP Project — Terminal Setup Guide

---

# MAC SETUP

---

## Step 1 — Find Your Tomcat Path

Run this in Terminal to find where Tomcat 9 is installed:

```bash
brew --prefix tomcat@9

# Check the folder looks correct:
ls $(brew --prefix tomcat@9)/libexec

# Expected output:
# bin  conf  lib  logs  webapps  work
```

> Your Tomcat path will be:
> `/opt/homebrew/opt/tomcat@9/libexec`

---

## Step 2 — Create Shortcuts (Do This Once)

Open Terminal and run all these lines together:

```bash
echo 'alias tomcat-start="/opt/homebrew/opt/tomcat@9/libexec/bin/startup.sh"' >> ~/.zshrc
echo 'alias tomcat-stop="/opt/homebrew/opt/tomcat@9/libexec/bin/shutdown.sh"' >> ~/.zshrc
echo 'alias tomcat-deploy="cp target/*.war /opt/homebrew/opt/tomcat@9/libexec/webapps/"' >> ~/.zshrc
echo 'alias tomcat-logs="tail -f /opt/homebrew/opt/tomcat@9/libexec/logs/catalina.out"' >> ~/.zshrc
echo 'alias tomcat-clean="rm -rf /opt/homebrew/opt/tomcat@9/libexec/webapps/jsp-unit3*"' >> ~/.zshrc
source ~/.zshrc
```

### Verify shortcuts work:

```bash
tomcat-start    # Should print "Tomcat started."
tomcat-stop     # Should print config and return to prompt
```

### What each shortcut does:

| Shortcut | What it does |
|----------|-------------|
| `tomcat-start` | Starts Tomcat server |
| `tomcat-stop` | Stops Tomcat server |
| `tomcat-deploy` | Copies WAR file to Tomcat webapps |
| `tomcat-logs` | Shows live Tomcat logs (good for debugging) |
| `tomcat-clean` | Removes old deployed files |

## Step 3 — Create New JSP Project

```bash
# Go to Desktop (or any folder you prefer)
cd ~/Desktop

# Generate Maven web project
mvn archetype:generate \
  -DgroupId=com.student.jsp \
  -DartifactId=jsp-unit3 \
  -DarchetypeArtifactId=maven-archetype-webapp \
  -DinteractiveMode=false

# Open project in VS Code
cd jsp-unit3
code .
```

Your project structure will look like:

```
jsp-unit3/
├── pom.xml
└── src/
    └── main/
        ├── java/               ← Java Servlet files go here
        └── webapp/
            ├── index.jsp       ← JSP files go here
            └── WEB-INF/
                └── web.xml
```

---

## Step 4 — Setup pom.xml

Delete everything in `pom.xml` and paste this:

```xml
<project xmlns="http://maven.apache.org/POM/4.0.0"
  xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
  xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
  http://maven.apache.org/xsd/maven-4.0.0.xsd">

  <modelVersion>4.0.0</modelVersion>
  <groupId>com.student.jsp</groupId>
  <artifactId>jsp-unit3</artifactId>
  <version>1.0-SNAPSHOT</version>
  <packaging>war</packaging>

  <properties>
    <maven.compiler.source>17</maven.compiler.source>
    <maven.compiler.target>17</maven.compiler.target>
    <failOnMissingWebXml>false</failOnMissingWebXml>
  </properties>

  <dependencies>
    <!-- Servlet API -->
    <dependency>
      <groupId>javax.servlet</groupId>
      <artifactId>javax.servlet-api</artifactId>
      <version>4.0.1</version>
      <scope>provided</scope>
    </dependency>

    <!-- JSTL -->
    <dependency>
      <groupId>javax.servlet</groupId>
      <artifactId>jstl</artifactId>
      <version>1.2</version>
    </dependency>

    <!-- MySQL JDBC -->
    <dependency>
      <groupId>mysql</groupId>
      <artifactId>mysql-connector-java</artifactId>
      <version>8.0.33</version>
    </dependency>
  </dependencies>

  <build>
    <plugins>
      <plugin>
        <groupId>org.apache.maven.plugins</groupId>
        <artifactId>maven-war-plugin</artifactId>
        <version>3.3.2</version>
      </plugin>
    </plugins>
  </build>

</project>
```

---

## Step 5 — Setup web.xml

Replace contents of `src/main/webapp/WEB-INF/web.xml` with:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee
         http://xmlns.jcp.org/xml/ns/javaee/web-app_4_0.xsd"
         version="4.0">
</web-app>
```

---

## Step 6 — Write Your First JSP

Replace `src/main/webapp/index.jsp` with:

```jsp
<%@ page language="java" contentType="text/html; charset=UTF-8" %>
<!DOCTYPE html>
<html>
<head>
    <title>JSP Unit 3</title>
</head>
<body>
    <h1>JSP is Working! </h1>
    <p>Server Time: <%= new java.util.Date() %></p>
</body>
</html>
```

---

## Step 7 — Build & Run

Run these commands in Terminal (inside your project folder):

```bash
# 1. Start Tomcat
tomcat-start

# 2. Build the project
mvn clean package

# 3. Deploy to Tomcat
tomcat-deploy

# 4. Open browser and go to:
# http://localhost:8080/jsp-unit3-1.0-SNAPSHOT/
```

You should see **"JSP is Working! "** with the current server time.

# WINDOWS SETUP

---

## Step 1 — Find Your Tomcat Path

Open Command Prompt (`Win + R` → type `cmd` → Enter):

```cmd
:: Check if Tomcat folder exists
dir C:\tomcat9
:: OR
dir C:\apache-tomcat-9.0.x

:: Expected output should show:
:: bin   conf   lib   logs   webapps   work
```

> Note your full Tomcat path. Common locations:
> - `C:\tomcat9`
> - `C:\apache-tomcat-9.0.x`
> - `C:\Program Files\Apache Software Foundation\Tomcat 9.0`

---

## Step 2 — Create Shortcuts (Do This Once)

Open **Notepad**, paste the following content, and save the file as `shortcuts.bat` on your **Desktop**.

> Replace `C:\tomcat9` with **your actual Tomcat path** if it is different.

```bat
@echo off
set TOMCAT_HOME=C:\tomcat9

if "%1"=="start"  call %TOMCAT_HOME%\bin\startup.bat
if "%1"=="stop"   call %TOMCAT_HOME%\bin\shutdown.bat
if "%1"=="deploy" copy /Y target\*.war %TOMCAT_HOME%\webapps\
if "%1"=="logs"   type %TOMCAT_HOME%\logs\catalina.out
if "%1"=="clean"  del /Q %TOMCAT_HOME%\webapps\jsp-unit3*

if "%1"=="" (
  echo.
  echo Available commands:
  echo   shortcuts start   - Start Tomcat
  echo   shortcuts stop    - Stop Tomcat
  echo   shortcuts deploy  - Deploy WAR to Tomcat
  echo   shortcuts logs    - Show Tomcat logs
  echo   shortcuts clean   - Remove old deployment
  echo.
)
```

### Add shortcuts.bat to PATH so it works from anywhere:

1. Press `Win + S` → search **"Environment Variables"**
2. Click **"Edit the system environment variables"**
3. Click **"Environment Variables"** button
4. Under **User Variables** → select `Path` → click **Edit**
5. Click **New** → type `%USERPROFILE%\Desktop`
6. Click OK on all windows
7. **Close and reopen** Command Prompt

### Now use these commands from any folder:

```cmd
shortcuts start     :: Start Tomcat
shortcuts stop      :: Stop Tomcat
shortcuts deploy    :: Deploy WAR file
shortcuts logs      :: View logs
shortcuts clean     :: Remove old deployment
```

# Daily Workflow

Every time you make changes to your code, run these commands:

```bash
# Mac
tomcat-start          # only if Tomcat is not already running
mvn clean package     # build fresh WAR file
tomcat-clean          # remove old deployed version
tomcat-deploy         # deploy new version
# Refresh browser: http://localhost:8080/jsp-unit3-1.0-SNAPSHOT/
tomcat-stop           # when done for the day
```

```cmd
:: Windows
shortcuts start       :: only if Tomcat is not already running

shortcuts clean       :: remove old deployed version
shortcuts deploy      :: deploy new version
:: Refresh browser: http://localhost:8080/jsp-unit3-1.0-SNAPSHOT/

```

# Common Errors & Fixes

### Mac

| Error | Fix |
|-------|-----|
| `tomcat-start: command not found` | Run: `source ~/.zshrc` |
| `Port 8080 already in use` | Run: `kill -9 $(lsof -ti:8080)` |
| `BUILD FAILURE` | Check `pom.xml` for typos |
| 404 in browser | Make sure you ran `tomcat-deploy` after `mvn clean package` |
| Old code still showing | Run `tomcat-clean` then `tomcat-deploy` again |
| Tomcat permission denied | Run: `chmod +x /opt/homebrew/opt/tomcat@9/libexec/bin/*.sh` |

### Windows

| Error | Fix |
|-------|-----|
| `shortcuts` not recognized | Restart CMD after adding Desktop to PATH |
| `mvn` not recognized | Check Maven `bin` folder is in system PATH |
| `Port 8080 in use` | Run: `netstat -ano \| findstr :8080` then `taskkill /PID <number> /F` |
| 404 in browser | Check WAR file is inside `webapps` folder |
| Old code still showing | Run `shortcuts clean` then `shortcuts deploy` again |

### Both Platforms

| Error | Fix |
|-------|-----|
| JSP compiles but shows blank page | Check for Java errors using `tomcat-logs` |
| JSTL tags not working | Check JSTL dependency is in `pom.xml` |
| JDBC connection refused | Make sure MySQL is running on port 3306 |
