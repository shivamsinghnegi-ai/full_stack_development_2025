# Part 1: Servlet Basics

---

**Q1.** What is a Servlet?
> **Answer:** A Servlet is a Java program that runs on a server (not client) and handles client requests to generate dynamic responses.
> 
> **Key characteristics:**
> - Server-side technology (runs on web server like Tomcat)
> - Platform-independent (works on any OS with JVM)
> - Uses request-response model (client sends request, servlet sends response)
> - Part of Java EE (Enterprise Edition)
> - More efficient than CGI scripts (loaded once, handles many requests)

---

**Q2.** What is the Servlet Architecture?
> **Answer:** 
> 1. Client (browser) sends HTTP request to server
> 2. Web server forwards request to Servlet Container
> 3. Container invokes appropriate Servlet
> 4. Servlet processes request (may access database)
> 5. Servlet generates dynamic HTML response
> 6. Server sends response back to client
> 
> **Key components:** Client → Web Server → Servlet Container → Servlet → Database

---

**Q3.** What is the javax.servlet.Servlet interface?
> **Answer:** It's the core interface that all servlets must implement. It defines 5 methods:
> 
> **Lifecycle Methods (3):**
> - `init()` - Initialization
> - `service()` - Request processing
> - `destroy()` - Cleanup
> 
> **Non-Lifecycle Methods (2):**
> - `getServletConfig()` - Returns configuration
> - `getServletInfo()` - Returns servlet info

---

# Part 2: Servlet Lifecycle

---

**Q4.** What is the Servlet Lifecycle?
> **Answer:** The lifecycle describes how a servlet is created, used, and destroyed:
> 
> 1. **Loading:** Servlet class is loaded into memory
> 2. **Instantiation:** Servlet object is created
> 3. **init():** Initialization method called once
> 4. **service():** Called for each client request (multiple times)
> 5. **destroy():** Cleanup method called once before removal
> 
> **Key point:** init() and destroy() run once; service() runs many times.

---

**Q5.** Explain the init() method.
> **Answer:** The initialization method called only ONCE when servlet is first loaded.
> 
> **Purpose:** One-time setup tasks (database connections, loading config)
> 
> **Signature:**
> ```java
> public void init(ServletConfig config)
> ```
> 
> **Key points:**
> - Receives ServletConfig object containing initialization parameters
> - Must store ServletConfig for later use
> - Called before any client requests are processed

---

**Q6.** Explain the service() method.
> **Answer:** The method called EVERY TIME a client request arrives.
> 
> **Purpose:** Handle client requests and generate responses
> 
> **Signature:**
> ```java
> public void service(ServletRequest req, ServletResponse resp) 
>     throws ServletException, IOException
> ```
> 
> **Key points:**
> - Called multiple times (once per request)
> - Receives request and response objects
> - Main business logic goes here
> - Can throw ServletException and IOException

---

**Q7.** Explain the destroy() method.
> **Answer:** The cleanup method called only ONCE before servlet is removed from memory.
> 
> **Purpose:** Release resources (close database connections, files)
> 
> **Signature:**
> ```java
> public void destroy()
> ```
> 
> **Key points:**
> - No parameters and no return value
> - Cannot throw exceptions
> - Called when server shuts down or servlet is unloaded

---

**Q8.** What are the non-lifecycle methods?
> **Answer:**
> 
> **getServletConfig():**
> - Returns the ServletConfig object passed during init()
> - Contains initialization parameters
> ```java
> public ServletConfig getServletConfig()
> ```
> 
> **getServletInfo():**
> - Returns descriptive information about the servlet
> - Used for documentation purposes
> ```java
> public String getServletInfo()
> ```

# Part 3: Servlet Configuration

---

**Q9.** What is web.xml?
> **Answer:** The deployment descriptor file that configures servlets in a web application.
> 
> **Location:** WEB-INF/web.xml
> 
> **Purpose:**
> - Declare servlets (name and class)
> - Map URLs to servlets
> - Configure welcome files
> - Set initialization parameters
> 
> **Key sections:** `<servlet>` for declaration, `<servlet-mapping>` for URL mapping

---

**Q10.** What are the components of web.xml?
> **Answer:**
> 
> **Servlet Declaration:**
> ```xml
> <servlet>
>     <servlet-name>hello</servlet-name>
>     <servlet-class>com.example.HelloServlet</servlet-class>
> </servlet>
> ```
> 
> **URL Mapping:**
> ```xml
> <servlet-mapping>
>     <servlet-name>hello</servlet-name>
>     <url-pattern>/hello</url-pattern>
> </servlet-mapping>
> ```
> 
> **Important:** servlet-name must match in both sections.

---

**Q11.** How does URL mapping work?
> **Answer:** When a request comes:
> 
> 1. Server checks URL pattern (e.g., `/hello`)
> 2. Finds matching `<servlet-mapping>` with that pattern
> 3. Gets `<servlet-name>` from mapping (e.g., "hello")
> 4. Finds `<servlet>` with same name
> 5. Invokes the servlet class specified
> 
> **Example:** `/hello` → maps to servlet "hello" → invokes `HelloServlet.class`

# Part 4: HttpServlet & HTTP Methods

---

**Q12.** What is HttpServlet?
> **Answer:** An abstract class that extends GenericServlet and provides HTTP-specific methods.
> 
> **Advantage over Servlet interface:**
> - Provides separate methods for each HTTP request type
> - No need to check request type manually
> - Easier to implement RESTful services
> 
> **HTTP-specific methods:** doGet(), doPost(), doPut(), doDelete()

---

**Q13.** What is the difference between doGet() and doPost()?
> **Answer:**
> 
> | Feature | doGet() | doPost() |
> |---------|---------|----------|
> | **Data location** | URL (query string) | Request body |
> | **Visibility** | Data visible in URL | Data hidden |
> | **Data limit** | Limited (~2KB) | Unlimited |
> | **Security** | Less secure | More secure |
> | **Bookmarkable** | Yes | No |
> | **Cached** | Can be cached | Not cached |
> | **Use case** | Fetching data | Submitting forms |

---

**Q14.** What are the four HTTP methods and their purposes?
> **Answer:**
> 
> | HTTP Method | Servlet Method | Purpose | SQL Equivalent |
> |-------------|----------------|---------|----------------|
> | GET | doGet() | Retrieve data | SELECT |
> | POST | doPost() | Create new data | INSERT |
> | PUT | doPut() | Update existing data | UPDATE |
> | DELETE | doDelete() | Remove data | DELETE |
> 
> This maps to CRUD operations (Create, Read, Update, Delete).

# Part 5: Request and Response

---

**Q15.** What is ServletRequest?
> **Answer:** An interface that provides information about the client's request.
> 
> **Common methods:**
> - `getParameter("name")` - Get form field value
> - `getAttribute("key")` - Get request attribute
> - `getContentType()` - Get content type
> - `getInputStream()` - Get request body as stream
> 
> **HttpServletRequest** adds HTTP-specific methods like `getMethod()`, `getSession()`.

---

**Q16.** What is ServletResponse?
> **Answer:** An interface used to send response back to the client.
> 
> **Common methods:**
> - `setContentType("text/html")` - Set response type
> - `getWriter()` - Get PrintWriter for text response
> - `getOutputStream()` - Get stream for binary response
> 
> **HttpServletResponse** adds HTTP-specific methods like `sendRedirect()`, `setStatus()`.

---

**Q17.** How to get form data in a Servlet?
> **Answer:** Using `request.getParameter("fieldName")`
> 
> ```java
> String name = request.getParameter("name");
> String email = request.getParameter("email");
> ```
> 
> **Important:** The parameter name must match the HTML form's `name` attribute exactly.

---

**Q18.** How to send HTML response from Servlet?
> **Answer:**
> ```java
> response.setContentType("text/html");
> PrintWriter out = response.getWriter();
> out.println("<h1>Hello World</h1>");
> ```
> 
> **Steps:**
> 1. Set content type to "text/html"
> 2. Get PrintWriter from response
> 3. Use println() to write HTML

# Part 6: Servlet with Database (CRUD)

---

**Q19.** What is CRUD?
> **Answer:** The four basic database operations:
> 
> | Letter | Operation | HTTP Method | SQL Command |
> |--------|-----------|-------------|-------------|
> | C | Create | POST | INSERT |
> | R | Read | GET | SELECT |
> | U | Update | PUT | UPDATE |
> | D | Delete | DELETE | DELETE |

---

**Q20.** How to connect Servlet to MySQL database?
> **Answer:** Create a connection helper class:
> ```java
> public class DBConnection {
>     public static Connection getConnection() {
>         Class.forName("com.mysql.cj.jdbc.Driver");
>         return DriverManager.getConnection(url, user, pass);
>     }
> }
> ```
> 
> Then use in servlet's init(): `conn = DBConnection.getConnection();`

---

**Q21.** How does doGet() implement READ operation?
> **Answer:**
> ```java
> public void doGet(HttpServletRequest req, HttpServletResponse resp) {
>     String sql = "SELECT * FROM users";
>     Statement stmt = conn.createStatement();
>     ResultSet rs = stmt.executeQuery(sql);
>     while(rs.next()) {
>         // Display each row
>     }
> }
> ```
> 
> - Uses SELECT query
> - Returns ResultSet with data
> - Loops through results

---

**Q22.** How does doPost() implement CREATE operation?
> **Answer:**
> ```java
> public void doPost(HttpServletRequest req, HttpServletResponse resp) {
>     String name = req.getParameter("name");
>     String sql = "INSERT INTO users(name) VALUES(?)";
>     PreparedStatement ps = conn.prepareStatement(sql);
>     ps.setString(1, name);
>     ps.executeUpdate();
> }
> ```
> 
> - Gets form data using getParameter()
> - Uses INSERT query with PreparedStatement
> - executeUpdate() runs the query

---

**Q23.** How does doPut() implement UPDATE operation?
> **Answer:**
> ```java
> public void doPut(HttpServletRequest req, HttpServletResponse resp) {
>     String sql = "UPDATE users SET name=? WHERE id=?";
>     PreparedStatement ps = conn.prepareStatement(sql);
>     ps.setString(1, newName);
>     ps.setInt(2, id);
>     ps.executeUpdate();
> }
> ```
> 
> - Uses UPDATE query with WHERE clause
> - Specifies which record to update using ID

---

**Q24.** How does doDelete() implement DELETE operation?
> **Answer:**
> ```java
> public void doDelete(HttpServletRequest req, HttpServletResponse resp) {
>     String sql = "DELETE FROM users WHERE id=?";
>     PreparedStatement ps = conn.prepareStatement(sql);
>     ps.setInt(1, id);
>     ps.executeUpdate();
> }
> ```
> 
> - Uses DELETE query with WHERE clause
> - Removes specific record by ID

# Part 7: Important Concepts

---

**Q25.** What is the difference between Statement and PreparedStatement in Servlets?
> **Answer:**
> 
> | Statement | PreparedStatement |
> |-----------|-------------------|
> | Query as complete string | Uses ? placeholders |
> | Vulnerable to SQL injection | Prevents SQL injection |
> | Compiled each time | Pre-compiled (faster) |
> | For simple queries | For parameterized queries |
> 
> **Always use PreparedStatement** when handling user input!

---

**Q26.** What is the difference between executeQuery() and executeUpdate()?
> **Answer:**
> 
> - **executeQuery():** For SELECT statements
>   - Returns ResultSet containing data
>   - Used in doGet() for reading data
> 
> - **executeUpdate():** For INSERT, UPDATE, DELETE
>   - Returns int (number of rows affected)
>   - Used in doPost(), doPut(), doDelete()

---

**Q27.** What is a Servlet Container?
> **Answer:** A web server component that manages servlet lifecycle and handles requests.
> 
> **Responsibilities:**
> - Loads and instantiates servlets
> - Calls lifecycle methods (init, service, destroy)
> - Creates request/response objects
> - Manages threads for concurrent requests
> 
> **Examples:** Apache Tomcat, Jetty, GlassFish

---

**Q28.** Why is init() called only once?
> **Answer:** Because initialization is expensive (loading drivers, opening connections). Calling it once and reusing the servlet for multiple requests is more efficient.
> 
> **Analogy:** Opening a restaurant once and serving many customers vs. rebuilding the restaurant for each customer.

---

**Q29.** What is the flow when a form is submitted to a Servlet?
> **Answer:**
> 1. User fills form and clicks submit
> 2. Browser sends HTTP request (GET/POST) to server
> 3. Server routes request to Servlet based on URL pattern
> 4. Servlet's doGet()/doPost() method is called
> 5. Servlet processes data, accesses database if needed
> 6. Servlet generates HTML response
> 7. Response sent back to browser

---

**Q30.** What are common Servlet errors and solutions?
> **Answer:**
> 
> | Error | Cause | Solution |
> |-------|-------|----------|
> | 404 Not Found | Wrong URL pattern | Check web.xml mapping |
> | 500 Internal Error | Code exception | Check server logs |
> | ClassNotFoundException | Missing JAR | Add required libraries |
> | NullPointerException | Uninitialized objects | Initialize in init() |

---

**Q32.** Compare Servlet Interface vs HttpServlet.
> **Answer:**
> 
> | Servlet Interface | HttpServlet |
> |-------------------|-------------|
> | Basic interface | Abstract class |
> | Implement 5 methods | Override only needed methods |
> | Single service() method | Separate doGet(), doPost(), etc. |
> | Protocol-independent | HTTP-specific |
> | More code | Less code |
> 
> **Recommendation:** Use HttpServlet for web applications.
