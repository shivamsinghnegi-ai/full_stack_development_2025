# 1. Student Management System

## Problem Statement

Educational institutes need a system to manage student records efficiently. Manual systems lead to data inconsistency and difficulty in updates. This project provides a centralized system where administrators manage student data and users can view records securely.

## Features

Authentication and Authorization

* User registration and login
* Role-based access (ADMIN, USER)

Admin Features

* Add student
* Update student
* Delete student
* View all students
* Search students

User Features

* View students
* View student details

System Features

* Pagination
* Validation
* Error handling

## Database Design

users

* id (Primary Key)
* username (Unique)
* password (Hashed)
* role
* created_at

students

* id (Primary Key)
* name
* email (Unique)
* course
* created_at
* updated_at

# 2. Online Quiz System

## Problem Statement

Educational platforms require systems to conduct quizzes and evaluate users automatically. Manual checking is inefficient and error-prone. This system enables quiz creation, participation, and automatic evaluation.

## Features

Authentication and Authorization

* Login and registration
* Role-based access (ADMIN, USER)

Admin Features

* Create quiz
* Add questions and options
* Edit or delete quiz
* View results

User Features

* Attempt quiz
* Submit answers
* View results

System Features

* Auto evaluation
* Score calculation
* Prevent multiple attempts
* Optional timer

## Database Design

users

* id
* username
* password
* role

quizzes

* id
* title
* description
* created_at

questions

* id
* quiz_id (Foreign Key)
* question_text

options

* id
* question_id (Foreign Key)
* option_text
* is_correct

results

* id
* user_id (Foreign Key)
* quiz_id (Foreign Key)
* score
* attempted_at

# 3. Leave Management System

## Problem Statement

Organizations need a system to manage employee leave requests efficiently. Manual systems lack transparency and tracking. This system automates leave application, approval workflows, and tracking.

## Features

Authentication and Authorization

* Login system
* Role-based access (ADMIN, EMPLOYEE)

Employee Features

* Apply for leave
* View leave history
* Check leave balance

Admin Features

* View all leave requests
* Approve or reject leave
* Manage employees

System Features

* Leave balance calculation
* Status tracking
* Date validation

## Database Design

users

* id
* username
* password
* role

employees

* id
* name
* email
* leave_balance

leaves

* id
* employee_id (Foreign Key)
* start_date
* end_date
* reason
* status
* applied_at
