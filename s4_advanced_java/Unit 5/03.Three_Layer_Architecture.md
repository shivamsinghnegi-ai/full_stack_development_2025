## 2. The 3-Layer Architecture

Every Spring Boot application that connects to a database follows a 3-layer architecture. This is a standard pattern used in real industry projects.

```
HTTP Request from Client (Postman / Browser)
              |
              v
+---------------------------+
|     Controller Layer      |   @RestController
|  Receives HTTP request    |
|  Sends HTTP response      |
+---------------------------+
              |
              | calls
              v
+---------------------------+
|      Service Layer        |   @Service
|  Contains business logic  |
|  Decides what to do       |
+---------------------------+
              |
              | calls
              v
+---------------------------+
|    Repository Layer       |   @Repository / JpaRepository
|  Talks to the database    |
|  Runs SQL (via Hibernate) |
+---------------------------+
              |
              v
         MySQL Database
```

### Why This Separation?

Each layer has one responsibility. This is the Single Responsibility Principle.

| Layer | Responsibility | What it should NOT do |
|---|---|---|
| Controller | Receive request, send response | Business logic, database calls |
| Service | Process data, apply rules | HTTP handling, direct DB access |
| Repository | Database operations only | Business logic, HTTP handling |

If a controller directly queries the database, the code becomes difficult to test, maintain, and change. Keeping layers separate means changing the database logic does not affect the controller, and changing the API does not affect the database logic.

## 4. FILE 1 — Student.java (Entity Class)

Create this file inside the `model` package.

```java
package com.college.studentcrud.model;

import jakarta.persistence.Column;
import jakarta.persistence.Entity;
import jakarta.persistence.GeneratedValue;
import jakarta.persistence.GenerationType;
import jakarta.persistence.Id;
import jakarta.persistence.Table;

@Entity
@Table(name = "students")
public class Student {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private int id;

    @Column(name = "name", nullable = false, length = 100)
    private String name;

    @Column(name = "email", nullable = false, unique = true, length = 150)
    private String email;

    @Column(name = "course", nullable = false, length = 100)
    private String course;

    @Column(name = "phone", length = 15)
    private String phone;

    // Default constructor — required by JPA and @RequestBody
    public Student() {}

    // Parameterized constructor
    public Student(String name, String email, String course, String phone) {
        this.name   = name;
        this.email  = email;
        this.course = course;
        this.phone  = phone;
    }

    // Getters
    public int    getId()     { return id; }
    public String getName()   { return name; }
    public String getEmail()  { return email; }
    public String getCourse() { return course; }
    public String getPhone()  { return phone; }

    // Setters
    public void setId(int id)          { this.id     = id; }
    public void setName(String name)   { this.name   = name; }
    public void setEmail(String email) { this.email  = email; }
    public void setCourse(String c)    { this.course = c; }
    public void setPhone(String p)     { this.phone  = p; }
}
```

## 5. JPA Annotations — Explained One by One

### @Entity

```java
@Entity
public class Student { }
```

Tells JPA: "This Java class represents a database table."

When the application starts, Hibernate reads this annotation and creates the corresponding table in MySQL if it does not exist.

Without `@Entity`, Hibernate ignores the class completely.

### @Table

```java
@Table(name = "students")
public class Student { }
```

Specifies the name of the table in the database.

If `@Table` is not written, the table name defaults to the class name. In this case, the table would be named `Student` (capital S). Writing `@Table(name = "students")` gives it a proper lowercase plural name, which is standard convention.

| @Table attribute | Purpose | Example |
|---|---|---|
| `name` | Table name in DB | `name = "students"` |
| `schema` | Database schema name | `schema = "college_db"` |

### @Id

```java
@Id
private int id;
```

Marks this field as the **Primary Key** of the table. Every Entity must have exactly one `@Id` field. Hibernate maps this to the `PRIMARY KEY` column in MySQL.

### @GeneratedValue

```java
@GeneratedValue(strategy = GenerationType.IDENTITY)
private int id;
```

Tells Hibernate how the primary key value should be generated.

`GenerationType.IDENTITY` means: use the database's built-in auto-increment feature.

In MySQL, this creates: `id INT NOT NULL AUTO_INCREMENT`

You do not send the `id` field when creating a new student. The database assigns it automatically.

**All GenerationType options:**

| Strategy | Meaning |
|---|---|
| `IDENTITY` | Uses database AUTO_INCREMENT (MySQL default) |
| `SEQUENCE` | Uses a database sequence (Oracle, PostgreSQL) |
| `AUTO` | Hibernate chooses the strategy automatically |
| `TABLE` | Uses a separate table to track IDs |

For MySQL, always use `IDENTITY`.

### @Column

```java
@Column(name = "email", nullable = false, unique = true, length = 150)
private String email;
```

Maps a Java field to a specific database column and defines its constraints.

| @Column attribute | Purpose | SQL equivalent |
|---|---|---|
| `name` | Column name in DB | column name |
| `nullable = false` | Column cannot be null | NOT NULL |
| `unique = true` | No duplicate values | UNIQUE |
| `length = 150` | Max character length | VARCHAR(150) |
| `columnDefinition` | Raw SQL type | e.g., `TEXT` |

If `@Column` is not written, the column name defaults to the field name and all constraints are optional.

### What Table Gets Created

Based on the Student class above, Hibernate generates and runs this SQL when the app starts:

```sql
CREATE TABLE students (
    id      INT          NOT NULL AUTO_INCREMENT,
    name    VARCHAR(100) NOT NULL,
    email   VARCHAR(150) NOT NULL UNIQUE,
    course  VARCHAR(100) NOT NULL,
    phone   VARCHAR(15),
    PRIMARY KEY (id)
);
```

You never wrote this SQL. Hibernate generated it from the annotations.

### Why Default Constructor is Required

JPA (Hibernate) creates Java objects from database rows internally. To do this, it first calls the default (no-argument) constructor to create an empty object, then sets each field using reflection. If the default constructor is missing, Hibernate throws an error at startup.

The same applies to `@RequestBody` — Spring uses the default constructor when converting incoming JSON to a Java object.

### Why Getters and Setters are Required

Jackson library (which handles JSON conversion) uses getters to convert a Java object to JSON, and setters to convert incoming JSON to a Java object.

```
Java Object to JSON (response):    Jackson calls getters
JSON to Java Object (request body): Jackson calls default constructor + setters
```

If getters are missing, the JSON response will be empty `{}`.
If setters are missing, all fields in the received object will be null.

## 7. JpaRepository — Detailed Explanation

### What is JpaRepository?

`JpaRepository` is an interface provided by Spring Data JPA. When your repository interface extends it, Spring automatically creates a complete implementation at runtime — you do not write the implementation yourself.

```java
public interface StudentRepository extends JpaRepository<Student, Integer>
```

The two type parameters:
- `Student` — the Entity class this repository manages
- `Integer` — the data type of the primary key (`id` field is `int`)

### Methods You Get for Free

By extending `JpaRepository`, these methods are available immediately:

| Method | SQL Generated | Returns |
|---|---|---|
| `findAll()` | `SELECT * FROM students` | `List<Student>` |
| `findById(id)` | `SELECT * FROM students WHERE id = ?` | `Optional<Student>` |
| `save(student)` | `INSERT` or `UPDATE` (auto-decided) | `Student` |
| `deleteById(id)` | `DELETE FROM students WHERE id = ?` | void |
| `existsById(id)` | `SELECT COUNT(*) WHERE id = ?` | boolean |
| `count()` | `SELECT COUNT(*) FROM students` | long |
| `findAll(Sort)` | `SELECT * ORDER BY ...` | `List<Student>` |
| `saveAll(list)` | Multiple INSERT statements | `List<Student>` |

### How save() Decides INSERT or UPDATE

`save()` checks the `id` field of the object:
- If `id` is 0 or null — the record is new — runs `INSERT`
- If `id` has a value and that record exists — runs `UPDATE`

This is why you do not set the `id` when adding a new student. You let the database assign it via AUTO_INCREMENT, and `save()` returns the saved student with the generated `id`.

### Custom Query Methods — Spring Data JPA Magic

Spring Data JPA reads the method name and generates the SQL automatically. This is called **Derived Query Methods**.

```java
// You write only the method name — no body, no SQL
List<Student> findByCourse(String course);
```

Spring reads: `find` `By` `Course` and generates:
```sql
SELECT * FROM students WHERE course = ?
```

More examples:

| Method Name | SQL Generated |
|---|---|
| `findByCourse(String course)` | `WHERE course = ?` |
| `findByNameContaining(String kw)` | `WHERE name LIKE %?%` |
| `findByEmail(String email)` | `WHERE email = ?` |
| `findByCourseAndName(String c, String n)` | `WHERE course = ? AND name = ?` |
| `findByCourseOrderByName(String c)` | `WHERE course = ? ORDER BY name` |
| `countByCourse(String course)` | `SELECT COUNT(*) WHERE course = ?` |
| `deleteByCourse(String course)` | `DELETE WHERE course = ?` |

### Naming Rules

| Keyword | SQL |
|---|---|
| `findBy` | SELECT WHERE |
| `countBy` | SELECT COUNT WHERE |
| `deleteBy` | DELETE WHERE |
| `Containing` | LIKE %value% |
| `StartingWith` | LIKE value% |
| `EndingWith` | LIKE %value |
| `And` | AND |
| `Or` | OR |
| `OrderBy` | ORDER BY |
| `IgnoreCase` | LOWER() comparison |

## 8. FILE 3 — StudentService.java

Create this file inside the `service` package.

```java
package com.college.studentcrud.service;

import com.college.studentcrud.model.Student;
import com.college.studentcrud.repository.StudentRepository;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import java.util.List;
import java.util.Optional;

@Service
public class StudentService {

    private final StudentRepository studentRepository;

    @Autowired
    public StudentService(StudentRepository studentRepository) {
        this.studentRepository = studentRepository;
    }

    public Student addStudent(Student student) {
        return studentRepository.save(student);
    }

    public List<Student> getAllStudents() {
        return studentRepository.findAll();
    }

    public Optional<Student> getStudentById(int id) {
        return studentRepository.findById(id);
    }

    public List<Student> getStudentsByCourse(String course) {
        return studentRepository.findByCourse(course);
    }

    public List<Student> searchByName(String keyword) {
        return studentRepository.findByNameContaining(keyword);
    }

    public Optional<Student> updateStudent(int id, Student updatedData) {
        Optional<Student> existing = studentRepository.findById(id);
        if (existing.isPresent()) {
            Student s = existing.get();
            s.setName(updatedData.getName());
            s.setEmail(updatedData.getEmail());
            s.setCourse(updatedData.getCourse());
            s.setPhone(updatedData.getPhone());
            return Optional.of(studentRepository.save(s));
        }
        return Optional.empty();
    }

    public boolean deleteStudent(int id) {
        if (studentRepository.existsById(id)) {
            studentRepository.deleteById(id);
            return true;
        }
        return false;
    }

    public long getTotalCount() {
        return studentRepository.count();
    }
}
```

## 9. Service Layer 

### @Service Annotation

```java
@Service
public class StudentService { }
```

Marks this class as a Spring Bean. Spring creates one instance of this class and stores it in the IoC Container. The controller does not create `StudentService` using `new` — Spring injects it through DI.

`@Service` is semantically identical to `@Component`. The difference is only readability — `@Service` communicates that this class contains business logic.

### Constructor Injection

```java
private final StudentRepository studentRepository;

@Autowired
public StudentService(StudentRepository studentRepository) {
    this.studentRepository = studentRepository;
}
```

Spring sees `@Autowired` on the constructor. When creating the `StudentService` bean, Spring looks in the IoC Container for a bean of type `StudentRepository`, finds it (because it is annotated with `@Repository`), and passes it as the constructor argument.

`private final` ensures the dependency cannot be changed after injection.

### updateStudent Method — Why save() Works for Update

```java
Student s = existing.get();       // fetch current record from DB
s.setName(updatedData.getName()); // update the fields
studentRepository.save(s);        // save() sees id exists → runs UPDATE
```

The student object already has an `id`. When `save()` is called with an object that has an existing `id`, Hibernate generates:
```sql
UPDATE students SET name=?, email=?, course=?, phone=? WHERE id=?
```

This is why the same `save()` method handles both INSERT and UPDATE.

## 10. FILE 4 — StudentController.java

Create this file inside the `controller` package.

```java
package com.college.studentcrud.controller;

import com.college.studentcrud.model.Student;
import com.college.studentcrud.service.StudentService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.DeleteMapping;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PathVariable;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.PutMapping;
import org.springframework.web.bind.annotation.RequestBody;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;
import java.util.List;
import java.util.Optional;

@RestController
@RequestMapping("/api/students")
public class StudentController {

    private final StudentService studentService;

    @Autowired
    public StudentController(StudentService studentService) {
        this.studentService = studentService;
    }

    @GetMapping
    public ResponseEntity<List<Student>> getAllStudents() {
        return ResponseEntity.ok(studentService.getAllStudents());
    }

    @GetMapping("/{id}")
    public ResponseEntity<Student> getStudentById(@PathVariable int id) {
        Optional<Student> student = studentService.getStudentById(id);
        if (student.isPresent()) {
            return ResponseEntity.ok(student.get());
        }
        return ResponseEntity.notFound().build();
    }

    @GetMapping("/search")
    public ResponseEntity<List<Student>> getByCourse(@RequestParam String course) {
        return ResponseEntity.ok(studentService.getStudentsByCourse(course));
    }

    @GetMapping("/searchByName")
    public ResponseEntity<List<Student>> searchByName(@RequestParam String name) {
        return ResponseEntity.ok(studentService.searchByName(name));
    }

    @GetMapping("/count")
    public ResponseEntity<String> getCount() {
        return ResponseEntity.ok("Total students: " + studentService.getTotalCount());
    }

    @PostMapping
    public ResponseEntity<Student> addStudent(@RequestBody Student student) {
        return ResponseEntity.status(201).body(studentService.addStudent(student));
    }

    @PutMapping("/{id}")
    public ResponseEntity<Student> updateStudent(
            @PathVariable int id,
            @RequestBody Student updatedData) {
        Optional<Student> result = studentService.updateStudent(id, updatedData);
        if (result.isPresent()) {
            return ResponseEntity.ok(result.get());
        }
        return ResponseEntity.notFound().build();
    }

    @DeleteMapping("/{id}")
    public ResponseEntity<String> deleteStudent(@PathVariable int id) {
        boolean deleted = studentService.deleteStudent(id);
        if (deleted) {
            return ResponseEntity.ok("Student with ID " + id + " deleted successfully.");
        }
        return ResponseEntity.notFound().build();
    }
}
```

## 11. Controller Layer — Key Points

### @RestController

```java
@RestController
public class StudentController { }
```

Combination of `@Controller` and `@ResponseBody`. Every method's return value is written directly to the HTTP response body as JSON. No JSP page is looked up.

### @RequestMapping at Class Level

```java
@RequestMapping("/api/students")
```

Sets the base URL for all methods in this controller. Every method URL is appended to this base. Without this, you would have to write `/api/students` in every single `@GetMapping`, `@PostMapping`, etc.

### ResponseEntity

Allows you to control both the response body and the HTTP status code.

| Method | Status Code | When to Use |
|---|---|---|
| `ResponseEntity.ok(body)` | 200 OK | Successful GET, PUT, DELETE |
| `ResponseEntity.status(201).body(obj)` | 201 Created | Successful POST |
| `ResponseEntity.notFound().build()` | 404 Not Found | ID does not exist |
| `ResponseEntity.badRequest().body(msg)` | 400 Bad Request | Invalid input |

## 16. Questions

**Q1. What is @Entity? What is its purpose in Spring Boot?**

Ans: `@Entity` is a JPA annotation placed on a Java class to indicate that it represents a database table. When Spring Boot starts with `spring.jpa.hibernate.ddl-auto=update`, Hibernate reads all classes annotated with `@Entity` and automatically creates the corresponding tables in the database. Each field in the class maps to a column, and the class name (or name specified in `@Table`) becomes the table name.

---

**Q2. Explain @Id and @GeneratedValue annotations.**

Ans: `@Id` marks a field as the primary key of the entity. Every `@Entity` class must have exactly one `@Id` field. `@GeneratedValue(strategy = GenerationType.IDENTITY)` tells Hibernate to use the database's auto-increment feature to generate primary key values. For MySQL, this maps to `INT AUTO_INCREMENT`. The application does not need to provide the ID when inserting a new record — the database assigns it automatically and the saved object is returned with the generated ID.

---

**Q3. What is JpaRepository? What methods does it provide?**

Ans: `JpaRepository` is an interface provided by Spring Data JPA. When a repository interface extends `JpaRepository<EntityClass, PrimaryKeyType>`, Spring automatically generates a complete implementation at runtime. The following methods are available without writing any code:
- `findAll()` — retrieves all records
- `findById(id)` — retrieves one record by primary key
- `save(entity)` — inserts if new, updates if existing
- `deleteById(id)` — deletes a record by primary key
- `count()` — returns total number of records
- `existsById(id)` — checks if a record exists

---

**Q4. How does Spring Data JPA generate SQL from method names?**

Ans: Spring Data JPA supports Derived Query Methods — you write only the method name following a naming convention, and Spring generates the SQL query automatically. For example, `findByCourse(String course)` generates `SELECT * FROM students WHERE course = ?`. The naming rules are: `findBy` prefix for SELECT, followed by the field name. Keywords like `Containing` (LIKE), `And`, `Or`, `OrderBy` can be chained to build complex queries without writing any SQL.

---

**Q5. Explain the 3-layer architecture in Spring Boot.**

Ans: The 3-layer architecture separates a Spring Boot application into three distinct layers:
- Controller layer (`@RestController`) — handles incoming HTTP requests and sends responses. It does not contain business logic.
- Service layer (`@Service`) — contains business logic, validates data, and decides what operations to perform.
- Repository layer (`@Repository`) — interacts with the database using JPA methods.

This separation follows the Single Responsibility Principle. Each layer has one job. Changes in one layer do not affect the others, making the code easier to maintain, test, and extend.

---

**Q6. Why does the save() method in JpaRepository work for both INSERT and UPDATE?**

Ans: `save()` checks the primary key (`id`) of the entity. If the `id` is 0 or null (a new object), Hibernate executes an INSERT query. If the `id` has a value and that record exists in the database, Hibernate executes an UPDATE query. This logic is built into the `SimpleJpaRepository` implementation that Spring generates automatically. This is why when updating a student, the existing object is first fetched from the database, its fields are updated, and then `save()` is called on the same object — the id is preserved, so UPDATE runs.
