## What is React?

**React** is a JavaScript library used for building user interfaces (UIs) ‚Äî especially for single-page applications (SPAs) where content updates dynamically without reloading the whole page.

- React helps you build dynamic web pages that update automatically when data changes ‚Äî without refreshing the browser.

### Example of React:

```javascript
// Simple React component
function Welcome() {
  return <h1>Hello, World!</h1>;
}
```

## Why React?

React has gained immense popularity due to several key advantages:

### 1. Single Page Application (SPA)

**What is SPA?**

- A web application that loads a single HTML page
- Updates content dynamically without full page reloads
- Provides a smoother, app-like user experience

**Benefits:**

- Faster navigation (no page reloads)
- Better user experience (feels like a native app)
- Reduced server load

**How React Enables SPA:**
**_ React Router handles client-side routing _**

- Client-side routing means handling page navigation inside the browser using JavaScript, instead of asking the server for a new page each time.
- Traditional (Server-Side) Routing
- In normal websites (without React):
- Every time you click a link, the browser requests a new page from the server.
- The entire page reloads, even if most parts stay the same.
- Client-Side Routing (used in React)
- In React (and other modern front-end frameworks):
- The app loads only once (a single HTML page).
- When you click on a link, React changes the view on the client side (in the browser) ‚Äî without reloading the page.
- The URL changes, but no full page refresh happens.
  **_ Components render based on route changes _**
- It means different components are shown (rendered) on the screen depending on the current route (URL).
  **_ Only necessary parts of the page update _**

### 2. Virtual DOM

**What is Virtual DOM?**

- A lightweight JavaScript representation of the real DOM
- React creates a virtual copy of the DOM in memory
- When state changes, React compares (diffs) the new Virtual DOM with the previous one
- Only the differences are updated in the real DOM

![Virtual DOM vs Real DOM](https://miro.medium.com/1*kQZmpJF-78XHOxzbiOUQqg.jpeg)

![Virtual DOM vs Real DOM](https://media.geeksforgeeks.org/wp-content/uploads/20241212235246933476/Browser-DOM-Virtual-DOM.webp)
**Why Virtual DOM is Powerful:**

```
Traditional DOM Update:
1. Find element in DOM
2. Update element
3. Re-render entire page (expensive!)

React Virtual DOM:
1. Update Virtual DOM (fast, in-memory)
2. Compare with previous Virtual DOM (diffing)
--> Diffing means comparing two versions of the Virtual DOM to find out what has changed.
3. Update only changed elements in real DOM (efficient!)
```

**Benefits:**

- **Performance**: Only updates what changed
- **Efficiency**: Batches multiple updates
- What does this mean?
- When you change state many times in quick succession, React doesn‚Äôt update the DOM immediately every single time.
- Instead, it:
- Collects (batches) all the state updates
- Processes them together in one go.
- Updates the Virtual DOM once.
- Applies the minimum required changes to the Real DOM.
- This makes React very fast and efficient.
- **What is ‚ÄúState‚Äù in React ?**
- State means data that can change over time inside a component.
- **Is state like a varibale.**
- Yes ‚Äî State is like a variable, but smarter.
- Define using **useState()**
- When that data (state) changes, React automatically re-renders the component to show the new data on the screen.
- **Optimization**: Minimizes expensive DOM operations
- **Smooth UX**: Faster rendering, less lag

**Example:**

```javascript
// React efficiently updates only the changed parts
function Counter() {
  const [count, setCount] = useState(0);
  // useState returns a two-item array:
  // The current state value (count)
  // A function to update that state (setCount)

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}
// Only the <p> tag updates when count changes, not the entire page
```

- **What does isolation means**

- **Collaboration**: Different developers can work on different components
- **Maintainability**: Easier to debug and update

## JSX Syntax & Rules

### What is JSX?

**JSX (JavaScript XML)** is a syntax extension for JavaScript that allows you to write HTML-like code in JavaScript. It makes React components more readable and easier to write.

### JSX vs HTML

**Similarities:**

- Looks like HTML
- Uses tags and attributes
- Can be nested

**Differences:**

- JSX is JavaScript, not HTML
- Must return a single parent element
- Uses `className` instead of `class`
- Uses `camelCase` for attributes

### JSX Rules

#### 1. Must Return a Single Parent Element

```javascript
// ‚ùå WRONG - Multiple root elements
function App() {
  return (
    <h1>Hello</h1>
    <p>World</p>
  );
}

// ‚úÖ CORRECT - Single parent element
function App() {
  return (
    <div>
      <h1>Hello</h1>
      <p>World</p>
    </div>
  );
}

// ‚úÖ CORRECT - Using React Fragment
function App() {
  return (
    <>
      <h1>Hello</h1>
      <p>World</p>
    </>
  );
}
```

#### 2. Use `className` Instead of `class`

```javascript
// ‚ùå WRONG
<div class="container">Content</div>

// ‚úÖ CORRECT
<div className="container">Content</div>
```

#### 3. Self-Closing Tags Must Have `/`

```javascript
// ‚ùå WRONG
<img src="image.jpg">
<br>
<input type="text">

// ‚úÖ CORRECT
<img src="image.jpg" />
<br />
<input type="text" />
```

#### 4. Use `camelCase` for Attributes

- Attributes = properties of an HTML tag

```javascript
// HTML ‚Üí JSX
// class ‚Üí className
// for ‚Üí htmlFor
// tabindex ‚Üí tabIndex
// onclick ‚Üí onClick
// onchange ‚Üí onChange

// Example
<label htmlFor="email">Email:</label>
<input
  type="text"
  id="email"
  tabIndex={1}
  onChange={handleChange}
/>
```

#### 5. JavaScript Expressions in JSX

Use curly braces `{}` to embed JavaScript:

```javascript
function App() {
  const name = "John";
  const age = 25;
  const isLoggedIn = true;

  return (
    <div>
      <h1>Hello, {name}!</h1>
      <p>You are {age} years old</p>
      <p>Next year you'll be {age + 1}</p>
      {isLoggedIn && <p>Welcome back!</p>}
    </div>
  );
}
```

#### 6. Conditional Rendering

```javascript
function App() {
  const isLoggedIn = true;

  return (
    <div>{isLoggedIn ? <h2>Welcome Back</h2> : <h2>Please Login</h2>}</div>
  );
}
```

#### 7. Rendering Lists

```javascript
function App() {
  const items = ["Apple", "Banana", "Orange"];

  return (
    <ul>
      {items.map((item, index) => (
        <li key={index}>{item}</li>
      ))}
    </ul>
  );
}
```

- **array.map(function(currentValue, index, array) {// return something});**
- index - (optional) The index of the current element
- array - (optional) The original array itself
  **Important:** Always provide a `key` prop when rendering lists!

#### 8. Inline Styles (Object Syntax)

```javascript
function App() {
  const style = {
    color: "blue",
    fontSize: "20px", // camelCase, not font-size
    backgroundColor: "yellow",
  };

  return <div style={style}>Styled div</div>;

  // Or inline
  return <div style={{ color: "blue", fontSize: "20px" }}>Styled div</div>;
}
```

#### 9. Comments in JSX

```javascript
function App() {
  return (
    <div>
      {/* This is a JSX comment */}
      <h1>Hello</h1>
      {/* 
        Multi-line
        comment
      */}
    </div>
  );
}
```

### JSX Compilation

JSX is transformed into JavaScript by Babel:

```javascript
// JSX
const element = <h1 className="title">Hello</h1>;

// Compiled to JavaScript
const element = React.createElement("h1", { className: "title" }, "Hello");
```

## Functional Components & Props

### Functional Components

**What are Functional Components?**

- JavaScript functions that return JSX
- Modern, preferred way to write React components
- Simpler and more concise than class components

**Basic Syntax:**

```javascript
// Arrow function component
const Welcome = () => {
  return <h1>Welcome!</h1>;
};

// Regular function component
function Welcome() {
  return <h1>Welcome!</h1>;
}

// With implicit return
const Welcome = () => <h1>Welcome!</h1>;
```

**Component Naming:**

- Must start with uppercase letter
- Use PascalCase (e.g., `UserProfile`, `NavBar`)

### Props (Properties)

**What are Props?**

- Short for "properties"
- Data passed from parent to child components
- Read-only (immutable)
- Make components reusable and dynamic
- **Props** are like arguments passed into a function ‚Äî you can use them, but you shouldn‚Äôt modify them.

**Passing Props:**

```javascript
// Parent component
// App.jsx
import Greeting from "./Greeting";

function App() {
  return <Greeting name="Vikas" />;
}

export default App;


// Child component receiving props
// Greeting.jsx
function Greeting(props) {
  return <h2>Hello, {props.name}! üëã</h2>;
}

export default Greeting;

```

- The App component passes name="Vikas" as a prop.
- The Greeting component receives it as an object called props.
- We access it using {props.name}.

**Passing multiple Props.**

```javascript
// App.jsx
import UserCard from "./UserCard";

function App() {
  return <UserCard name="Riya" age={22} country="India" />;
}

export default App;

// UserCard.jsx
function UserCard(props) {
  return (
    <div>
      <h3>Name: {props.name}</h3>
      <p>Age: {props.age}</p>
      <p>Country: {props.country}</p>
    </div>
  );
}

export default UserCard;

```

**Destructuring Props:**

```javascript
// Method 1: Destructure in function parameters
// UserCard.jsx
function UserCard({ name, age, country }) {
  return (
    <div>
      <h3>{name}</h3>
      <p>Age: {age}</p>
      <p>Country: {country}</p>
    </div>
  );
}

export default UserCard;


// Method 2: Destructure in function body

function UserCard(props) {
  const {name, age, country} = props;
  return (
    <div>
      <h3>{name}</h3>
      <p>Age: {age}</p>
      <p>Country: {country}</p>
    </div>
  );
}

export default UserCard;

```

**Default Props:**

```javascript
// UserCard.jsx
function UserCard({ name, age = 19, country }) {
  return (
    <div>
      <h3>{name}</h3>
      <p>Age: {age}</p>
      <p>Country: {country}</p>
    </div>
  );
}

export default UserCard;

// Or using defaultProps (less common in modern React)
function User({ name, age, email }) {
  return (
    <div>
      <h3>Name: {name}</h3>
      <p>Age: {age}</p>
      <p>Email: {email}</p>
    </div>
  );
}

// Define defaultProps after the component
User.defaultProps = {
  age: 18,
  email: "no-email@example.com",
};

export default User;

```

**Passing Functions as Props:**

```javascript
// Parent
function App() {
  const handleClick = () => {
    alert("Button clicked!");
  };

  return <Button onClick={handleClick} text="Click Me" />;
}

// Child
function Button({ onClick, text }) {
  return <button onClick={onClick}>{text}</button>;
}
```

**Children Prop:**

```javascript
// Parent
function App() {
  return (
    //We are rendering the Card component.
    // Inside its opening and closing tags, we are placing:
    // These nested elements automatically become the children prop for Card.
    <Card>
      <h2>Card Title</h2>
      <p>Card content goes here</p>
    </Card>
  );
}

// Child
function Card({ children }) {
  // We receive the nested JSX (passed inside <Card>...</Card>) as children.
  // {children} is just like any other prop ‚Äî it can hold: Text, JSX elements, Components, Even arrays or fragments of elements
  return <div className="card">{children}</div>;
}
```

**Props are Read-Only:**

```javascript
// ‚ùå WRONG - Don't modify props
function User({ name }) {
  name = name.toUpperCase(); // Error!
  return <h1>{name}</h1>;
}

// ‚úÖ CORRECT - Use props as-is or create new variables
function User({ name }) {
  const upperName = name.toUpperCase();
  return <h1>{upperName}</h1>;
}
```

## React Developer Tools

### What are React Developer Tools?

**React DevTools** is a browser extension that provides debugging and inspection capabilities for React applications. It's an essential tool for React development.

### Installation

**Chrome/Edge:**

1. Visit [Chrome Web Store](https://chrome.google.com/webstore)
2. Search for "React Developer Tools"
3. Click "Add to Chrome"

**Firefox:**

1. Visit [Firefox Add-ons](https://addons.mozilla.org)
2. Search for "React Developer Tools"
3. Click "Add to Firefox"

**Standalone (for other browsers):**

- Can be used as a standalone app
- Works with Safari and other browsers

### Features

#### 1. Component Tree

- Visual representation of component hierarchy
- Shows parent-child relationships
- Displays component names and props

**How to Use:**

1. Open browser DevTools (F12)
2. Click on "Components" tab
3. Navigate through component tree
4. Select a component to inspect

#### 2. Props Inspection

- View all props passed to components
- See prop values in real-time
- Edit props to test different scenarios

**Example:**

```
Selected Component: UserCard
Props:
  name: "John Doe"
  age: 25
  email: "john@example.com"
```

#### 3. State Inspection

- View component state (when using hooks)
- See state changes in real-time
- Edit state values for testing

#### 4. Profiler

- Performance analysis tool
- Records component render times
- Identifies performance bottlenecks
- Shows which components re-render and why

**How to Use:**

1. Open "Profiler" tab
2. Click record button
3. Interact with your app
4. Stop recording
5. Analyze results

#### 5. Component Search

- Search for components by name
- Quickly find specific components
- Navigate large component trees

### Using React DevTools

**Basic Workflow:**

1. **Inspect Components**: Click on components in the tree
2. **View Props**: Check props panel on the right
3. **View State**: Check state/hooks panel
4. **Edit Values**: Modify props/state to test
5. **Highlight Updates**: See which components update

**Tips:**

- Use the search bar to find components quickly
- Right-click components for more options
- Use the profiler to optimize performance
- Check the console for React warnings

### Common Use Cases

**Debugging Props:**

**Tracking State Changes:**

// src/components/Product.js
import React from "react";
import "./Product.css";

function Product({ name, price, image, description, inStock }) {
  return (
    <div className="product-card">
      <div className="product-image-container">
        <img src={image} alt={name} className="product-image" />
        {!inStock && <div className="out-of-stock-badge">Out of Stock</div>}
      </div>
      <div className="product-info">
        <h3 className="product-name">{name}</h3>
        <p className="product-description">{description}</p>
        <div className="product-footer">
          <span className="product-price">${price.toFixed(2)}</span>
          <button
            className={`buy-button ${inStock ? "available" : "disabled"}`}
            disabled={!inStock}
          >
            {inStock ? "Add to Cart" : "Out of Stock"}
          </button>
        </div>
      </div>
    </div>
  );
}

export default Product;
