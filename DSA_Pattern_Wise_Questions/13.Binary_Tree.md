**Binary Trees** follow **Recursive Divide & Conquer** on hierarchical structure — process root + recurse on subtrees. Many problems are solved with DFS (pre/in/post-order) or BFS (level-order). Time is usually O(n) where n is number of nodes.

**Core Mindset**:
- **DFS**: Recursion on left/right subtrees. Great for paths, validation, sums.
- **BFS**: Queue for level-by-level processing. Ideal for levels, shortest path in unweighted tree.
- **BST-specific**: Leverage sorted inorder (left < root < right).
- Always handle base case: `if (!root) return ...;`

**Four main patterns** (covers most Binary Tree questions):

---

## Pattern 1: Tree Traversals (DFS & BFS)
**What happens?**  
Visit all nodes in specific order (pre/in/post for DFS; level for BFS).

**When to spot?**  
- Return all values in certain order  
- “Level order”, “Zigzag”, “Right side view”  
- Need to process every node exactly once

**C++ Template – Recursive DFS (Inorder example)**
```cpp
void inorder(TreeNode* root, vector<int>& res) {
    if (!root) return;
    inorder(root->left, res);  // Left
    res.push_back(root->val);  // Root
    inorder(root->right, res); // Right
}
```

**C++ Template – Iterative BFS (Level Order)**
```cpp
vector<vector<int>> levelOrder(TreeNode* root) {
    vector<vector<int>> res;
    if (!root) return res;
    queue<TreeNode*> q;
    q.push(root);
    while (!q.empty()) {
        int size = q.size();
        vector<int> level;
        for (int i = 0; i < size; ++i) {
            TreeNode* node = q.front(); q.pop();
            level.push_back(node->val);
            if (node->left) q.push(node->left);
            if (node->right) q.push(node->right);
        }
        res.push_back(level);
    }
    return res;
}
```

## Pattern 2: Recursive DFS / Tree Properties (Depth, Diameter, Path)
**What happens?**  
Post-order style: compute for left + right subtrees, then combine at root (height, max path, etc.).

**When to spot?**  
- Max/Min depth, diameter, balance  
- Path sum, max path sum  
- “Longest path”, “Good nodes”, subtree properties

**C++ Template – Max Depth / Height**
```cpp
int maxDepth(TreeNode* root) {
    if (!root) return 0;
    int left = maxDepth(root->left);
    int right = maxDepth(root->right);
    return 1 + max(left, right);
}
```

**C++ Template – Diameter of Binary Tree** (global max)
```cpp
int diameter = 0;
int depth(TreeNode* root) {
    if (!root) return 0;
    int left = depth(root->left);
    int right = depth(root->right);
    diameter = max(diameter, left + right); // path through root
    return 1 + max(left, right);
}
```

## Pattern 3: Binary Search Tree (BST) Properties & Operations
**What happens?**  
Leverage inorder sorted property. Search/insert/delete in O(h), validate BST, kth smallest, etc.

**When to spot?**  
- “Validate BST”, “Kth smallest”, “Range sum”, “Lowest Common Ancestor in BST”

**C++ Template – Validate BST**
```cpp
bool isValidBST(TreeNode* root, long minVal = LONG_MIN, long maxVal = LONG_MAX) {
    if (!root) return true;
    if (root->val <= minVal || root->val >= maxVal) return false;
    return isValidBST(root->left, minVal, root->val) &&
           isValidBST(root->right, root->val, maxVal);
}
```

**C++ Template – Search in BST / LCA in BST**
```cpp
TreeNode* searchBST(TreeNode* root, int val) {
    if (!root || root->val == val) return root;
    return val < root->val ? searchBST(root->left, val) : searchBST(root->right, val);
}
```

## Pattern 4: Lowest Common Ancestor (LCA), Paths & Construction
**What happens?**  
Find paths from root, LCA, or build tree from traversals.

**When to spot?**  
- “Lowest Common Ancestor”, “Path to node”, “Construct from preorder+inorder”

**C++ Template – LCA of Binary Tree**
```cpp
TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
    if (!root || root == p || root == q) return root;
    TreeNode* left = lowestCommonAncestor(root->left, p, q);
    TreeNode* right = lowestCommonAncestor(root->right, p, q);
    if (left && right) return root;
    return left ? left : right;
}
```

**Tips for Beginners (C++)**
- Always handle `nullptr` base case first.
- Use global variables or return multiple values (pair/tuple) for diameter/max-path style problems.
- For BST: Pass min/max bounds or use inorder traversal.
- Draw the tree and simulate recursion for small examples.
- Practice traversals first — they are the foundation.
- Time Complexity: Most are O(n). Space: O(h) for recursion (h = height).
- Common combinations: BFS for levels + DFS for paths.
