
These problems focus on **foundational algorithms and data structures** — Arrays, Strings, Linked Lists, Two Pointers, Hashing, Sliding Window, Greedy, and basic Recursion/Patterns. They are frequently asked in **internship coding rounds** (1st/2nd round) because they test clean implementation, edge cases, and time/space optimization without needing advanced techniques.

### Pattern 1: Two Pointers & Sliding Window / Greedy Pointers
**Core idea**  
Use two pointers (left/right or slow/fast) to traverse arrays/strings efficiently in one or two passes. Often avoids nested loops and achieves O(n) time.

**When to think of this pattern**  
- Sorted arrays, find pair with sum/target, maximize area/volume
- Move/remove elements (zeros, duplicates)
- Palindromes, reverse strings/words, subsequences
- Container/max water type problems

**Real-life analogy**  
Two people walking towards each other on a bridge to meet in the middle, or using two hands to squeeze maximum water from a sponge by adjusting grip.

### Pattern 2: Hashing & Frequency Counting
**Core idea**  
Use HashMap / array / set to track frequencies, complements, or seen elements for O(1) lookups instead of nested loops.

**When to think of this pattern**  
- Anagrams, duplicates, unique/first occurrence
- Count characters, check isomorphism
- Majority element, disappeared numbers

**Real-life analogy**  
Using a dictionary to quickly find if a word exists or count how many times each item appears in a shopping list.

### Pattern 3: Stack, Linked List & Pointer Manipulation
**Core idea**  
Use stack for matching pairs (parentheses, etc.) or reverse traversal. For Linked Lists: slow/fast pointers, reverse in-place, or merge.

**When to think of this pattern**  
- Valid parentheses, cycle detection
- Reverse list/string/words
- Middle element, merge sorted lists

**Real-life analogy**  
Stack of plates (LIFO) for matching open/close actions, or two people walking on a linked chain to find middle/cycle.

### Bonus: Basic Searching & Sorting (Explain Time Complexity!)
**Searching**
1. Linear Search → O(n)
2. **Binary Search (VERY IMPORTANT for internships)** → O(log n) on sorted array. Master left/right/mid calculation and loop conditions.
   - Related: Search Insert Position, Find First and Last Position of Element in Sorted Array.

**Sorting (be ready to code + explain complexity)**
- Bubble Sort → O(n²)
- Selection Sort → O(n²)
- Insertion Sort → O(n²) best for nearly sorted

### Bonus: Recursion Basics & Pattern Printing (Common in College/Internship Rounds)
**Simple Recursion Problems**
- Print numbers 1 to N
- Factorial
- Fibonacci
- Reverse a string using recursion
- Power of a number (xⁿ)

**Pattern Printing (Practice loops + inner/outer bounds)**
These test loop logic, spaces, and alignment — very common in campus internships.

1. **Right-angled Triangle — Stars** (increasing)  
   ```
   *
   **
   ***
   ****
   *****
   ```  
   Outer loop rows 1 to n; inner loop prints row number of stars.

2. **Right-angled Triangle — Numbers** (increasing)  
   ```
   1
   1 2
   1 2 3
   ...
   ```  
   Inner loop prints 1 to row.

3. **Right-angled Triangle — Alphabets** (increasing)  
   Similar, map column to 'A' + offset.

4–6. **Right-Aligned Triangles** (Stars, Numbers, Alphabets) — Add leading spaces `(n - row)`.

7–9. **Full (Centered) Pyramids** (Stars, Numbers, Alphabets) — Leading spaces `(n - row)`, then odd count (2*row-1) items.

10. **Inverted Right-Angled Triangle** (Stars)  
   ```
   *****
   ****
   ***
   **
   *
   ```  
   Inner loop: `n - row + 1` stars.

**Quick Tips for Internship Prep**
- Master **edge cases**: empty array/string, all zeros/duplicates, single element, max/min values.
- Always aim for **optimal time/space** (e.g., O(n) time, O(1) extra space where possible).
- Practice explaining your approach aloud (time complexity, why you chose hash vs two pointers).
- Order: Solve all Easy first → then Medium in each pattern.
- Many problems combine patterns (e.g., Hash + Two Pointers).
- For pattern printing: Draw on paper first, then code loops carefully (watch `<` vs `<=` and spaces).
