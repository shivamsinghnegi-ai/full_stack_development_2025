
**Binary Search** follows **Divide and Conquer** on a **monotonic** search space — halve the possibilities each step → O(log n).

Useful for:
- Exact match / boundary in sorted data
- Optimization (“minimum / maximum something”)
- Feasibility (“is it possible with value X?”)
- Implicitly sorted spaces (rotated, 2D, answer range)

**Four main patterns** (covers 90%+ of BS questions):

---

## Pattern 1: Classic / Standard Binary Search
**What happens?**  
Find exact match in fully sorted array.

**When to spot?**  
- Array/matrix explicitly sorted  
- “Find the index of …”, “Search insert position”

**C++ Template**

```cpp
int search(vector<int>& nums, int target) {
    int left = 0, right = nums.size() - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;     // prevent overflow
        if (nums[mid] == target) return mid;
        else if (nums[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return -1;
}
```

## Pattern 2: Binary Search for Boundaries (Lower/Upper Bound, First/Last Occurrence)
**What happens?**  
Find **leftmost** or **rightmost** valid position (handles duplicates).

**When to spot?**  
- Duplicates, “first / last / leftmost”  
- Peak / mountain / “number of elements ≤ X”

**C++ Template – Lower Bound (First True)**

```cpp
int lowerBound(vector<int>& nums, int target) {
    int left = 0, right = nums.size();          // note: right = n
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] >= target) right = mid;   // try smaller index
        else left = mid + 1;
    }
    return left;
}
```

**C++ Template – Upper Bound (Last True)**

```cpp
int upperBound(vector<int>& nums, int target) {
    int left = 0, right = nums.size();
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] > target) right = mid;
        else left = mid + 1;
    }
    return left - 1;   // last index <= target
}
```

## Pattern 3: Modified Binary Search (Rotated / Bitonic / Custom)
**What happens?**  
Array is **almost sorted** (rotated, mountain, unknown size).

**When to spot?**  
- “Rotated sorted array”, “mountain / bitonic”  
- Custom comparison needed

**C++ Template – Search in Rotated Sorted Array**

```cpp
int search(vector<int>& nums, int target) {
    int left = 0, right = nums.size() - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] == target) return mid;
        
        if (nums[left] <= nums[mid]) {              // left half sorted
            if (nums[left] <= target && target < nums[mid])
                right = mid - 1;
            else
                left = mid + 1;
        } else {                                    // right half sorted
            if (nums[mid] < target && target <= nums[right])
                left = mid + 1;
            else
                right = mid - 1;
        }
    }
    return -1;
}
```

## Pattern 4: Binary Search on Answer (Minimize / Maximize the Answer)
**What happens?**  
Search on **possible answer range** + feasibility check.

**When to spot?**  
- “Minimum / maximum of …”, “smallest / largest such that …”  
- Capacity, speed, days, height, split

**C++ Template**

```cpp
bool feasible(int mid, /* params */) {
    // return true if mid works
    return true;
}

int binarySearchOnAnswer(/* params */) {
    int left = minPossible, right = maxPossible;   // e.g., 1 and max(arr) or sum(arr)
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (feasible(mid)) {
            right = mid;           // try better (smaller)
        } else {
            left = mid + 1;
        }
    }
    return left;
}
```

**Tips for Beginners (C++)**
- Always use `left + (right - left) / 2` → no overflow.
- Decide `left <= right` vs `left < right` carefully (most use `<` for bounds).
- Write `feasible()` function first — it decides everything.
- For answer range: `left = 1` or `min(arr)`, `right = 1e9` / `sum(arr)` / `max(arr)`.
- Draw 5–10 elements and simulate.
- Practice 5 easy + 10 medium per pattern → hards become easy.
- Time: O(log(range) × check cost) — each element pushed/popped at most once.
